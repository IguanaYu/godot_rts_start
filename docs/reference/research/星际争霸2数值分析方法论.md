## 1. 方法论概述

本文档系统性地定义一套可用于分析 RTS 游戏数值的方法，包括战斗力量化、经济建模、科技约束和平衡性评估。每项方法都附有计算公式、使用场景和局限性说明。

## 2. 战斗力量化方法

### 2.1 HP × DPS（基础战斗力量化）

**公式：**
```
UnitPower = HP × DPS
```

**原理：** 一个单位在死亡前能输出的总伤害量，是最基础的单体战力指标。HP 决定存活时间，DPS 决定输出速率，两者的乘积近似于"单位一生能造成的总伤害"。

**适用场景：** 同类型单位横向对比、Tier 基准线建立。

**局限：**
- 不区分近战和远程（近战需要先走到敌人面前才能输出）
- 不考虑 Overkill（过量击杀损耗）
- 不考虑技能价值（EMP、Storm 等不直接反映在面板 DPS 中）
- 不考虑溅射/AOE 效果

### 2.2 eHP（有效 HP）

**公式：**
```
eHP = Shield + (HP / max(AttackDamage - Armor, 0.5) × AttackDamage)
```

**原理：** 护盾不受护甲减免，血量部分每点需要承受攻击力/实际伤害倍的等效生命。例如 100HP+1甲 对 6伤害 = 100/(6-1)×6 = 120 eHP。

**使用方式：**
- eHP 总是针对特定攻击力，没有"绝对 eHP"
- 对比不同护甲下的生存力变化
- 评估护甲升级的边际收益

**标准参照攻击力：** 通常以 Marine 的 6 伤害为基准。

### 2.3 有效战斗力（引入射程/机动/溅射修正）

**公式：**
```
EffectivePower = HP × DPS × RangeMult × SpeedMult × SplashMult
```

**修正系数表：**

| 维度 | 条件 | 系数 | 依据 |
|------|------|------|------|
| 射程 | 近战(0.1) | 0.6 | 需要走位才能输出，实际输出时间折损 |
| 射程 | 中程(4-6) | 1.0 | 基准 |
| 射程 | 远程(7-9) | 1.15 | 白嫖机会更多 |
| 射程 | 超远程(13+) | 1.3 | 几乎无风险输出 |
| 机动 | 慢(<2.5) | 0.8 | 追不上、跑不掉 |
| 机动 | 中等(2.5-3.5) | 1.0 | 基准 |
| 机动 | 快(3.5-4.5) | 1.15 | 追逃优势 |
| 机动 | 极快(>4.5) | 1.3 | 战场选择权 |
| 溅射 | 单体 | 1.0 | 基准 |
| 溅射 | 小范围/线性 | 1.5-2.0 | 理想团战命中 2-3 目标 |
| 溅射 | 大范围 | 2.0-3.0 | 理想团战命中 3-5 目标 |

**注意：** 这些系数是理想值，实际团战中因阵型、操作、地形会有很大偏差。

### 2.4 Lanchester 平方律（部队规模效应）

**公式：**
```
ArmyPower = N² × UnitEffectivePower
```

**推导：** 源自 Lanchester 平方律——A 队战斗力正比于 N²，因为 N 个兵同时输出 N 倍伤害，同时承受伤害分布在 N 个单位上。

**实战含义：**
- 10个 Marine 的战斗力 ≈ 100 个单位的战力
- 5个 Marine 的战斗力 ≈ 25 个单位的战力
- 战力比 4:1 时，强方损失约 13%（用平方律计算残局）

**残局公式：**
```
A_remaining = A_initial × sqrt(1 - B_initial² × B_power / (A_initial² × A_power))
```

**局限：**
- 假设所有单位同时输出（实际有阵型和射程差异）
- 不考虑集火/Overkill
- 不考虑士气、阵型崩溃等非量化因素

### 2.5 性价比模型

**公式：**
```
CostEffectiveness = UnitEffectivePower / Cost
Cost = Mineral + Gas × 2
```

**气体价值系数：** 气体价值约等于矿物的 2 倍。这个系数来自：
- 气体建筑额外需要 75 矿 + 3 农民
- 气体农民占用采矿农民的机会成本
- 实际天梯中气体资源更稀缺

**使用方式：**
- 低 Tier 纯矿兵：CostEffectiveness 应高（6-20 范围）
- 高 Tier 耗气兵：CostEffectiveness 应低（2-7 范围）
- 偏差过大的兵种需要审视：给的是否有足够的隐性价值（AOE、技能、肉盾）

## 3. 经济建模方法

### 3.1 单矿收入模型

**基础公式：**
```
每分钟矿物收入 = 采矿农民数 × 41.0
每分钟气体收入 = 采气农民数 × 37.6 / 3 × 2  (每气矿3农民最优，每气矿约75/分)
```

**饱和曲线（每矿）：**
- 8 水晶矿 + 2 气矿
- 最优采矿农民数 = 16（2倍于水晶数）
- 最优采气农民数 = 3/气矿（循环采集无提升）
- 满负荷：16×41 + 6×37.6 = 656 + 225 = ~881 总资源/分

### 3.2 边际收益分析

**采矿边际收益递减：**

| 农民范围 | 每增1农民收益 | 说明 |
|----------|-------------|------|
| 1-16 | +~41 矿/分 | 线性增长，每个农民效率一致 |
| 17-24 | +~25 矿/分 | 等待距离变长，效率下降 |
| 25+ | +~5 矿/分 | 几乎饱和，接近无效 |

**结论：** 单矿超过 16 农民后，开分矿的 ROI 远高于继续补农民。

### 3.3 多矿经济曲线

**双矿满负荷：** 1310 矿 + 450 气/分
**三矿满负荷：** 1965 矿 + 675 气/分

**从开局到双矿满负荷时间：** 约 5-6 分钟
**期间累积收入：** 约 3000-4000 矿 + 800-1200 气

## 4. 科技树约束建模

### 4.1 真实成本模型

**公式：**
```
TrueCost = DirectCost + TimeCost + OpportunityCost
```

其中：
- `DirectCost` = 矿物 + 气体 × 2
- `TimeCost` = 建造时间 × 当前经济收入率（资源闲置成本）
- `OpportunityCost` = 同样资源投入兵力的潜在战斗力

### 4.2 科技延迟

**最短成型时间：**
```
TimeToField = Σ(所有前置建筑建造时间) + 单位生产时间
```

**科技回本时间：**
```
科技投资回本时间 = 科技投资额 / (单位战斗力提升带来的收益 - 机会成本)
```

SC2 中典型科技回本时间在 20-60 秒。

### 4.3 三条策略曲线

- **爆兵策略**：0 科技投资 → 最早获得战斗力，但 5:00 后经济被拉开
- **经济策略**：优先补农民和开矿 → 5:00 后碾压，但前期薄弱怕 Timing
- **科技策略**：优先升级 → 6:00 后人口效率高，但总兵力少怕一波

**Timing 窗口定义：** 对方战斗力曲线 < 己方战斗力曲线的时间区间。

## 5. 平衡性评估方法

### 5.1 TTK（击杀时间）矩阵

**公式：**
```
TTK(Attacker, Defender) = Defender_eHP(Attacker_Damage) / Attacker_DPS
```

**使用方式：**
- 横向对比：同攻击者对不同目标的击杀速度差异
- 纵向对比：不同攻击者对同一目标的击杀速度
- 交叉对比：找出克制关系（TTK 显著短于平均的配对）

**评估标准：**
- T1 vs T1：TTK 应在 2-6 秒
- T1 vs T3：TTK 应在 15 秒以上（T3 应明显肉）
- 专精克制：克制目标 TTK ≤ 非克制目标 TTK 的 60%

### 5.2 Overkill 指数

**公式：**
```
OverkillIndex = 单发伤害 / 目标HP
EfficiencyLoss = max(单发伤害 - 目标HP, 0) / 单发伤害 × 100%
```

**评估标准：**
- OverkillIndex > 1.5：严重溢出，该兵种对此目标效率低
- OverkillIndex 1.0-1.5：轻微溢出，可接受
- OverkillIndex < 1.0：无溢出

**设计含义：**
- 高伤低频兵种（Siege Tank/DT）对小兵有严重 Overkill
- 低伤高频兵种（Marine/Battlecruiser）几乎没有 Overkill
- Overkill 是"高伤兵种怕杂兵"的数学原因

### 5.3 人口效率

**公式：**
```
DPSPerPop = DPS / Supply
HP×DPS_PerPop = HP × DPS / Supply
```

**SC2 规律：**
- 低 Tier + 纯矿兵：DPS/Pop 高（6-20）
- 高 Tier + 耗气兵：DPS/Pop 低（2-7）
- 高价兵的优势不在面板 DPS，而在 AOE/技能/肉盾/射程

### 5.4 护甲边际收益

**公式：**
```
每级护甲减免比例 = 1 / AttackDamage  （针对特定攻击力）
边际收益 = 当前护甲下减免的伤害 - 上一级护甲下减免的伤害
```

**核心发现：**
- 对低伤害攻击（Marine 6）：每级护甲减免 16.7% → 非线性增长
- 对高伤害攻击（Immortal 20）：每级护甲减免 5%
- 护甲对高频低伤单位的克制是指数级的

### 5.5 造价回归

**公式：**
```
Cost = α × HP + β × DPS + γ × Speed + δ × Range + ε × Supply + ζ
```

**简化版（单因子）：**
```
Cost / (HP × DPS) 的比值反映"每单位面板战斗力的价格"
```

**种族间对比：**
- 同族内所有战斗单位的 HP×DPS/Cost 均值作为基准
- 偏离基准超过 ±30% 的单位需要审视
- 偏差通常由技能价值、AOE 潜力等隐性因素解释

## 6. 全链路整合方法

### 6.1 资源 → 战斗力 转换

```
F(t) = Σ(Resource_i(t - Delay_i)) × ConversionRate_i
```

**简化估算：**
```
F(t) ≈ Income(t) × (1 - TechRatio(t)) × AvgEfficiency × (1 - DeadTime)
```

### 6.2 战斗力-时间曲线

**构建步骤：**
1. 确定运营路径（建造顺序、农民分配）
2. 计算每分钟经济收入
3. 计算每分钟资源投入兵力的比例
4. 用 Cost × AvgEfficiency 换算为战斗力
5. 考虑科技升级对战斗力的非线性提升

### 6.3 三族曲线对比

- **Terran**：均衡爬坡，Stim Timing 提供爆发窗口
- **Protoss**：前期精兵优势，Warp Gate 后爆发增长
- **Zerg**：扩张快后期碾压，前期兵力薄弱

## 8. 局限与注意事项

1. **面板数据 ≠ 实战价值** — AOE、技能、射程等隐性价值无法在面板中体现
2. **团战 ≠ 单挑** — 平方律是近似，实际阵型、集火、微操影响巨大
3. **经济模型简化** — 未考虑农民行走时间、采集延迟、建筑队列等微观因素
4. **不考虑玩家操作** — 高手和普通玩家的单位利用率差异可达 2-3 倍
5. **不考虑地图因素** — 地图大小、路口宽度、高台位置影响实际交战效率

所有分析结果应视为"设计参考线"而非"绝对真理"——数值平衡最终需要实战验证。
