# RTS 小地图功能深度调研

> 对标星际争霸2、魔兽争霸3、亿万僵尸三款经典 RTS 的小地图设计

---

## 一、StarCraft 2 小地图

SC2 的小地图是行业标杆，技术上使用**双渲染通道**：主场景 3D 引擎 + 独立的**正交俯视相机**渲染到 SubViewport 生成地形缩略图底图。

### 功能清单

| 功能 | 实现方式 | 说明 |
|------|----------|------|
| 地形缩略图 | 俯视相机 + SubViewport | 显示悬崖、路径、岩石、瓦斯气泉、水晶矿 |
| 单位点 | 程序化绘制圆点 | 按队伍颜色（最多 8 色），大小根据单位类型微调 |
| 建筑方块 | 程序化绘制矩形 | 比单位大一圈，同样按队伍着色 |
| 英雄/重要单位 | 特殊星形图标 | 比普通单位大，带闪烁边框 |
| 相机视口框 | 白色矩形边框 | 精准对应屏幕可见范围，缩放时实时变化大小 |
| 战争迷雾 | Alpha 遮罩层 | 全黑（未探索）/ 半暗（已探索无视野）/ 正常 |
| 任务目标标记 | 程序化菱形 + 脉冲 | 金色=主目标，蓝色=奖励，绿色=保护，红色=击杀 |
| 攻击波警告 | 红点沿路径移动 | 敌方攻击波在小地图上显示为移动的红点（可被突变禁用） |
| 被攻击警告 | 小地图边缘红色闪烁 | 基地/单位被攻击时，小地图边缘对应方向闪红 |
| 玩家 Ping | 短暂闪光 + 扩散动画 | Alt+左键=攻击ping，Alt+右键=警告ping，Alt+G=前进ping |
| 队友 Ping | 同上 + 队友色 | 显示队友的 ping 位置和类型 |
| 右键移动指令 | 小地图上右键 | 直接在小地图上右键发移动指令 |
| 左键跳转 | 左键点击 | 点击位置成为屏幕中心 |
| 标记快捷键 | Ctrl+左键 | 在小地图上放置持续标记（紫色旗子），队友可见 |
| 缩放 | 滚轮缩放 | 部分版本/Mod 支持 |

### 关键设计细节

- 单位点只有 2-3 像素，但**重要目标**（英雄、虫后等）用 4-5 像素 + 特殊形状
- 脉冲动画使用 **sin 波**驱动大小和 Alpha，频率约 1-2Hz
- 攻击波警告的移动红点带有**淡入淡出轨迹**
- 被攻击警告的红边只在**第一次被攻击时闪烁 2-3 秒**，不是持续闪烁

## 二、Warcraft 3 小地图

与 SC2 同源引擎，但有几个独特设计。

### 功能清单

| 功能 | 实现方式 | 说明 |
|------|----------|------|
| 地形缩略图 | 俯视渲染 | 显示树木、悬崖、水域、路径，比 SC2 更精细 |
| 单位点 | 程序化绘制 | 按**玩家颜色**（12 色），比 SC2 颜色系统更丰富 |
| 英雄标记 | **星形 + 金色边框** | 英雄带金色边框的特殊星形，一眼可辨 |
| 野怪营地 | **黄色/橙色点** | 中立野怪即使没视野也默认可见（WC3 重要设计） |
| 中立建筑 | 特殊颜色 | 商店、雇佣兵营地、地精实验室等有独立标记 |
| 金矿 | 金色闪烁点 | 金矿在小地图上始终可见 |
| 树木/可破坏物 | 绿色块 | 显示树木覆盖区域 |
| 悬崖/地形 | 灰度渐变 | 高地/低地通过颜色深浅区分 |
| 相机视口框 | 白色矩形 | 与 SC2 相同 |
| 战争迷雾 | 黑/半暗 | 与 SC2 相同的双状态迷雾 |
| Ping 信号 | 闪光 + 音效 | Alt+左键发 ping，有不同颜色 |
| 右键指令 | 右键点击 | 发移动/攻击指令 |
| 队伍颜色编码 | 12 色系统 | 比 SC2 的 8 色更多 |

### 与 SC2 的关键差异

| 特性 | WC3 | SC2 |
|------|-----|-----|
| 野怪可见性 | 始终可见（黄点） | 有视野才可见 |
| 英雄标记 | 金色星形，显眼 | 略大的圆点 |
| 资源点 | 金矿永久可见 | 水晶/气泉初始可见 |
| 玩家颜色 | 12 色 | 8 色 |
| 中立建筑 | 有独立标记 | 无 |

## 三、They Are Billions（亿万僵尸）小地图

亿万僵尸的小地图设计**非常不同**，因为信息控制是其核心玩法。

### 底层架构

- 使用**程序化生成的灰度地形图**作为小地图底图
- 没有战争迷雾，取而代之的是 **"黑色未探索区域"**
- 探索过的区域会**永久保持可见**（显示地形和建筑），但失去视野后单位不再更新
- 实际上使用了一个**独立的低分辨率数据图**来驱动小地图显示，而不是直接渲染场景

### 功能清单

| 功能 | 实现方式 | 说明 |
|------|----------|------|
| 地形缩略图 | **灰度地形图** | 显示绿（草地）/ 灰（石地）/ 棕（沙地），非常简洁 |
| 资源点 | 彩色点 | 石头=灰色方块，木头=绿色方块，铁=深色方块，油田=黑色方块 |
| 建筑 | 灰色方块 | 己方建筑显示为灰色/白色方块，非常简约 |
| 单位 | **不显示** | 单位**不**在小地图上显示为独立点！只显示建筑 |
| 敌人 | **红色聚集区域** | 尸潮显示为**红色团块**而不是离散点，这是关键设计 |
| 探索区域 | 灰暗色 | 已探索但无视野的区域，地形可见但动态信息不更新 |
| 未探索区域 | 全黑 | 完全未知，不显示任何信息 |
| 尸潮预警 | **红色边框方向指示** | 尸潮从某个方向来袭时，小地图**边缘对应侧闪红光** |
| 泰坦/重要单位 | 特殊标记 | 有但非常简约 |
| 暂停功能 | 暂停时小地图停止更新 | 核心玩法：暂停时查看小地图规划 |
| 缩放 | 支持滚轮缩放 | 可以放大查看局部细节 |

### 与 SC2/WC3 的关键差异

| 特性 | 亿万僵尸 | SC2/WC3 |
|------|----------|----------|
| 单位显示 | **不显示** | 显示为点 |
| 敌人显示 | **红色团块/区域** | 红色离散点 |
| 地形风格 | 灰度/简约 | 彩色/精细 |
| 战争迷雾 | 无，只有未探索区域 | 黑 + 半暗 |
| 信息控制 | 核心玩法 | 辅助功能 |
| 尸潮预警 | 边框红闪 + 方向 | 攻击波红点路径 |
| 小地图缩放 | 支持 | 基本不支持 |

**设计哲学：** 信息是稀缺资源。你看不到敌人单位的精确位置和数量，只能看到"那个方向有一大坨红色"。这增加了紧张感和战略规划的必要性。

## 五、通用设计模式总结

### 5.1 图层架构

所有三款游戏的小地图都采用**多层叠加**的绘制架构：

```
底层: 地形缩略图（俯视渲染 / 灰度图）
  ↓
中层: 静态游戏元素（资源点、野怪营地、中立建筑）
  ↓
上层: 动态游戏元素（单位、建筑、任务目标）
  ↓
顶层: 临时特效（Ping、攻击警告、脉冲标记）
  ↓
最顶层: 相机视口框 + 边框
```

### 5.2 动画模式

| 动画类型 | 用途 | 实现方式 |
|----------|------|----------|
| 脉冲 | 任务目标、求救信号 | sin(时间) 驱动大小和 Alpha |
| 闪烁 | 警告、被攻击 | 方波（开关切换），约 1-2Hz |
| 扩散环 | Ping 信号 | 半径扩大 + Alpha 衰减 |
| 轨迹 | 攻击波移动 | 路径点插值 + 淡入淡出 |
| 团块 | 尸潮/大量敌人 | 半透明圆块，大小代表数量 |

### 5.3 交互模式

| 交互 | 用途 |
|------|------|
| 左键点击 | 跳转相机到目标位置 |
| 右键点击 | 发移动/攻击指令（SC2/WC3） |
| 滚轮 | 缩放小地图（亿万僵尸） |
| Alt+左键 | 攻击 Ping |
| Alt+右键 | 警告/防御 Ping |
| Ctrl+左键 | 放置持续标记（SC2） |

## 六、对你项目的启示

### 最值得优先补充的功能

1. **被攻击警告（红边闪烁）** — SC2/亿万僵尸都有的功能，被攻击时小地图边缘闪红 + 方向指示。信息密度最高、实现成本最低。

2. **玩家 Ping 集成到小地图** — 已有 Alt+左/右键 ping 系统但只显示在世界空间。SC2 的做法是世界空间 + 小地图**同时**显示 ping。

3. **攻击波路径预警** — 亿万僵尸的红色团块逼近 + 边框方向指示比离散点更直观。SC2 的移动红点也更有预警感。

4. **标记注册 API** — 架构层面的开放接口。有了它，所有未来功能都可以不修改小地图源码。

### 技术上可借鉴的思路

- **亿万僵尸的"团块"方案**：大量敌人在同一区域时，不要画几十个红点，而是画一个**红色半透明圆块**，大小代表数量。
- **SC2 的 sin 波脉冲**：用 `sin()` 统一驱动所有脉冲动画，而不是手写闪烁逻辑。
- **亿万僵尸的灰度地形**：可以用预生成的 `Image` 作为小地图底图，开销比 SubViewport 俯视相机小得多。
