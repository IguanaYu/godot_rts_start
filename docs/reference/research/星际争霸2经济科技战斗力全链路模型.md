## 1. 单位战斗力公式

### 1.1 基础模型：HP × DPS

```
UnitPower = HP × DPS
```

这是最核心的战斗力度量，反映单位在死亡前能输出的总伤害量。

| 单位 | HP | DPS | HP×DPS | 造价(矿+气×2) | 战力/资源 |
|------|-----|-----|--------|--------------|----------|
| Marine | 45 | 9.8 | 441 | 50 | 8.8 |
| Marauder | 125 | 6.7 | 838 | 150 | 5.6 |
| Zealot | 150(eHP) | 13.3 | 1995 | 100 | 20.0 |
| Zergling(×2) | 70 | 20.0 | 1400 | 50 | 28.0 |
| Stalker | 160(eHP) | 11.4 | 1824 | 225 | 8.1 |
| Roach | 145 | 8.0 | 1160 | 125 | 9.3 |
| Immortal | 300(eHP) | 28.8 | 8640 | 475 | 18.2 |
| Ultralisk | 500 | 27.3 | 13650 | 700 | 19.5 |

**重要规律**：近战单位 HP×DPS 效率远高于远程，因为近战需要承受更多伤害才能输出——这个公式没有考虑"能否打到人"。

### 1.2 有效战斗力：引入射程和机动修正

```
EffectivePower = HP × DPS × RangeMult × SpeedMult × SplashMult
```

**射程修正（相对 Marine 的 5 射程为基准 1.0）：**

| 类型 | 射程 | RangeMult | 典型单位 |
|------|------|-----------|---------|
| 近战 | 0.1 | 0.6 | Zealot/Zergling/Ultralisk |
| 短程 | 4-5 | 1.0 | Marine/Roach/Hydralisk |
| 中程 | 6 | 1.1 | Marauder/Stalker |
| 远程 | 7-9 | 1.2 | Ghost/Siege Tank |
| 超远程 | 13-14 | 1.3 | Siege Tank(攻城)/Tempest |

**机动修正（相对 Marine 的 3.15 速度为基准 1.0）：**

| 速度 | SpeedMult | 典型单位 |
|------|-----------|---------|
| <2.5 | 0.8 | Siege Tank/Thor/Colossus |
| 2.5-3.5 | 1.0 | Marine/Marauder/Stalker/Roach |
| 3.5-4.5 | 1.15 | Hellion/Zergling/Mutalisk |
| >4.5 | 1.3 | Zergling(菌毯)/Phoenix |

**溅射/AOE修正：**

| 类型 | SplashMult | 典型单位 |
|------|-----------|---------|
| 单体 | 1.0 | Marine/Marauder/Stalker |
| 线性/小范围 | 1.5-2.0 | Hellion/Baneling/Colossus |
| 大范围 | 2.0-3.0 | Siege Tank(攻城)/HT Storm |
| 无攻击(法系) | 0.3 | Ghost/Raven/Infestor/Viper |

> 注：SplashMult 表示在理想团战中对多个目标的有效伤害倍率，实际值取决于阵型。

### 1.3 战场战斗力：Lanchester 平方律

```
ArmyPower = (UnitCount)^2 × UnitEffectivePower
```

**意义**：10个 Marine 的战斗力不是 1个×10，而是 ×100。
**实际含义**：10个 Marine 可以打赢 5个 Marine（战力比 100:25 = 4:1，剩余约 8.7个）。

**残局计算公式：**

```
A_remaining = A_initial × sqrt(1 - (B_initial^2 × B_power) / (A_initial^2 × A_power))
```

当 A_initial^2 × A_power > B_initial^2 × B_power 时 A 胜。

### 1.4 性价比公式（用于设计决策）

```
CostValue = UnitEffectivePower / (mineral + gas × 2)
```

这是"每单位资源能买到的战斗力"。设计时应确保：
- 低Tier纯矿兵：CostValue 高（6-20），但受人口和科技限制
- 高Tier耗气兵：CostValue 低（2-7），但提供隐性价值（AOE/技能/肉盾）

## 2. 经济产出曲线

### 2.1 单矿收入基准

| 时间点 | 农民数 | 矿物/分钟 | 气体/分钟 | 累计收入(到此时) |
|--------|--------|----------|----------|----------------|
| 0:00 | 12 | 0(起始) | 0 | 0 |
| 1:00 | ~15 | ~300 | 0 | ~200 |
| 2:00 | ~18 | ~450 | ~80(气矿刚好) | ~550 |
| 3:00 | ~22 | ~550 | ~150 | ~1050 |
| 4:00 | ~24(单矿饱和) | ~600 | ~225 | ~1650 |
| 5:00 | 24(饱和) | ~655 | ~225 | ~2300 |

**关键公式：**
```
单矿饱和收入 = 16农民采矿 + 6农民采气
             = ~655 矿物/分钟 + ~225 气体/分钟
             = ~880 总资源/分钟
```

### 2.2 多矿经济曲线

**双矿标准运营（人族 1-1-1 开局为例）：**

| 游戏时间 | 运营事件 | 农民数 | 矿物/分 | 气体/分 | 说明 |
|---------|---------|--------|--------|--------|------|
| 0:00-1:30 | 造农民到16，造气矿 | 16 | ~350 | ~80 | 单矿发展期 |
| 1:30-2:30 | 造兵营/补给站/二基地 | 18 | ~450 | ~150 | 二基地建设中 |
| 2:30-3:30 | 二基地完工，补农民 | 24+12=36 | ~800 | ~225 | 经济爬升期 |
| 3:30-4:30 | 补到双矿饱和 | 32+12=44 | ~1100 | ~225(单气) | 二矿渐满 |
| 4:30-5:30 | 开二气，补气矿农民 | 44+6=50 | ~1200 | ~350 | 双气采集 |
| 5:30+ | 双矿双气全饱和 | 44+12=56 | ~1310 | ~450 | 满负荷 |

**关键洞察**：从开局到双矿满负荷约需 **5-6 分钟**，期间总收入约 **3000-4000 矿物 + 800-1200 气体**。

### 2.3 经济公式总结

```
单矿每分钟收入(16农民+6采气):
  Income_mineral  = 41.0 × 16 = ~656/分
  Income_gas      = 37.6 × 3 × 2 = ~225/分
  Income_total    = ~880/分

N矿每分钟收入(N≥2, 满负荷):
  Income_mineral  = 656 × N
  Income_gas      = 225 × N
  Income_total    = 881 × N

边际农民效率(超过16农民后):
  第17-24农民: 每多1个农民 +~25 矿物/分 (边际收益低)
```

## 3. 科技树约束模型

### 3.1 科技成本 = 直接造价 + 时间成本 + 机会成本

```
TrueCost = Mineral + Gas × 2 + BuildTime × IncomeRate + OpportunityCost
```

其中 OpportunityCost = 这笔资源如果投入兵力能获得的战斗力。

### 3.2 人族关键科技路径与时间线

**1-1-1 开局（Barracks → Factory → Starport）：**

| 时间 | 建造/生产事件 | 累计矿物 | 累计气体 | 战斗力产出 |
|------|-------------|---------|---------|-----------|
| 0:00 | 12 SCV + 50矿 | 50 | 0 | 0 |
| 0:17 | 开始连续造 SCV | — | — | — |
| 0:50 | 造 Supply Depot(100矿, 30s) | 150 | 0 | 0 |
| 1:10 | 造 Barracks(150矿, 60s) | 300 | 0 | 0 |
| 1:20 | 造 Refinery(75矿, 30s) | 375 | 0 | 0 |
| 1:40 | 3 SCV 采气 | — | — | — |
| 2:10 | Barracks 完成 → 造 Marine + Orbital(150矿) | 525 | ~60 | 1 Marine |
| 2:30 | 造 Factory(150矿+100气, 60s) | 675+100气 | ~100 | 2 Marine |
| 3:00 | Orbital 完成 → 造 MULE | — | — | — |
| 3:30 | Factory 完成 → 造 Starport(150+100气) | ~900+200气 | ~180 | 3-4 Marine |
| 4:00 | 造 Reactor(50+50气) 或 Tech Lab | — | ~220 | — |
| 4:30 | Starport 完成 → 造 Medivac(100+100气) | ~1200+300气 | ~300 | 5-6 Marine |
| 5:00 | 开始 Stim 研发(100+100气, 170s) | ~1400+400气 | ~400 | 6-8 Marine |

**到 5:00 时的累计投资：**
- 建筑：~900 矿物 + ~300 气体
- SCV：~400 矿物
- 部队：~400 矿物 + ~100 气体
- 总：~1700 矿物 + ~400 气体
- 战斗部队：6-8 Marine + 1 Medivac = ~400 矿物 + ~100 气体

### 3.3 科技投资与战斗力延迟

**投资延迟公式：**
```
部队成型时间 = Σ(前置建筑时间) + 单位生产时间
战斗力释放 = 在 T+Delay 时刻得到 F 战斗力

"投资效率" = F / (Delay × Cost)
```

| 兵种 | 前置科技 | 最短成型时间 | 前置投资 | 说明 |
|------|---------|-------------|---------|------|
| Marine | 无 | 1:10(Barracks)+0:25=~1:35 | 150矿 | 最快战力 |
| Marauder | Barracks+Tech Lab | 1:10+0:30+0:30=~2:10 | 350+50气 | 需额外时间 |
| Siege Tank | Factory+Tech Lab | 2:30+0:30+0:45=~3:45 | 500+225气 | 中期核心 |
| Ghost | Barracks+Tech Lab+Ghost Acad | 1:10+0:30+0:50+0:36=~3:06 | 725+300气 | 功能型 |
| Battlecruiser | Starport+Fusion Core | 2:30+1:10+0:45=~4:25 | 950+500气 | 大后期 |

### 3.4 资源分配策略模型

**三种极端策略的战斗力曲线对比：**

```
策略A — "极限爆兵"：0科技投资，全部资源造 Marine
  优点：最早获得战斗力（~2:00有第一个兵）
  缺点：5:00后经济被拉开，且无科技加成

策略B — "经济优先"：大量投资农民和分基地
  优点：5:00后经济碾压，兵力持续力强
  缺点：2:00-5:00期间兵力薄弱，怕 Timing Rush

策略C — "科技攀爬"：优先升级和高级兵种
  优点：6:00后高级兵种战力/人口比高
  缺点：兵力总数少，怕被一波淹没
```

**关键概念 — Timing 窗口：**

```
Timing = 对方战斗力曲线 < 己方战斗力曲线 的时间区间
```

- 爆兵策略在 T1-T2 有战斗力优势
- 经济策略在 T2 之后反超
- 科技策略在 T3 之后达到峰值

## 4. 全链路整合模型

### 4.1 资源 → 战斗力 转换公式

```
F(t) = Σ(Resource_i(t - Delay_i)) × ConversionRate_i
```

其中：
- `Resource_i(t - Delay_i)` = 在时间 t-Delay_i 投入的第 i 笔资源
- `ConversionRate_i` = 该笔资源转化为战斗力的效率
- `Delay_i` = 从投资到部队成型的时间延迟

**简化公式：**
```
F(t) ≈ Income(t) × (1 - TechRatio(t)) × AvgEfficiency × (1 - DeadTime)
```

- `Income(t)` = 当前经济产出
- `TechRatio(t)` = 投入科技/经济的资源比例
- `AvgEfficiency` = 当前可用兵种的平均战力/资源比
- `DeadTime` = 资源闲置率（余钱越多说明运营越差）

### 4.2 战斗力-时间标准曲线（Terran 1-1-1）

|------|-----------|---------|---------|-----------------|--------|
| 2:00 | ~500+0 | ~300 | 1 Marine | 441 | — |
| 3:00 | ~800+100 | ~500 | 3 Marine | 1323 | ~440/分 |
| 4:00 | ~1200+200 | ~700 | 5 Marine + 1 Marauder | 2205+838=3043 | ~760/分 |
| 5:00 | ~1700+400 | ~1000 | 8 Marine + 2 Marauder + 1 Medivac | 3528+1676+?=~5300 | ~1325/分 |
| 6:00 | ~2500+700 | ~1600 | 12 Marine + 3 Marauder + 2 Medivac + Stim完成 | ~8000(Stim后~12000) | ~2000/分 |

**Stim 前后对比**：Stim 完成后 Marine 战斗力约提升 60%（攻速+50%），Marauder 约提升 40%。

## 5. 科技投资对战斗力的延迟影响

### 5.1 科技回本时间

```
科技投资回本时间 = 科技投资额 / 单位战斗力提升带来的收益
```

| 科技升级 | 投资(矿+气) | 效果 | 战力提升 | 回本时间(估算) |
|---------|-----------|------|---------|--------------|
| Stim(人族) | 100+100 | Marine +50%攻速 | +60% | ~30秒(接战后) |
| Charge(神族) | 200+200 | Zealot +移速+冲锋 | +80% | ~40秒 |
| Metabolic Boost(虫族) | 100+100 | Zergling +移速 | +100%(追击能力) | ~20秒 |
| +1 攻击 | 100+100 | 全兵种+1伤害 | +10-20% | ~60秒 |
| Blink(神族) | 150+150 | Stalker 闪烁 | +50%(机动) | ~45秒 |

**核心洞察**：科技升级的回本时间通常在 **20-60 秒** 内，前提是你已经在持续生产该兵种。

### 5.2 科技投资的"机会成本"量化

```
如果 5:00 时投资 200 资源（100+100气）研发 Stim：
  
  机会成本 = 200 资源 = 4 个 Marine(不造)
  
  收益 = 所有 Marine 战斗力 +60%
  
  盈亏平衡点：当拥有 ≥7 个 Marine 时，Stim 的收益超过 4 个 Marine
  
  考虑到 Marine 会持续生产，Stim 的投资通常在 6:00 前回本
```

### 5.3 科技等级与战斗力的非线性关系

```
+0 → +1 攻：每枪 Marine 伤害 6→7 (+16.7%)
+1 → +2 攻：每枪 Marine 伤害 7→8 (+14.3%)
+2 → +3 攻：每枪 Marine 伤害 8→9 (+12.5%)

但面对高护甲目标时：
  Marine(6伤) vs 3甲Ultra：每枪 6→3 = 50%伤害
  Marine(+3攻,9伤) vs 3甲Ultra：每枪 9→6 = 100%伤害
  
结论：攻防升级对"射速高单伤低"的单位是**指数级收益**
```

## 6. 应用到本项目（Godot RTS）的设计框架

### 6.1 战斗力公式设计步骤

1. **定义基准单位** — 选一个 T1 兵种作为基准（比如 Marine 等效），设定其 Power = HP × DPS = 1.0
2. **校准射程/速度/溅射修正** — 根据定位给不同的修正系数
3. **用造价反推** — 战斗力应与造价成正比，偏差大的需要审视

```
单位造价 = k × (HP × DPS × RangeMult × SpeedMult × SplashMult)
其中 k 是种族/阵营的经济系数
```

### 6.2 经济产出设计原则

```
每个采集单位每分钟产出 = 基准值（比如 40 单位/分）
每个基地饱和采集 = 基准值 × 基地最大采集单位数

关键约束：
- 单矿满负荷 = 基础兵种可持续生产的资源上限
- 高级兵种 = 需要多矿 + 气体限制
- 气体占比控制强度曲线
```

### 6.3 科技树时间约束设计

```
从开局到第一个兵 = ~60-90秒（给对手反应时间）
从中期到高级兵 = ~3-5分钟（科技投资窗口）
从高级到终极兵 = ~5-8分钟（大后期）
```

**每条科技线应有：**
1. **前置建筑树**（2-3 层深）
2. **累积投资成本**（资源 + 时间）
3. **战斗力释放曲线**（投资后多久形成战力）
4. **Timing 窗口**（什么时间段该科技最强）

### 6.4 "几分钟获得多少战斗力" — 设计检查表

| 检查项 | 问题 | 理想范围 |
|--------|------|---------|
| 首兵时间 | 玩家多久后能造出第一个战斗单位？ | 60-90s |
| 初始战力 | 2:00 时能有多少战斗力？ | 1-2个T1兵 |
| 中期爆发 | 5:00 时能有多少部队？ | 6-10个T1兵或等效 |
| 科技延迟 | 从开局到高级兵多久？ | 4-8分钟 |
| 经济爬坡 | 单矿到双矿满负荷多久？ | 4-6分钟 |
| 战斗力峰值 | 满人口时 HP×DPS 总和？ | 设计目标值 |

## 7. 计算案例：Terran 5:00 Stim Timing 的战斗力

### 7.1 运营路径（1-1-1 标准）

```
0:00 — 12 SCV
0:17 — 连续造 SCV
0:50 — Supply Depot
1:10 — Barracks
1:20 — Refinery
2:10 — Orbital Command (OC) 升级开始 + 第一个 Marine
2:30 — Factory
3:00 — OC 完成 → 第一个 MULE
3:30 — Starport
4:00 — Reactor (Factory)
4:15 — Stim 研发开始
4:30 — Starport 完成 → Medivac
5:00 — 第一个 MULE 用完
```

### 7.2 资源消耗计算

| 类别 | 项目 | 矿物 | 气体 |
|------|------|------|------|
| 建筑 | Supply Depot ×2 | 200 | 0 |
| 建筑 | Barracks | 150 | 0 |
| 建筑 | Factory | 150 | 100 |
| 建筑 | Starport | 150 | 100 |
| 建筑 | Reactor | 50 | 50 |
| 建筑 | Refinery ×2 | 150 | 0 |
| 升级 | Orbital Command | 150 | 0 |
| 单位 | SCV ×10 (12→22) | 500 | 0 |
| 单位 | Marine ×6 | 300 | 0 |
| 单位 | Marauder ×1 | 100 | 25 |
| 单位 | Medivac ×1 | 100 | 100 |
| 科技 | Stim | 100 | 100 |
| **总计** | | **2100** | **475** |

### 7.3 经济产出 vs 消耗

```
5:00 总经济产出(估算): ~2300 矿物 + ~450 气体
消耗: 2100 矿物 + 475 气体
余钱: ~200 矿物 + ~-25 气体(少量赤字)

效率: 2300/2100 = ~91% 矿物利用率 (优秀运营)
```

### 7.4 5:00 战斗力

```
部队:
  6 Marine × 45 × 9.8 = 2646
  1 Marauder × 125 × 6.7 = 838
  1 Medivac (治疗/运输价值 ≈ 1 Marine)
  Stim 研发完成 → 接战后 Marine DPS 翻倍

5:00 面板战斗力: ~3500 (HP×DPS)
5:00 Stim后接战: ~5500 (30秒内爆发)

对比:
  同等经济的神族 ~5:00: ~6000-8000 (Zealot+Stalker)
  同等经济的虫族 ~5:00: ~8000-12000 (Zergling+Roach)
  
→ Terran 5:00 的正面战斗力偏弱，靠 Stim Timing 和操作弥补
→ 这就是为什么 Terran 需要空投/多线来创造优势
```
