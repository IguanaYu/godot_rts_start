## 第一部分：Warcraft 3 案例穷举

## 第三部分：归纳分类（从案例自然形成）

把上面所有案例按"特效的**视觉职责**"归纳，得到 **7 大类别**：

### 类别 A：标识型（Identifier）— "这是谁/什么状态"

> 头顶/身上的小图标和符号，告诉玩家这个单位的身份和状态

- WC3：Aura Buff Indicator / Hero 等级标识 / 眩晕星星 / 任务目标图标
- SC2：Buff bar 图标 / Bunker 驻军指示
- 项目已覆盖：[objective_marker.gd](../../../scripts/effects/objective_marker.gd) 头顶 7 种（CROWN/SHIELD/SKULL/CROSSHAIR/STAR/FLAG/DIAMOND）

### 类别 B：地面贴花型（Ground Decal）— "脚底有什么"

> 单位脚下的圆形/区域贴花，告诉玩家"这个单位站在什么上面/被什么影响"

- WC3：Aura 源头底盘 / Aura 受影响圆环 / Mass Teleport 落点 / 反魔法 Shell / 神圣护盾
- SC2：Selection Circle / Force Field / Guardian Shield / Defensive Matrix / Fungal Growth / Psi Storm
- 项目已覆盖：[outpost_status_marker.gd](../../../scripts/outpost/outpost_status_marker.gd) 4 种 effect_id（inspire/heal/shield/attack）

### 类别 C：身体材质变化型（Material Override）— "整体外观改变"

> 单位 sprite/model 整体染色、变透明、变形

- WC3：受击泛白 / 死亡爆裂 / Demon Hunter 变身 / Polymorph / Invisibility / Bloodlust 红 / Slow 蓝 / Ethereal 紫
- SC2：受击白闪 / 死亡爆裂 / Cloak 半透 / Burrow / Blink / Mind Control 紫 / Corruption 紫
- 项目已覆盖：受击伤害数字（[damage_number.gd](../../../scripts/effects/damage_number.gd)）
- 项目缺失：**受击泛白（P0）** / 死亡爆裂 / 半透 / 染色 tint

### 类别 D：持续状态粒子型（Status Particle）— "持续处于什么状态"

> 围绕单位的持续粒子，告诉玩家"这个单位正在被某种状态影响"

- WC3：中毒紫烟 / 燃烧火 / 冰冻 / 治疗十字 / Bloodlust 红雾 / 净化
- SC2：Stimpack 红脉冲 / Corruption 持续紫 / Baneling 爆浆
- 项目已覆盖：[heal_effect.gd](../../../scripts/effects/heal_effect.gd) 部分治疗粒子
- 项目缺失：中毒/燃烧/冰冻（项目暂无单位级 DOT 状态）

### 类别 E：连接型（Linking Beam）— "两个单位之间有什么"

> 两个单位之间的持续光束/触手，表示"A 正在对 B 做什么"

- WC3：Mind Control 触手 / Spirit Link 灵魂链接
- SC2：Medivac Heal Beam / MULE 修理 beam / Void Ray Prismatic Beam / Neural Parasite 触手
- 项目缺失：治疗光束 / 持续伤害光束（当前单体 heal_effect 够用，未来加持续治疗单位时再补）

### 类别 F：天降/事件型（Sky Event）— "刚刚发生了一个高潮事件"

> 从天上降下的光柱/导弹/陨石，标记"事件 climax"

- WC3：Holy Light 圣光柱 / Resurrection 多光柱 / Mass Teleport 蓝光柱 / Hero Level Up 金光柱 / Summon Infernal 火球坠落
- SC2：Nuclear Missile 红点 / Yamato Cannon 蓄能光柱 / MULE 降落光柱 / Nydus Worm 出土 / Mothership Mass Recall / Warp-in 召唤光柱 / Phoenix Graviton Beam 反重力举起 / Mothership Vortex 紫色漩涡
- 项目缺失：**天降光柱（P1）** — Boss 入场、大战术释放、关卡事件都需要

### 类别 G：区域效果型（Zone Effect）— "这里有什么在持续发生"

> 地面上一片区域持续发生某种效果，单位进入就受影响

- WC3：Blight 蔓延 / Blizard 持续冰柱 / Starfall 流星雨 / Earthquake 地震
- SC2：Psi Storm 闪电区域 / Fungal Growth 真菌 / Creep 蔓延
- 项目已覆盖：[outpost_telegraph_overlay.gd](../../../scripts/outpost/outpost_telegraph_overlay.gd)（据点指挥官技能预警）
- 项目缺失：战斗型 AOE 持续伤害区（Psi Storm 风格）

## 第四部分：项目优先级建议

按"项目当前缺失度 × 实现难度 × 设计价值"评估：

### P0（基础手感缺失，必做）

| 方向 | 类别 | 理由 |
|---|---|---|
| **受击泛白 Hit Flash** | C | SLOW 节奏下每次受击都很重要；目前完全缺失；shader 实现简单（半天工作量） |

### P1（"高潮瞬间"和"特殊感"，强烈推荐）

| 方向 | 类别 | 理由 |
|---|---|---|
| **天降光柱 Pillar of Light** | F | Boss 入场、大战术释放、关卡事件都需要"高潮瞬间"；视觉冲击最强；与项目"玩家主动决策点"原则契合（预警版光柱） |
| **Boss/英雄级发光 Outline + Glow** | C | grand_tactic_releaser / 关键 Boss 需要"特殊感"；SC2 哲学：只给特殊单位发光 |

### P2（设计依赖型，等机制加进来再做）

| 方向 | 类别 | 理由 |
|---|---|---|
| 中毒/燃烧/冰冻持续粒子 | D | 等设计师加 DOT 单位时再补 |
| 治疗光束 / 持续伤害光束 | E | 等加持续治疗/连续伤害单位时再补 |
| 战斗型 AOE 持续区域（Psi Storm 风格） | G | 等加 AOE 持续技能时再补 |

### 已有架构可扩展（加 effect_id 即可）

| 方向 | 现有代码 | 扩展点 |
|---|---|---|
| 地面光环扩展 | [outpost_status_marker.gd](../../../scripts/outpost/outpost_status_marker.gd) | 加 effect_id：`burn`（橙）、`poison`（紫）、`rage`（红）、`slow`（蓝） |
| 头顶图标扩展 | [objective_marker.gd](../../../scripts/effects/objective_marker.gd) | 已支持 7 种 + 3 等级 + danger 切换，可按需加新类型 |

## 第六部分：综合设计原则（基于案例归纳）

从 WC3/SC2 的案例归纳出 5 条通用原则：

1. **职责单一**：每个特效只解决一个信息传达问题（标识 / 地面 / 材质 / 粒子 / 连接 / 天降 / 区域）—— SC2 严格遵守，混搭只在 Boss 大招时出现。
2. **克制原则**：所有单位都发光 = 没有发光；所有受击都强闪 = 玩家麻木。**只给"特殊瞬间/特殊单位"加特效**。
3. **颜色编码统一**：
   - 金色 = 神圣 / 英雄 / 治疗（WC3 Paladin）
   - 紫色 = Boss / 虚空 / 魔法（SC2 Zerg Boss、WC3 Undead）
   - 红色 = 伤害 / 敌方 / 狂暴（Bloodlust）
   - 蓝色 = 护盾 / 我方精英 / 反魔法（WC3 Slow）
   - 绿色 = 自然 / 治疗（SC2 Medivac）
4. **玩家主动决策点**（项目核心原则）：所有"强特效"（天降光柱、Boss 发光）必须配合玩家可反应的窗口——预警期、可打断、可远离。
5. **性能预算**：项目是 2D + SLOW 节奏，理论上可承受更多特效，但单位数量上限要考虑（参考 [mass-battle-test skill](../../../../../.claude/skills/mass-battle-test/) 已有的性能测试方法）。
