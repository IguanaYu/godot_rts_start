# 近战兵打不到人 + 群体作战卡住 — 完整调研报告与实施计划

---

## 第一部分：问题现状分析

### 1.1 表面现象

| 场景 | 现象 |
|------|------|
| 1个士兵A到城堡 | 士兵走到建筑附近后原地跑步，永远不攻击 |
| 3个士兵横排A到城堡正前方 | 第1个到达但也不攻击，后面2个发呆 |
| 5个己方兵围攻1个敌方兵 | 前1-2个在打，后面3个站着不动 |
| 弓箭手攻击建筑 | 正常（射程200，远大于任何建筑的半径问题） |

### 1.2 根因分析

Bug的完整链路（以士兵攻击城堡为例）：

```
士兵 stats: attack_range=40, move_speed=120
城堡 stats: 3x3格子 = 192x192px, nav margin=20px

步骤1: command_attack(castle)
  → nav_agent.target_position = castle.global_position  (建筑中心)
  → NavMesh把建筑+20px标记为障碍区

步骤2: NavAgent导航到障碍区边缘
  → 实际停在与中心距离 ≈ 116px 的地方

步骤3: _attack_process() 距离判断
  → dist = distance_to(castle.global_position) = 116
  → effective_attack_range = 40 + (96 * 0.5) = 88
  → 116 > 88 → "还不够近，继续走"

步骤4: 但NavAgent已经导航结束了！
  → is_navigation_finished() = true
  → 不再产生新路径点 → velocity = 0
  → 但距离判断说还不够近 → 每帧重复"走过去"但走不动

结果: 原地跑步，永远不攻击
```

**两个独立Bug叠加：**
1. **距离判断Bug**：`distance_to(target.global_position)` 测量到建筑**中心**的距离，但建筑是不可穿越的，实际可达距离只到边缘
2. **导航目标Bug**：`nav_agent.target_position = target.global_position` 设为目标中心，但中心在障碍区内，NavAgent只能导航到障碍区边缘

`effective_attack_range` 的补偿（`+ building_radius * 0.5`）是个不精确的 hack，只补了约一半的偏差。

### 1.3 后排卡住问题

即使修好距离判断，多单位攻击同一目标时仍有问题：

```
          建筑
     ┌─────────┐
     │         │
友A ──→│         │
友B ──→│         │  ← A和B的射线几乎重合，指向同一点
友C ──→│         │
     └─────────┘
```

- A、B、C 从同一方向接近，NavAgent 导航到障碍区边缘的**同一个点**
- A先到，占据位置，开始攻击
- B和C被A挡住，NavAgent绕不开（A不在NavMesh障碍区里），速度接近0
- B和C永远走不到攻击距离，一直发呆

**本质**：当前系统没有"我被挡住了，换个方向走"的机制。

## 第三部分：技术调研

### 3.1 RVO/ORCA 算法族

| 算法 | 年份 | 核心思想 | 备注 |
|------|------|---------|------|
| RVO | 2008 | 双方各让一半 | 基础版，可能振荡 |
| ORCA | 2011 | 数学最优避让，保证无碰撞 | **Godot 4 内置** |
| HRVO | 2011 | 混合版，处理不确定性 | 学术用 |

**RVO核心**：假设对方也会配合避让，双方各让一半。不是一个人全让。
- 没有RVO：两人面对面走，都往左让，又撞上了
- 有RVO：我往左偏一半，你往右偏一半，擦肩而过

### 3.2 Godot 4 中 RVO 的启用方式

Godot已经内置了ORCA，但需要正确连接信号：

```gdscript
# 当前代码（错误 — RVO未生效）：
velocity = direction * move_speed
move_and_slide()

# 正确方式：
nav_agent.set_velocity(desired_velocity)  # 告诉RVO我想往哪走
# → RVO通过 velocity_computed 信号返回 safe_velocity
# 在信号回调中：
velocity = safe_velocity
move_and_slide()
```

**所需改动很小**：连接信号 + 改3处设velocity的地方。但效果也有限——只解决走路不撞人，不解决战斗站位。

### 3.3 RVO 的局限

实测问题（来自其他开发者报告）：
- 只用 RVO/分离力 → 单位互相推开，散太开，打不到敌人
- 凹地形里单位卡在墙角出不来
- 战斗时单位排成一条线而不是包围

**结论**：RVO 解决"走路不撞人"，不解决"打架时怎么站"。

### 3.4 Boids 三规则

| 行为 | 作用 | 范围 |
|------|------|------|
| 分离 (Separation) | 不挤在一起 | 近距离（2-3个单位半径） |
| 对齐 (Alignment) | 和周围人走同方向 | 中距离（5-10个单位半径） |
| 凝聚 (Cohesion) | 不散开，靠近群体中心 | 中距离 |

**单独用分离 = 越推越远。三个一起用 = 自然形成包围圈。**

### 3.5 商业RTS怎么做

| 游戏 | 全局路径 | 局部避让 | 战斗站位 |
|------|---------|---------|---------|
| 帝国时代4 | Flow Field | 转向行为 | 战斗时降避让+增凝聚 |
| 星际争霸2 | Flow Field | 转向行为 | 允许轻微重叠（攻击优先） |
| 魔兽争霸3 | A* | 槽位分配 | 每个单位分配目标周围固定位置 |

**共同模式**：走路和战斗用**不同参数**——走路避让优先，战斗凝聚优先。

### 3.6 理想技术栈

```
┌─────────────────────────────────┐
│        游戏逻辑层                │  unit.gd: 攻击判断、卡住绕行
├─────────────────────────────────┤
│       Boids 行为层               │  新增: 分离/对齐/凝聚
│  战斗: 高凝聚 + 低分离            │
│  走路: 正常分离 + 正常凝聚        │
```

#### A.2 新增三个辅助函数（`_perform_attack()` 之前）

**函数1：`_get_building_nav_target(target, angle_offset=0.0)`**
- 建筑目标：从建筑中心向单位方向（+可选角度偏移）射线，求与矩形边缘的交点
- 不同单位从不同角度接近 → 不同射线 → 不同导航目标 → 自然散开
- `angle_offset` 用于卡住时偏转到另一侧
- 非建筑目标（单位）：直接返回 `target.global_position`

```gdscript
func _get_building_nav_target(target, angle_offset: float = 0.0) -> Vector2:
    if not target.has_method("get_rect"):
        return target.global_position
    var brect: Rect2 = target.get_rect()
    var center := brect.get_center()
    var to_unit := global_position - center
    if to_unit.length_squared() < 1.0:
        return center + Vector2.RIGHT * brect.size.x / 2.0
    var angle := to_unit.angle() + angle_offset
    var half := brect.size / 2.0
    var ca := cos(angle)
    var sa := sin(angle)
    var dx := half.x / abs(ca) if abs(ca) > 0.001 else 999999.0
    var dy := half.y / abs(sa) if abs(sa) > 0.001 else 999999.0
    var d := min(dx, dy)
    return center + Vector2(ca, sa) * d
```

**函数2：`_get_dist_to_target(target)`**
- 建筑目标：`global_position.clamp(brect.position, brect.end)` 找最近边缘点，返回到该点的距离
- 非建筑目标：返回到中心的距离
- 这是精确距离判断，替代原来的 hack

```gdscript
func _get_dist_to_target(target) -> float:
    if not target.has_method("get_rect"):
        return global_position.distance_to(target.global_position)
    var brect: Rect2 = target.get_rect()
    var nearest := global_position.clamp(brect.position, brect.end)
    return global_position.distance_to(nearest)
```

**函数3：`_get_detour_target(target)`**
- 建筑目标：偏转射线角度60-120度（随机方向），绕到建筑另一侧
- 非建筑目标：偏移到目标侧面（垂直方向50px），从侧面包抄
- 纯自我检测，无需单位间通信

```gdscript
func _get_detour_target(target) -> Vector2:
    if target.has_method("get_rect"):
        var offset := deg_to_rad(randf_range(60.0, 120.0))
        if randi() % 2 == 0:
            offset = -offset
        return _get_building_nav_target(target, offset)
    else:
        var to_target := target.global_position - global_position
        var perp := Vector2(-to_target.y, to_target.x).normalized()
        var side := perp * (50.0 if randi() % 2 == 0 else -50.0)
        return target.global_position + side
```

#### A.3 重写 `_attack_process()` 核心逻辑（第358-379行）

```
原逻辑:
  dist = 到中心距离
  if dist > effective_attack_range:
    nav_target = 中心
    走过去

新逻辑:
  dist = 到最近边缘距离 (精确)
  if dist > attack_range:
    卡住检测 (velocity接近0 超过1.5秒)
    nav_target = 射线到边缘点
    if 卡住了: nav_target = 绕行目标 (偏转角度/侧面)
    nav_agent.target_position = nav_target
    走过去
  else:
    攻击
```

完整替换第358-379行：

```gdscript
    var dist := _get_dist_to_target(attack_target)
    if dist > attack_range:
        if hold_position_mode:
            attack_target = null
            _is_attacking = false
            state = UnitState.HOLD_POSITION
            return
        # 卡住检测：速度接近零 = 被友军挡住了
        if velocity.length_squared() < 4.0:
            _stuck_timer += delta
        else:
            _stuck_timer = 0.0
        var nav_target := _get_building_nav_target(attack_target)
        # 卡住超过1.5秒 → 自动绕行（建筑和单位通用）
        if _stuck_timer > 1.5:
            nav_target = _get_detour_target(attack_target)
            _stuck_timer = 0.0
        nav_agent.target_position = nav_target
        if not nav_agent.is_navigation_finished():
            var next_pos := nav_agent.get_next_path_position()
            velocity = global_position.direction_to(next_pos) * move_speed
    else:
        _stuck_timer = 0.0
        if attack_timer <= 0.0:
            _perform_attack()
            attack_timer = attack_cooldown
```

### 阶段C：Boids行为层（未来）

**目标**：战斗时单位自然形成包围圈，走路时保持队形。

**核心思路**：在 RVO 之前，给 desired_velocity 加上 Boids 三力的混合：

```gdscript
func _calculate_boids_force() -> Vector2:
    var separation := Vector2.ZERO  # 推开太近的友军
    var cohesion := Vector2.ZERO    # 靠近群体中心
    var alignment := Vector2.ZERO   # 和周围人走同方向
    var neighbor_count := 0

    for other in get_nearby_allies():
        var dist = global_position.distance_to(other.global_position)
        if dist < separation_radius:
            separation += (global_position - other.global_position).normalized() / max(dist, 1.0)
        if dist < cohesion_radius:
            cohesion += other.global_position
            alignment += other.velocity
            neighbor_count += 1

    if neighbor_count > 0:
        cohesion = (cohesion / neighbor_count - global_position).normalized()
        alignment = (alignment / neighbor_count).normalized()

    # 战斗模式和走路模式用不同权重
    var sep_w := 0.8 if in_combat else 1.5
    var coh_w := 1.5 if in_combat else 0.5
    var ali_w := 0.5 if in_combat else 0.3

    return separation * sep_w + cohesion * coh_w + alignment * ali_w
```

**参数建议**：

| 参数 | 走路 | 战斗 |
|------|------|------|
| 分离半径 | 30px | 20px |
| 分离权重 | 1.5 | 0.8 |
| 凝聚半径 | 100px | 80px |
| 凝聚权重 | 0.5 | 1.5 |
| 对齐权重 | 0.3 | 0.5 |

**性能考虑**：每个单位每帧扫描所有友军是 O(n^2)。可以用 `get_tree().get_nodes_in_group()` + 距离过滤，50个单位以内不需要空间哈希。

## 第五部分：验证方法

### 阶段A验证（本次）

**建筑攻击：**
1. 1个士兵攻击城堡 → 走到边缘，正常攻击
2. 3个士兵横排A到城堡正前方 → 第1个到达攻击，后2个1.5秒后自动绕到两侧
3. 5+士兵攻击城堡 → 逐渐散开，包围城堡各侧

**单位对战：**
4. 5个己方兵攻击1个敌方兵 → 前排攻击，后排1.5秒后从侧面绕过
5. 混编（近战+远程）攻击 → 远程在后面打，近战散开包围

**回归测试：**
6. 弓箭手攻击建筑 → 射程200，不受影响
7. 1v1 对战 → 不走卡住分支，正常攻击
8. 守卫模式 → 用 clamp 距离判断，正常
9. 右键攻击 → 用射线导航目标，正常
10. 空格键回基地 → 不涉及战斗逻辑，不受影响
