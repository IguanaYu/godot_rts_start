## 1. 六条核心设计经验

每条经验配有：原则、为什么、正例、反例。

---

### 1.1 一眼可识别（Visual Identity）

**原则**：玩家在混乱战场上必须能立刻认出"这是大战术释放者"。

**为什么**：玩家决策链是 **看见 → 识别 → 评估 → 决策**。任何环节缺失都会导致挫败感。"识别"失败尤其致命——玩家看见但不知道这是优先目标。

**正例**：
- Captain Winters（Payday 2）：蓝色警灯 + 专属造型 + 出场音效
- Blightstormer（Vermintide 2）：臃肿裸体 + 金属面具 + 紫色施法光效
- Helldivers 2 信号弹：红色烟雾 + 特殊音效

**反例**：
- Helldivers 2 早期版本：太多小怪都能 call reinforcements，玩家挫败感爆棚（"我打哪个？"）

**应用到 RTS**：
- 大战术释放者必须有独特造型（颜色、体型、装饰）
- 出场时小地图标记 + 警告音效
- 施法时独特的光效（不要复用其他单位的施法特效）

---

### 1.2 击杀奖励立竿见影

**原则**：击杀他应该立刻产生玩家能感知的效果（buff 消失、压制解除、召唤停止）。

**为什么**：玩家集中火力击杀高 HP 单位是"投资"，需要立刻看到回报。如果效果延迟或不可感知，玩家会觉得"打不打都一样"。

**正例**：
- Captain Winters：击杀后压制 wave 立刻结束，buff 立刻消失——爽快感拉满
- Fallen Shaman：击杀后小怪不再复活，玩家清场速度立竿见影

**反例**：
- 假设某设计：击杀后 buff 缓慢衰减（5s 内消失）——玩家感知不到效果，下次就不打了
- 假设某设计：击杀后召唤物不消失，只是停止召唤——玩家觉得"打他没用，还得清场"

**应用到 RTS**：
- 击杀时全图视觉/音效反馈（屏幕震屏、专属音效、光柱）
- buff/aura 立刻解除（不要 fade）
- 召唤物联动死亡时全数爆炸（强反馈）

---

### 1.3 不攻击的单位必须有保护机制

**原则**：自身不输出的大战术释放者，必须有保护机制，否则就是送。

**为什么**：高 HP + 不攻击 = 玩家可以慢慢打。如果不保护，玩家用任何远程单位都能风筝死他，机制失去意义。

**正例**：
- Captain Winters：Phalanx 盾兵护卫
- Glyphid Menace（DRG）：远程骚扰 + 召唤护卫
- Blightstormer（Vermintide 2）：远距离施法 + 隐蔽站位

**保护机制的类型**：
1. **护卫**：召唤专属护卫单位（如 Captain Winters 的 Phalanx）
2. **aura**：自身周围有伤害 aura，玩家近战单位受损
3. **位移**：移动 AI（远离玩家部队）
4. **远程骚扰**：自身有弱攻击（阻止玩家无伤贴近）
5. **地形**：出现在难以接近的位置（敌后、高台）

**应用到 RTS**：
- 默认要有至少 1 种保护机制
- 进阶战术可以有 2 种以上（如召唤护卫 + aura）

---

### 1.4 引导阶段要有视觉差异

**原则**：玩家必须能区分"还能阻止"vs"已经晚了"。

**为什么**：玩家需要决策"现在打还来得及吗"。视觉差异让玩家感知阶段，做出合理决策。

**正例**：
- Blightstormer：读条时蓝色光，完成时紫色风暴成形
- Captain Winters：出现时蓝灯闪烁（压制开始），击杀时灯灭（压制结束）

**视觉差异的阶段**：
- **出场**：明确"他来了"（标记 + 音效）
- **引导中**：施法光柱/光环，可见的进度（如光柱逐渐变粗）
- **即将完成**：最后 5s 强化 telegraph（闪烁、颜色变化）
- **完成**：明确的效果爆发

**应用到 RTS**：
- 引导开始：光柱 + 音效
- 引导进度：光柱颜色/亮度/大小变化（玩家能粗略判断"还有多久"）
- 即将完成（最后 5s）：强化闪烁（强迫玩家做最后一击决策）
- 完成：明确的视觉爆发（不仅是"召唤物出现了"）

---

### 1.5 不要防秒杀

**原则**：不要加 damage cap、阶段无敌、最小存在时间等"防秒杀"机制。

**为什么**：这些机制让"专注火力"不被奖励。玩家堆 DPS 想秒杀他，结果被 cap 挡住，会觉得挫败。在 RTS 里尤其严重——玩家用大量部队集中火力是合理策略，不应该被惩罚。

**反例**：
- Captain Winters 的 damage cap：被玩家骂得很惨
- WoW 部分 boss 的"阶段切换无敌"：玩家觉得"为啥不让我直接秒"

**正确做法**：
- 用 HP 数值平衡，而不是机制 cap（要 30s 才能秒就给足够 HP）
- 用保护机制（护卫、aura）增加击杀成本，而不是禁止秒杀
- 玩家如果能秒杀，说明 DPS 足够，是好事——奖励玩家

**例外**：
- 多阶段 boss 可以有"阶段切换无敌"（剧情节奏需要），但 RTS 里少用
- 自身护盾型（J）可以有"必须先打发生器"机制，但这是设计的一部分，不是临时防秒杀

---

### 1.6 决策深度 > 单纯击杀

**原则**：好的战术不止"打/不打"二选一，而是给玩家多个有效选项。

**为什么**：决策深度决定机制的可玩性。单纯击杀（如"他出现→玩家打→结束"）只有 1 次决策；多决策点（"打哪个、用什么打、什么时候打"）能让同一战术在不同局势下有不同最优解。

**正例**：
- **Shared Life Meter**（联动死亡）：玩家可以选择"打主目标"vs"分散打精英"
- **aura 类**（D/E/F）：玩家可以选择"撤出范围"vs"硬冲"
- **召唤类**（A/B）：玩家可以选择"打释放者"vs"清召唤物"

**反例**：
- 单纯"出现 → 30s 内必须杀"：决策只有打/不打，深度低
- 全图 buff：玩家无路可退，决策只有打/不打

**应用到 RTS**：
- 优先做"决策深度高"的战术（如 C - 联动死亡）
- 设计aura 类时确保有"撤出"选项
- 避免全图效应（让玩家有位置选择）

## 2. 五个常见陷阱（反模式）

### 2.1 ❌ 全图 aura / 全图 buff
- **问题**：玩家无路可退，决策只有打/不打
- **修正**：aura 范围有限，玩家可撤出

### 2.2 ❌ 无 telegraph
- **问题**：玩家不知道发生了什么，被动忍受
- **修正**：出场/引导/完成每阶段都要有明确视觉/音效

### 2.3 ❌ 同时超过 2 个
- **问题**：玩家无法决策先打哪个
- **修正**：默认 1 个，最多 2 个（且战术类型不同）

### 2.4 ❌ 永久后果
- **问题**：30s 后召唤的单位永久存活，地图爆满
- **修正**：召唤物有寿命，或他死后消失，或正常清场（不爆图）

### 2.5 ❌ damage cap / 防秒杀
- **问题**：见 §1.5
- **修正**：用 HP 数值平衡

## 3. 设计 Checklist（推出新战术时检查）

### 视觉与识别
- [ ] 独特造型（颜色/体型/装饰与其他单位明显不同）
- [ ] 出场 telegraph（小地图标记 + 警告音效）
- [ ] 引导中可见进度（光柱颜色/亮度变化）
- [ ] 即将完成（最后 5s）强化 telegraph
- [ ] 完成时明确视觉爆发

### 决策与可玩性
- [ ] 玩家有 ≥ 2 个有效选项（不是只有打/不打）
- [ ] 失败惩罚可控（不是 gg，是"挑战加剧"）
- [ ] 击杀奖励立竿见影（buff/aura 立刻解除）
- [ ] 同时存在数量 ≤ 2

### 保护机制
- [ ] 至少 1 种保护机制（护卫/aura/位移/远程骚扰/地形）
- [ ] HP 足够高（玩家需要派专门部队）
- [ ] 没有防秒杀机制（damage cap 等）

### 后果与清理
- [ ] 战术后果是可逆的（buff/aura 解除后恢复）
- [ ] 召唤物有清理路径（不会永久堆积）
- [ ] aura 范围有限（不是全图）

## 4. 决策维度参考

设计新战术时，从下列维度思考：

### 4.1 位置维度
- 他出现在哪里？敌方阵内 / 战场中央 / 接近玩家基地
- 玩家需要走多远？机会成本多大

### 4.2 时间维度
- 引导时长（30s 是基准，可调）
- 即将完成的 telegraph 时长（建议 5s）
- 后果持续时间（立即消失 / 持续 X 秒 / 永久）

### 4.3 空间维度
- aura 范围（有限 vs 全图）
- 移动模式（静止 / 缓慢移动 / 主动远离玩家）
- 召唤物分布（自身周围 / 随机位置 / 玩家基地附近）

### 4.4 数值维度
- HP（参考 SLOW 方案，明显肉于普通单位）
- 移动速度（建议慢于普通单位，给玩家追击窗口）
- buff 强度（aura 内 vs aura 外的差异要明显）

### 4.5 反馈维度
- 出场音效（专属）
- 击杀音效（专属 + 震屏）
- 后果发生音效（buff 起效 / 召唤物出现）

## 5. 与现有项目原则的对齐

参考 [feedback_mutator_design_preference](../../C:/Users/86132/.claude/projects/e--godot-rts-godot-rts-start/memory/feedback_mutator_design_preference.md)：

> 必须给玩家主动决策点（派部队、选目标、选位置），避开被动忍受型（视野限制、debuff、纯 buff 敌方）

**大战术释放通过主动决策测试**：
- 派部队 ✓（高 HP 需要专门派遣）
- 选目标 ✓（明确优先目标）
- 选位置 ✓（移动 + aura 制造位置决策）

**需要注意的边界**：
- D（鼓舞士气）和 E（持续回血）如果设计成全图 buff，会变成"被动忍受"——必须改成 aura 形式
- 多个同时存在会让玩家"无法主动决策"——必须限制数量
