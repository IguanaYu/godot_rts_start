# 星际争霸2 vs 魔兽争霸3 — RTS 数值设计与节奏对比分析

> 基于 sc2_standard.md / sc2_coop.md / wc3.md 完整数据 + 分析脚本计算结果
> 分析日期: 2026-07-01

---

## 0. 核心结论（太长不看版）

| 维度 | SC2 | WC3 | 对本项目的意义 |
|------|-----|-----|--------------|
| **战斗节奏** | 快（TTK 1-15s） | 慢（TTK 10-150s） | 决定玩家注意力分配 |
| **护甲公式** | 减法（线性收益） | 百分比（递减收益） | 减法简单但高甲克制极端 |
| **经济节奏** | 快速扩张，多矿运营 | 单矿为主， upkeep 税 | 开矿难度决定游戏时长 |
| **科技深度** | 3 Tier，快速攀升 | 3 Tier，每级成本高 | Tier 数量影响中期博弈 |
| **英雄** | 无英雄（合作模式有） | 英雄为核心 | 英雄增加战术深度和成长感 |
| **人口上限** | 200 硬上限 | upkeep 软上限（税） | 软上限鼓励精兵，硬上限鼓励爆兵 |
| **克制深度** | 轻甲/重甲+加成伤害 | 6 攻击类型×6 护甲类型 | 克制关系越多战术越丰富 |

## 1. 伤害公式对比 — 决定战斗节奏的根本

### 1.1 公式本身

```
SC2（减法）：  实际伤害 = max(攻击力 - 护甲, 0.5)
WC3（百分比）：实际伤害 = 攻击力 × 类型倍率 × (1 - armor×0.06/(1+0.06×armor))
```

### 1.2 护甲收益曲线对比

| 护甲值 | SC2 减伤(对6伤害) | SC2 减伤(对20伤害) | WC3 百分比减伤 |
|--------|------------------|------------------|--------------|
| 0 | 0% | 0% | 0% |
| 1 | 16.7% | 5% | 5.66% |
| 3 | 50% | 15% | 15.25% |
| 5 | 83.3% | 25% | 23.08% |
| 10 | 无限刮痧 | 50% | 37.50% |

**关键差异：**
- SC2 的减法公式对**低伤高频单位**极度不友好（Marine 打 3 甲伤害减半，打 5 甲直接刮痧）
- WC3 的百分比公式护甲收益递减，不存在"完全打不动"的情况
- 结果：SC2 中护甲升级的边际收益**递增**（越到后期越值钱），WC3 中护甲收益**递减**

### 1.3 攻击类型克制系统复杂度

```
SC2：2 种护甲类型（轻甲/重甲）+ bonus_vs 加成
WC3：6 种攻击类型 × 6 种护甲类型 = 36 种组合
```

**SC2 的克制更简洁：**
- 每个单位只标记轻甲或重甲
- 克制通过 `bonus_vs` 实现（如 Immortal +30 对重甲）
- 玩家不需要记忆倍率表，看到单位类型就知道克制关系

**WC3 的克制更深：**
- 36 种组合意味着丰富的战术选择
- Piercing vs Light = 200%（弓箭手天克轻甲）
- Magic vs Heavy = 200%（法师克重甲）
- Siege vs Fortified = 150%（攻城克建筑）
- **战术深度 vs 学习成本**的经典权衡

### 1.4 对战斗节奏的影响

```
SC2 战斗：接触 → 秒杀（1-5s） → 胜负已定
WC3 战斗：接触 → 消耗（10-30s） → 技能释放 → 撤退或围杀
```

SC2 的战斗更"脆"，谁先集火谁赢；WC3 的战斗更"黏"，给了英雄技能和微操更多时间窗口。

## 2. TTK（击杀时间）对比 — 最直观的节奏指标

### 2.1 同级别单位互打

| 对比场景 | SC2 TTK | WC3 TTK | 差异倍数 |
|---------|---------|---------|---------|
| T1 近战互打 | Marine vs Marine **4.6s** | Footman vs Footman **~50s** | ~11x |
| T1 远程互打 | Marine vs Marine **4.6s** | Rifleman vs Rifleman **~41s** | ~9x |
| T1 肉盾互打 | Zealot vs Zealot **11.3s** | Grunt vs Grunt **57.9s** | ~5x |
| T3 重甲互打 | Immortal vs Immortal **6.5s** | Tauren vs Tauren **65.4s** | ~10x |
| 高DPS打脆皮 | Hydra vs Zergling **1.4s** | Frost Wyrm vs Ghoul **6.4s** | ~4.5x |

**核心发现：SC2 的击杀速度是 WC3 的 5-11 倍。**

这意味着：
- SC2 一局战斗在 **5-15 秒**内结束
- WC3 一局战斗持续 **30-120 秒**
- SC2 玩家需要在更短时间内做出正确决策
- WC3 玩家有更多时间操作英雄技能、调整阵型

### 2.2 TTK 分布对比

```
SC2 TTK 分布：1-5s（快速击杀）占 40%，5-15s（中等）占 40%，15s+（慢速）占 20%
WC3 TTK 分布：5-15s（快速）占 10%，15-60s（中等）占 60%，60s+（慢速）占 30%
```

### 2.3 这个差异是怎么产生的？

| 因素 | SC2 | WC3 |
|------|-----|-----|
| HP 数值 | 35-550 | 245-1300 |
| DPS 数值 | 6-40 | 4-26 |
| HP/DPS 比（T1） | ~5-15 | ~30-55 |
| HP/DPS 比（T3） | ~15-55 | ~30-60 |

**SC2 的 HP/DPS 比更低**，意味着单位更"脆"，战斗节奏更快。这是暴雪在 SC2 设计时有意的选择——让比赛更刺激、观赏性更强。

## 3. 经济节奏对比

### 3.1 基础采集效率

| 指标 | SC2 | WC3 |
|------|-----|-----|
| 初始工人数 | 12 | 5 |
| 单矿满负荷收入 | ~880 资源/分 | ~600 金 + ~150 木/分 |
| 开矿成本 | 400 矿（CC）+ ~300 农民 | 385 金（分矿）+ 防守成本 |
| 单矿采干时间 | ~8-10 分钟 | 无限（金矿不枯竭） |
| 特殊经济机制 | MULE/Chrono/幼虫 | 民兵/嗜金/腐地/月井 |

### 3.2 出兵节奏对比

| 指标 | SC2 Marine | WC3 Footman |
|------|-----------|------------|
| 造价 | 50 矿 | 135 金 |
| 建造时间 | 25s（Reactor: 12s） | 20s |
| 单矿同时出兵数 | 1-2（Reactor） | 1 |
| 首兵时间 | ~50s | ~50s |
| 60s 内可出兵数 | 2-5 | 2-3 |

### 3.3 经济规模效应

```
SC2：多矿 → 多兵营 → 指数级兵力增长
      关键在"开矿速度"和"产能建筑数量"

WC3：单矿为主，开矿成本高，upkeep 税限制
      关键在"英雄等级"和"兵力质量"
```

**WC3 的 Upkeep 系统：**
- 人口 0-40：无税（100% 收入）
- 人口 41-70：中税（70% 收入）
- 人口 71+：高税（40% 收入）

这个系统**主动限制**了部队规模，鼓励玩家在精兵和扩张之间做选择。SC2 没有 upkeep，200 人口硬上限更直接。

### 3.4 经济对节奏的影响

| | SC2 | WC3 |
|------|-----|-----|
| 前期（0-4min） | 快速扩张，2-3 基地 | 单矿，英雄练级 |
| 中期（4-10min） | 爆兵，Timing 进攻 | 二本科技，英雄 3-5 级 |
| 后期（10min+） | 多矿运营，200 人口大战 | 三本科技，英雄 6+ 级 |
| 游戏时长 | 10-25 分钟 | 15-40 分钟 |

## 4. 科技/兵种节奏对比

### 4.1 Tier 系统

| | SC2 | WC3 |
|------|-----|-----|
| Tier 数 | 3（T1/T2/T3） | 3（T1/T2/T3） |
| T1→T2 成本 | 150-200 矿+气 | 230-265 金+160-180 木 |
| T2→T3 成本 | 200-300 矿+气 | 285-385 金+210-220 木 |
| T1→T3 最短时间 | ~5 分钟（速科技） | ~8 分钟（速科技） |

### 4.2 各 Tier 兵种 HP 对比

```
SC2 HP 梯度：
  T1: 35-100（HP）     → 倍率 1x
  T2: 90-180           → 倍率 2-4x
  T3: 200-550          → 倍率 4-10x

WC3 HP 梯度：
  T1: 245-700（HP）    → 倍率 1x
  T2: 265-1300         → 倍率 1-2.5x
  T3: 400-1200         → 倍率 1.5-2.5x
```

**关键差异：SC2 的 Tier 跨度更大，T3 兵种相对 T1 的强度提升更显著。**
- SC2：T3 兵种 HP 是 T1 的 4-10 倍 → 科技领先 = 巨大优势
- WC3：T3 兵种 HP 是 T1 的 1.5-2.5 倍 → 科技领先 ≠ 碾压，英雄等级更重要

### 4.3 兵种数量与质量权衡

```
SC2：T1 兵种（Zergling DPS/Pop = 20）＞ T3 兵种（Thor DPS/Pop = 3.9）
     → 低Tier 兵种的人口效率更高，爆兵有优势
     → 高Tier 兵种的优势在生存力和隐性价值（AOE/射程）

WC3：T1 兵种（Grunt HP×DPS/Cost = 44.8） vs T3（Tauren = 81.3）
     → T3 兵种的性价比更高
     → 但受 upkeep 限制，不能无脑爆高级兵
```

## 5. 人口/供给系统对比

| | SC2 | WC3 |
|------|-----|-----|
| 上限 | 200 硬上限 | 100 软上限（upkeep 税） |
| 初始人口 | 12/200 | 5/100 |
| T1 兵人口 | 1-2 | 2-3 |
| T3 兵人口 | 4-6 | 4-7 |
| 满人口兵力 | ~50-80 单位 | ~20-40 单位 |
| 经济惩罚 | 无 | 41+人口 70%收入，71+人口 40%收入 |

**影响：**
- SC2 的 200 人口大战 ≈ 80 个单位同时作战 → **大规模会战**
- WC3 的 100 人口（实际保持 40-70）≈ 20-30 个单位 → **小规模精英战**
- WC3 的 upkeep 税让玩家必须考虑"人口质量"，不能无脑爆兵

## 6. 英雄系统的影响

### 6.1 SC2（无英雄）vs WC3（有英雄）

| 维度 | SC2 | WC3 |
|------|-----|-----|
| 战术核心 | 兵种组合 + 集火 | 英雄技能 + 光环 |
| 成长性 | 无（兵种固定数值） | 英雄升级，技能变强 |
| 翻盘能力 | 靠运营/多线/操作 | 靠英雄等级优势和终极技能 |
| 容错率 | 低（兵死完就输） | 中（英雄不死就有希望） |
| 操作焦点 | 多线运营 + 集火 | 英雄走位 + 技能释放时机 |

### 6.2 英雄对节奏的影响

```
WC3 有英雄：
  - 前期：英雄练级（0-5min）→ 和平发育期
  - 中期：英雄 3 级 + 二本科技 → 第一次进攻
  - 后期：英雄 6 级 + 终极技能 → 决战
  → 英雄的成长曲线 = 游戏节奏的天然分界

SC2 无英雄：
  - 前期：侦察 + 扩张（0-3min）
  - 中期：第一次 Timing 进攻（4-7min）
  - 后期：多矿运营 + 科技成型（8min+）
  → 节奏由经济/科技决定，更灵活多变
```

## 7. 综合节奏曲线对比

### 7.1 战斗力-时间曲线

```
战斗力
  ^
  |                    SC2（指数增长）
  |                  /
  |                /
  |              /    WC3（线性+阶梯增长）
  |            /    /
  |          /  /  /
  |        / / / /
  |      / / / /
  |    / / //
  |  / / //
  | / // 
  +--------------------------------→ 时间
  0    5    10    15    20    25(min)
```

**SC2 的战斗力曲线更陡峭：**
- 多矿 → 多产能 → 指数级兵力增长
- 科技领先 = 兵种质量碾压
- 一波动辄定胜负

**WC3 的战斗力曲线更平缓：**
- 英雄等级 + 科技 → 阶梯式增长
- 兵力质量受 upkeep 限制
- 翻盘可能性更大

## 8. 对本项目的启示

### 8.1 节奏定位选择

我们的 RTS 项目需要决定**战斗节奏的定位**：

| 节奏类型 | TTK 范围 | 适合场景 | 代表游戏 |
|---------|---------|---------|---------|
| 极快节奏 | 1-3s | 电竞观赏性强，容错率低 | SC2 |
| 中等节奏 | 3-10s | 平衡微操和战略 | 大多数 RTS |
| 慢节奏 | 10-30s+ | 强调阵型和技能，容错率高 | WC3 |

**建议：** 采用 **中等节奏（TTK 3-8s）**，介于 SC2 和 WC3 之间。这样既有操作的紧张感，又给技能释放留出时间窗口。

### 8.2 伤害公式选择

| 方案 | 优点 | 缺点 |
|------|------|------|
| SC2 减法公式 | 简单直观 | 高甲克制极端 |
| WC3 百分比公式 | 平衡性好 | 计算复杂 |
| **混合公式** | 兼顾两者优点 | 需要调试 |

**建议：** 采用**简化版减法公式** + **护甲上限控制（0-5）**。这样既保持计算简单，又避免 Marine 打不动 Ultralisk 的极端情况。同时引入 WC3 式的**攻击类型克制倍率**（3-4 种护甲类型 × 3-4 种攻击类型），增加战术深度。

### 8.3 经济节奏设计

```
开局速度：SC2 式快开局（6-8 初始工人）
扩张成本：WC3 式高成本开矿（鼓励战术博弈而非纯运营）
人口系统：SC2 式硬上限（简单明了）+ WC3 式质量权衡（高人口单位占更多人口）
```

### 8.4 英雄系统参考

如果项目有英雄系统：
- **英雄成长曲线**：参考 WC3 的 3+1 技能结构，让英雄升级成为节奏分界点
- **英雄强度控制**：英雄 DPS 控制在普通 T3 单位的 2-3 倍（而非 SC2 合作模式的 5-10 倍）
- **英雄重生**：可以采用 CD 较长（60-120s）的重生机制，平衡容错率和惩罚

### 8.5 具体数值建议

| 参数 | 建议值 | 依据 |
|------|-------|------|
| T1 兵种 HP | 50-100 | SC2 的 Marine(45)-Zealot(150) 之间 |
| T2 兵种 HP | 120-250 | 介于两者之间 |
| T3 兵种 HP | 300-600 | 避免 WC3 的 1300HP 极端肉盾 |
| T1 TTK | 3-6s | 比 SC2 稍慢，给操作空间 |
| T3 TTK | 8-15s | 比 WC3 快，保持紧张感 |
| 护甲范围 | 0-5 | 避免无限刮痧 |
| 克制倍率 | 1.0-1.5x | 比 WC3 的 2.0x 温和 |
| 人口上限 | 100-150 | 介于两者之间 |
| 开局工人 | 6-8 | 比 SC2 少，比 WC3 多 |
| 首兵时间 | ~45s | 与两者一致 |
| 科技 Tier | 3 级 | 经典结构 |
