# RTS 血条显示调研报告

> 调研日期：2026-06-10

## 一、主流 RTS 血条颜色方案

### 1. 星际争霸 2 (StarCraft II)
- **颜色阈值**：离散分档，非渐变
  - 绿色：86%~100% (6/7 以上)
  - 黄绿色：57%~86% (4/7~6/7)
  - 橙色：29%~57% (2/7~4/7)
  - 红色：0%~29% (2/7 以下)
- **团队色模式**：可选设置，用阵营色替代血量色（己方绿、敌方红）
- **黑边框**：约 1px 纯黑边框
- **附加信息**：护盾条（蓝色）与血条上下排列
- **可见性**：可设为始终显示/仅选中显示
- **无障碍**：提供色盲滤镜和音效提示

### 2. 魔兽争霸 3 (Warcraft III)
- **颜色阈值**：经典三色方案
  - 绿色：>50%
  - 黄色：25%~50%
  - 红色：<25%
- **可见性**：按 Alt 键显示所有血条，平时只显示选中单位
- **附加信息**：蓝条显示魔法值，与血条上下排列
- **快捷键**：可分别显示友方/敌方血条

### 3. 帝国时代系列 (Age of Empires)
- **颜色方案**：血条本身不变色，始终绿色
- **数字显示**：选中后在面板显示具体 HP 数字（如 80/100）
- **风格**：极简主义，减少视觉干扰

### 4. 命令与征服系列 (Command & Conquer)
- **颜色方案**：绿→黄→红三色
- **特点**：血条较短较粗，适合快速扫视

### 5. 最高指挥官 (Supreme Commander)
- **缩放适应**：近景显示完整血条，远景缩略为图标符号
- **设计思路**：大量单位时避免视觉混乱

### 6. OpenRA (开源 RTS 引擎)
- **支持圆形血条**：除横条外还提供圆形血环显示
- **可配置参数**：CircleColor, CircleWidth, 边框颜色/透明度等

## 二、实现技术方案

### 方案 A：StyleBoxFlat 动态换色
通过代码在血量变化时动态修改 ProgressBar 的 fill StyleBoxFlat 的 bg_color。

```
优点：
- 实现最简单，Godot 原生支持
- 项目中已有先例（护盾技能、建造进度条都用过 StyleBoxFlat）
- 无需额外资源（shader、纹理）

缺点：
- 颜色切换是瞬变的，没有渐变过渡
- 每次血量变化都要 new StyleBoxFlat 或修改已有实例
- 频繁创建/修改可能产生轻微 GC 压力
```

### 方案 B：分层渲染（底层红 + 上层绿填充）
底层放一个红色背景条，上层放绿色填充条。血量减少时绿色缩短，露出红色底色。

```
优点：
- 天然产生渐变效果（绿→红之间有视觉过渡区域）
- 不需要任何颜色计算逻辑
- 常见于大型 RTS 游戏

缺点：
- 需要两个 ProgressBar 节点叠加
- 阈值控制不如方案 A 精确（黄→红过渡区域靠视觉混合）
- 场景节点结构稍复杂
```

### 方案 C：Shader 颜色渐变
用自定义 shader 根据 UV 的 x 坐标和血量百分比做颜色映射。

```
优点：
- 效果最好，可以实现平滑渐变
- 性能最优（GPU 端计算，不产生 GC）
- 可以实现各种高级效果（闪烁、脉冲等）

缺点：
- 需要编写 shader 代码
- 调试相对复杂
- 对不熟悉 shader 的开发者维护成本高
```

---

## 三、通用设计建议

1. **黑边框**：几乎所有 RTS 都用 1px 黑边提高可读性
2. **阈值选择**：War3 风格（50%/25%）最直观易懂
3. **可见性**：建议选中/悬停时显示，避免满屏血条
4. **尺寸**：当前 36×8 对单位合适，不需要改动
5. **平滑过渡**：血条数值变化可以加 lerp 动画，提升视觉质感
