# 单位模板扩展设计：15 种新兵种

## Context（背景与目标）

当前游戏只有 4 种基础单位（战士 SOLDIER / 弓箭手 ARCHER / 枪兵 LANCER / 僧侣 MONK）+ 3 个强化变体（hero/boss/elite_archer）。想做出更多种类的单位，但因美术资源有限，无法为新单位做全新模型。

**方案**：复用现有 4 套精灵素材，通过「调色（tint）+ 缩放 + 攻击模式差异化」派生出 15 个新兵种，覆盖 RTS 经典设计的 12 个原型轴（参考 AoE2 / 魔兽争霸 2·3 / 要塞 / 星际争霸）。所有兵种**对称双方共用**（玩家和敌方都能用）。

**设计原则**：尽量复用已有系统（SplashEffect / SlowEffect / damage_reduction / ProjectileData / buff），把新代码量压到最小。按实现成本分 3 个 Tier，先做零成本的 Tier 1。

## 三、关键代码改动（前置 Enabler）

为了让变体可调色、可远程化、可克制，需要 4 个**小而通用的代码改动**。这些是 Tier 1/2 的前提。

### Enabler 1：单位调色（tint）— 让所有变体视觉可区分
**文件**：`shaders/unit_effects.gdshader` + `scripts/units/unit.gd`

- shader 新增 3 个 uniform（在现有 `slow_tint` 旁）：
  ```glsl
  uniform bool tint_enabled = false;
  uniform vec4 tint_color : source_color = vec4(1.0);
  uniform float tint_amount : hint_range(0.0, 1.0) = 0.5;
  ```
  fragment 里在 `slow` 分支之后加：`if (tint_enabled) { col.rgb = mix(col.rgb, tint_color.rgb, tint_amount); }`
- unit.gd 在 `@export_group("Variant Modifiers")`（第 54-58 行）下新增：
  ```gdscript
  @export var variant_tint: Color = Color.WHITE
  @export var variant_tint_amount: float = 0.5
  ```
- `_init_outline_materials()`（第 166-177 行）末尾设置 shader 参数：`tint_enabled = (variant_tint != Color.WHITE)`、`tint_color`、`tint_amount`。

### Enabler 2：远程攻击解耦 — 法师/诅咒师也能扔东西
**文件**：`scripts/stats/unit_stats.gd` + `scripts/units/unit.gd`

- UnitStats 新增：`@export var projectile_data: ProjectileData = null`（顶部 `preload("res://scripts/resources/projectile_data.gd")`）。
- `_perform_attack()`（第 910-922 行）改为按数据驱动：
  ```gdscript
  if stats_data and stats_data.projectile_data != null:
      _spawn_arrow(attack_target, damage)   # 走投射物
  elif unit_type == UnitType.ARCHER:
      _spawn_arrow(attack_target, damage)
  else:
      attack_target.take_damage(damage, self)
  ```
- `_spawn_arrow()`（第 923 行）把 `spawner.spawn_projectile` 的第一个参数从 `null` 改成 `stats_data.projectile_data`（让 shot_count/spread/effects 生效）。
- 顶部 MONK 不攻击的判断（第 912 行）增加条件：`if unit_type == UnitType.MONK and (stats_data == null or stats_data.projectile_data == null): return`，让 MONK 系远程兵（诅咒师/死灵师）能攻击。

### Enabler 3：反特化伤害加成 — 戟兵反骑、攻城槌反建筑
**文件**：`scripts/stats/unit_stats.gd` + `scripts/units/unit.gd` 的 `take_damage()`（第 928 行）

- UnitStats 新增：
  ```gdscript
  @export var bonus_vs_unit_types: Array[int] = []    # 对这些 UnitType 的伤害倍率生效
  @export var bonus_vs_multiplier: float = 1.0        # 倍率，如 2.0 = 双倍
  @export var bonus_vs_building_mult: float = 1.0     # 对建筑额外倍率
  ```
- 在 `take_damage` 调用点（`_perform_attack` 计算伤害后）加成：若目标 unit_type ∈ attacker.bonus_vs_unit_types → `damage = int(damage * bonus_vs_multiplier)`；若目标 is Building → `damage = int(damage * bonus_vs_building_mult)`。

### Enabler 4：单位光环 — 战鼓手
**文件**：`scripts/units/unit.gd`，复用 building.gd 第 149-151 行的 aura 模式

- UnitStats 新增：`@export var aura_range: float = 0.0`、`@export var aura_type: String = ""`（"attack"/"speed"/"defense"）、`@export var aura_value: float = 0.0`。
- unit.gd 用分帧扫描（已有 `SCAN_THROTTLE_FRAMES`）周期性给范围内的友军 `apply_buff()`。低频（10Hz）即可，性能无忧。

## 五、配置方式（如何添加一个新单位）

以「火焰法师 Pyromancer」为例，完整步骤：

**1. 创建属性资源**：在 `resources/stats/` 新建 `pyromancer_stats.tres`
- 设 `id = &"pyromancer"`、`unit_type = ARCHER`（复用弓箭手贴图与投射物逻辑）、`parent_id = &"archer"`（继承基础属性再覆盖）
- 填上表的 HP/攻击/射程/冷却/速度
- `projectile_data` 新建一个 `ProjectileData` 资源，`effects` 数组里拖入一个 `SplashEffect`（radius=70, splash_ratio=0.6）

**2. 创建场景**：在 `scenes/units/` 新建 `pyromancer.tscn`，继承 `archer.tscn`
- 根节点的 `variant_tint = Color("#E04020")`、`variant_tint_amount = 0.55`
- `stats_data` 指向刚建的 `pyromancer_stats.tres`

**3. 注册场景**：`scripts/systems/game_data.gd` 第 145 行 `ENEMY_VARIANT_SCENES` 加：
```gdscript
&"pyromancer": "res://scenes/units/pyromancer.tscn",
```

**4.（可选）让玩家可造**：若要玩家从建筑生产，加 Path A：
- game_data.gd `PlaceMode` 枚举加 `PYROMANCER`
- `COSTS` / `MODE_NAMES` / `MODE_ICONS` / `DISPLAY_ORDER` / 快捷键 各加一项
- building.gd 新增一个魔法塔建筑类型生产它，或让现有修道院/新建筑生产

**敌方关卡投放**：在 `resources/map*_config.tres` 的 `enemy_units` 数组加 `{type: 1, stats_id: "pyromancer", pos: ...}` 即可（驻军/波次系统已支持 stats_id，见 `docs/reference/驻军系统_配置教程.md`）。

## 七、验证方式

1. **Enabler 验证**：先做 Enabler 1（tint），运行游戏，在场景里手动给一个单位设 `variant_tint`，确认颜色变化且不影响势力贴图。
2. **Tier 1 逐个验证**：每做一个 Tier 1 兵种，在 map 配置里放一个敌方该兵种，运行：
   ```
   "E:\godot\Godot_v4.6.1-stable_win64.exe" --path "F:\godot_game\rts_base\rts-base"
   ```
   确认：贴图正确调色、攻击模式生效（散弹多箭、溅射多体掉血、减速冰蓝色、弩手高伤低频）。
3. **克制验证**（Tier 2）：放一个戟兵 vs 枪兵，确认 2.5x 伤害；放攻城槌打建筑，确认 4x 伤害。
4. **光环验证**：战鼓手站在友军旁，确认友军攻击力提升（伤害飘字变大）。
5. **新行为验证**（Tier 3）：自爆兵接近敌人确认爆炸+死亡；巨石卫士确认敌方转向打它；死灵法师确认骷髅生成且不超 3 只。

## 八、建议实施顺序

1. **第一步**：Enabler 1（tint）+ Enabler 2（projectile_data 解耦）—— 一次性解锁 Tier 1 的全部 6 个兵种。
2. **第二步**：批量做 Tier 1 的 6 个 `.tres` + `.tscn`，跑游戏验收。
3. **第三步**：Enabler 3 + 4，做 Tier 2 的 6 个兵种。
4. **第四步**：按兴趣挑 Tier 3 的 1-3 个做（自爆兵最便宜、死灵法师最贵）。
