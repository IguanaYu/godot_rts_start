# 单位模板扩展设计（第二组）：另 15 种新兵种

## Context（背景与目标）

第一组（`单位模板扩展设计_15兵种.md`）已经覆盖了肉盾/狂战/溅射/减速/反骑/攻城/光环/自爆/召唤等轴。**本组刻意避开重复**，探索第一组没碰过的全新机制轴：隐身、毒 DoT、吸血、连锁闪电、击退、闪现、劝化（招降）、火焰锥、护盾屏障、闪避、再生、复仇等。

设计原则不变：复用现有 4 套精灵 + 调色 + 缩放；尽量复用第一组的 Enabler（tint / projectile_data / bonus_vs / aura），新增的小机制作为 Enabler 5-11 补充。对称双方共用。

## 二、新增 Enabler（5-11，第一组 1-4 的补充）

第一组的 Enabler 1（tint）、2（projectile_data）、3（bonus_vs）、4（aura）本组直接复用。以下为本组特需的小机制。

### Enabler 5：闪避 / 吸血 / 再生（属性字段，最便宜）
**文件**：`scripts/stats/unit_stats.gd` + `scripts/units/unit.gd`
- UnitStats 新增：
  ```gdscript
  @export var dodge_chance: float = 0.0          # 0.0-1.0，受击时概率完全闪避
  @export var lifesteal_ratio: float = 0.0        # 造成伤害时按比例回血
  @export var regen_per_sec: float = 0.0          # 每秒被动回血
  ```
- `take_damage()`（第 928 行）开头：`if randf() < attacker_targeting_me... ` 实际是受击方判定，所以加在 `take_damage` 开头 `if randf() < stats_data.dodge_chance: return`（闪避飘字可选）。
- `_perform_attack()` 造成伤害后：`if lifesteal_ratio > 0: heal(int(damage * lifesteal_ratio))`。
- `_process` 加 regen 累积：`_regen_accumulator += regen_per_sec * delta; if >=1: heal(floor); ...`

### Enabler 6：毒 DoT（新 ProjectileEffect 子类）
**文件**：新建 `scripts/resources/poison_effect.gd`（仿 `slow_effect.gd`）+ unit.gd 加 DoT 追踪
- PoisonEffect：`@export var dps: float = 5.0`、`@export var duration: float = 4.0`，`apply()` 时调 `target.apply_poison(dps, duration)`。
- unit.gd 加 `_poison_dots` 数组与 `_process` 里扣血逻辑（类似 `_slow_timer` 的写法，第 79-80 行附近）。毒伤飘字用绿色。

### Enabler 7：连锁伤害（新攻击分支）
**文件**：`scripts/units/unit.gd` 的 `_perform_attack()`
- UnitStats 新增 `@export var chain_count: int = 0`、`@export var chain_falloff: float = 0.7`、`@export var chain_range: float = 120.0`。
- 攻击命中主目标后，按 `chain_range` 找最近 N 个敌方，依次造成 `damage * chain_falloff^i` 的伤害（仿飞龙弹射 / 闪电链）。伤害跳字用黄色。

### Enabler 8：击退（命中后位移）
**文件**：`scripts/units/unit.gd`
- UnitStats 新增 `@export var knockback_force: float = 0.0`。
- `take_damage` 后：`if attacker.knockback_force > 0: global_position += (global_position - attacker.global_position).normalized() * force`（瞬时位移，配合短暂的 nav 重置）。注意避开墙体——简单版直接位移即可。

### Enabler 9：驱散（清除 Buff/DeBuff）
**文件**：`scripts/units/unit.gd`
- UnitStats 新增 `@export var dispel_on_hit: bool = false`（清除目标身上的增益）和 `@export var cleanse_allies: bool = false`。
- 命中敌方时清空 `target.buffs`（第 78 行的 Dictionary）；治疗友军时清 `_slow_timer`。

### Enabler 10：隐身 / 闪现 / 复仇（新行为，中等代码）
- **隐身**：UnitStats `@export var stealth_idle: bool = false`——非攻击时 `body_sprite.visible = false`（或 alpha=0.3），且 `_find_closest_enemy_in_range` 跳过隐身单位。攻击/受击后短暂显形。
- **闪现**：UnitStats `@export var blink_range: float = 0.0`、`@export var blink_cooldown: float = 5.0`——追击时若距离 > 攻击范围且 blink 冷却好，直接 `global_position = 朝目标方向 blink_range`。
- **复仇**：UnitStats `@export var vengeance_on_ally_death: bool = false`——`_ready` 里连接附近友军的 `died` 信号，触发时 `apply_buff(ATTACK, +50%)` 持续 5 秒。

### Enabler 11：火焰锥 / 护盾屏障 / 劝化（较重，Tier 3）
- **火焰锥**：新攻击模式——不投射物，攻击瞬间对前方扇形（如 90°、range 80）内所有敌方造成伤害。在 `_perform_attack` 加 `cone_attack` 分支。
- **护盾屏障**：unit.gd 加 `_shield_hp` 字段，`take_damage` 先扣护盾；War Drummer 式 aura 给附近友军定期加护盾。
- **劝化**：MONK 系新状态——锁定一个敌方，引导 N 秒（不动、不能被打断阈值），成功后 `target.team = self.team`、`target.alliance_id` 切换。

## 四、两组设计的关系与搭配建议

第二组刻意和第一组**机制互补、不重叠**：

| 战术角色 | 第一组 | 第二组 | 搭配效果 |
|----------|--------|--------|----------|
| 肉盾 | 盾卫（减伤） | 再生者（续航）、守护者（护盾） | 不同抗伤模型 |
| 输出近战 | 狂战士（速攻） | 决斗者（闪避）、亡魂（吸血）、复仇者（死亡触发） | 不同生存模型 |
| 远程 | 弩手/散弹/火法/冰法 | 毒刃（DoT）、风暴（连锁）、投石车（超远溅射） | 不同伤害曲线 |
| 控场 | 冰法（减速）、诅咒 | 重锤（击退）、龙息（锥）、劝化（招降） | 不同控场维度 |
| 特殊 | 自爆、召唤、嘲讽 | 隐身、闪现、护盾 | 不同战术节奏 |
| 经济骚扰 | — | 狼骑兵/新兵暴兵 | 第一组缺骚扰轴 |

**关卡设计建议**：
- 第一关用第一组 Tier 1（直观的溅射/减速/穿甲）让玩家学机制。
- 中后期关卡混入第二组 Tier 2（毒/连锁/击退）增加变数。
- BOSS 关用第二组 Tier 3（劝化者招降你的兵、守护者给 BOSS 护盾）做机制战。

## 五、配置方式

与第一组完全一致，见 `单位模板扩展设计_15兵种.md` 第五节。差异点：

- **新 ProjectileEffect 子类**（PoisonEffect）：在 `scripts/resources/` 新建，仿 `slow_effect.gd`，挂到 ProjectileData 的 `effects` 数组。
- **新 UnitStats 字段**（dodge/lifesteal/regen/chain/knockback/stealth/blink 等）：在 `unit_stats.gd` 加导出变量后，所有 `.tres` 都能在编辑器看到并配置。
- **劝化/护盾/锥形**：需要新逻辑分支，建议放 Tier 3 最后做。

## 六、验证方式

延续第一组的逐 Tier 验证：

1. **Enabler 5 验证**：决斗者多次挨打观察闪避（伤害飘字偶尔缺失）；亡魂攻击后血条回涨；再生者站着不动血量缓慢上升。
2. **Enabler 6 验证**：毒刃射一个敌人，观察绿色毒伤持续掉血 4 秒。
3. **连锁验证**：风暴召唤者射密集敌群，确认 3 个目标依次受伤（黄色跳字递减）。
4. **击退验证**：重锤兵打近战，确认敌人被推开几像素、阵型被打散。
5. **隐身验证**：影刃非战斗时不可见，敌方不主动攻击它；攻击后短暂显形。
6. **劝化验证**：劝化者引导 4 秒后，一个敌方单位变蓝（加入玩家方）。

## 七、建议实施顺序（与第一组整合）

1. **先完成第一组的 Enabler 1-4 + Tier 1**（最高性价比）。
2. **第二组 Tier 1**（Enabler 5-6 极便宜）：新兵/决斗者/亡魂/再生者/毒刃/投石车——6 个新机制立刻可用。
3. **第二组 Tier 2**（Enabler 7-10）：连锁/击退/驱散/隐身/闪现/复仇——丰富战术维度。
4. **两组 Tier 3 按兴趣挑**：自爆、召唤、嘲讽（第一组）vs 火焰锥、护盾、劝化（第二组）。
