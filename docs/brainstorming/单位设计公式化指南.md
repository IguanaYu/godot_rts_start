# 单位设计公式化指南

把"设计一个新兵种"从一次性创作，变成**组件拼装 + 触发连线**。这样任何人（包括你自己以后回来）都能像配菜一样搓出新单位，而不必每次从零想。

---

## 零、核心模型

```
一个单位 = 基础属性(HP/攻/射程/速) + 若干【机制组件】 + 若干【触发连线】 + 外观(tint/scale)
```

- **机制组件**：从组件库挑（减伤、溅射、吸血……），每个组件有「强度档位」和「代价」。
- **触发连线**：把一个事件（命中、击杀、死亡……）连到一个效果（加速、回血、爆炸……）。这是组合爆炸的来源，也是最有意思的部分。
- **代价税则**：每加一份组件/连线，必须从基础属性里"还债"，保证平衡。

> 你的代码已经支持大半这套模型：`stat_set` 的 flat/multiplier 修饰器 = 强度档位；`ProjectileData.effects[]` = on_hit 触发连线；buff 系统 = 临时效果。少数连线（on_death、on_kill）需要少量新代码，文末标注。

---

## 一、机制组件库（带强度档位 + 代价）

每个组件给 **低 / 中 / 高** 三档数值，和加上它要付的**代价**。强度档位就是"一点 / 一些 / 全压"的量化。

### 生存类
| 组件 | 低档 | 中档 | 高档 | 代价（还债） |
|------|------|------|------|--------------|
| 减伤 | 0.15 | 0.30 | 0.50 | 降移速 10-30% |
| 闪避 | 15% | 30% | 45% | 降血 20-40%（看脸） |
| 再生 | 2/s | 4/s | 7/s | 降攻击或加造价 |
| 吸血 | 20% | 40% | 60% | 降血（脆但吸回） |
| 护盾（临时血） | 30 | 60 | 100 | 短时效，需刷新 |
| 嘲讽 | — | — | 开关 | 自身必须够硬 |

### 伤害类
| 组件 | 低档 | 中档 | 高档 | 代价 |
|------|------|------|------|------|
| 单发高伤 | 15 | 22 | 35 | 慢冷却（1.5-2.5s） |
| 高攻速 | 0.5s cd | 0.35s | 0.2s | 降单发伤害 |
| 反特化倍率 | 1.8x | 2.5x | 4x | 对其他目标很弱 |
| 近程投掷 | 射程 70 | 90 | 120 | 介于近战远程间 |

### 范围 AoE
| 组件 | 低档 | 中档 | 高档 | 代价 |
|------|------|------|------|------|
| 点溅射 radius | 40 | 70 | 110 | 降单发 30-50% |
| 散射 shot_count | 2 | 3 | 5 | 单发低、散布不准 |
| 火焰锥 | 60°/range 60 | 90°/80 | 120°/100 | 必须朝向贴脸 |
| 连锁 | 2 跳 | 3 跳 | 5 跳 | 单发低，跳目标随机 |

### DoT 持续伤害
| 组件 | 低档 | 中档 | 高档 | 代价 |
|------|------|------|------|------|
| 毒 | 4 dps / 4s | 8/4s | 12/5s | 爆发低 |
| 燃烧 | 6/2s | 10/3s | 16/3s | 怕驱散，可叠层 |

### 控制减益
| 组件 | 低档 | 中档 | 高档 | 代价 |
|------|------|------|------|------|
| 减速 | 25% | 40% | 55% | 伤害低 |
| 击退 force | 15 | 28 | 45 | 近战 |
| 眩晕 dur | 0.5s | 1s | 2s | 长冷却 + 低伤 |
| 驱散 | — | — | 开关 | 几乎无伤害 |

### 增益光环
| 组件 | 低档 | 中档 | 高档 | 代价 |
|------|------|------|------|------|
| 加攻光环 | +15% | +25% | +40% | 自身面板弱 |
| 减伤光环 | +10% | +20% | +30% | 同上 |
| 回血光环 | 2/s | 4/s | 7/s | 同上 |
| 移速光环 | +15% | +25% | +35% | 同上 |

### 机动类
| 组件 | 低档 | 中档 | 高档 | 代价 |
|------|------|------|------|------|
| 高速 | 140 | 170 | 210 | 降血或甲 |
| 闪现 | 100/cd5 | 160/cd5 | 240/cd4 | 冷却长 |
| 隐身（非战斗） | — | — | 开关 | 显形后脆 |
| 冲刺 | +40%/1s | +60%/1.5s | +100%/2s | 冷却制 |

> **一句话档位法**：低档 = "一点"（锦上添花），中档 = 兵种主特征，高档 = 兵种就是为这个活的。

## 二、触发连线（组合爆炸的来源）

这是公式化的灵魂。把**事件**接到**效果**上。

### 触发事件（左边）
| 事件 | 说明 | 代码状态 |
|------|------|----------|
| `on_attack_hit` | 攻击命中目标 | ✅ 已有（ProjectileData.effects / 近战命中点） |
| `on_attack_start` | 开始攻击动作 | 🟡 需小改（_perform_attack 开头挂钩） |
| `on_kill` | 击杀一个单位 | 🟡 需小改（take_damage 致死时回调攻击者） |
| `on_attacked` | 被攻击（无论是否命中） | 🟡 需小改 |
| `on_take_damage` | 实际掉血 | ✅ 已有（take_damage 内） |
| `on_death` | 自己死亡 | ✅ 已有（die() 信号） |
| `on_ally_death` | 附近友军死亡 | 🟡 需小改（连 died 信号） |
| `on_spawn` | 生成时 | ✅ _ready |
| `on_enter_combat` | 锁定首个敌人 | 🟡 需小改 |
| `tick_periodic` | 周期触发 | ✅ 已有（_process / aura 扫描） |
| `on_low_hp` | 血量低于阈值 | 🟡 需小改 |

### 效果动作（右边）
| 效果 | 说明 | 代码状态 |
|------|------|----------|
| `deal_damage` | 造伤害（单体/AoE/DoT） | ✅ |
| `buff_self` | 给自己临时 buff（加速/加攻/减伤/护盾） | ✅ apply_buff |
| `debuff_target` | 给目标 debuff（减速/中毒/降攻） | ✅ apply_slow / PoisonEffect |
| `heal` | 治疗 | ✅ heal() |
| `spawn_unit` | 召唤 | 🟡 死灵法师那套 |
| `blink` | 位移 | 🟡 闪现者那套 |
| `knockback` | 击退 | 🟡 重锤兵那套 |

### 标准连线写法
```
[触发事件] → [效果动作] + [强度档位] + [冷却/持续时间]
```

例：
- `on_attack_hit → debuff_target(减速, 中, 2s)`
- `on_kill → buff_self(加速, +50%, 1s)` ← 你说的"范围伤害攻击之后获得1秒加速"
- `on_death → deal_damage(AoE, radius 80, 80伤)`
- `on_ally_death → buff_self(加攻速, +50%, 5s)`

## 三、配方模板（常见有效组合，可直接抄）

把组件+连线打包成"风格配方"。每个配方给建议数值，你按需调档位。

### 配方 A — 骚扰刺客（高机动 + 爆发 + 脱离）
- 组件：高速(中 170) + 隐身 + 单发高伤(中 22)
- 连线：`on_kill → buff_self(加速+60%, 1.5s)`（杀完就跑）
- 代价：血量 55（脆）
- 手感：来无影去无踪，杀一个就溜

### 配方 B — 游击法师（你的例子1：一点机动 + 一点 AoE）
- 组件：高速(低 150) + 点溅射(低 radius 45)
- 代价：降血 20%（HP ~55）
- 手感：打得动一片，跑得也不慢，但都不极致

### 配方 C — 动能术士（你的例子2：AoE 后加速）
- 组件：点溅射(中 radius 70) + 标准 remote
- 连线：`on_attack_hit → buff_self(加速 +40%, 1s)`
- 代价：单发降 30%（伤害靠溅射 + 走位补）
- 手感：越打越欢，打完一发就位移拉开

### 配方 D — 团战核心（光环 + 半肉）
- 组件：加攻光环(中 +25%) + 减伤(低 0.15) + 中血 150
- 连线：无（被动）
- 代价：自身攻击平庸
- 手感：站后排给全队加攻，自己也不容易死

### 配方 E — 自爆冲锋（死亡触发）
- 组件：高速(低 140) + 低血 50
- 连线：`on_death → deal_damage(AoE radius 80, 80伤)`
- 代价：不能持续作战
- 手感：冲进人堆换伤害

### 配方 F — 续航战士（吸血 + 减伤）
- 组件：吸血(中 40%) + 减伤(低 0.2) + 标准近战
- 连线：无（被动吸血）
- 代价：爆发低
- 手感：越打血越多，怕控杀

### 配方 G — 风筝弓（投掷 + 减速）
- 组件：近程投掷(中 90) + 减速(低 25% on_hit)
- 代价：被贴脸就完
- 手感：边退边射，粘住对手

### 配方 H — 控场法师（减速 + 溅射）
- 组件：减速(中 40%) + 点溅射(低 radius 45)
- 连线：`on_attack_hit → debuff_target(减速, 中, 2.5s)`
- 代价：伤害低
- 手感：让对方走不动，慢慢磨

### 配方 I — 暴君狂战（攻速 + 死亡触发）
- 组件：高攻速(高 0.2s) + 低血 60
- 连线：`on_ally_death → buff_self(加攻速 +50%, 5s)`
- 代价：血薄
- 手感：友军一倒，自己变绞肉机

### 配方 J — 反肉专家（穿甲 + DoT）
- 组件：穿甲(无视减伤) + 毒(中 8dps)
- 连线：`on_attack_hit → apply_poison(中, 4s)`
- 代价：单发低
- 手感：专治盾卫/再生者这种大肉

## 四、平衡税则（每加一份机制要还多少债）

没有免费的组件。经验法则（可调）：

| 加的组件 | 建议还债 |
|----------|----------|
| 任何 AoE（溅射/散射/锥/连锁） | 单发伤害 -30~50%，或血量 -20% |
| 任何硬控（眩晕/击退/招降） | 伤害 -40%，或加长冷却 |
| 任何续航（吸血/再生/护盾） | 攻击力 -20%，或血量基准线下调 |
| 任何高机动（闪现/隐身/高速高档） | 血量 -25~40% |
| 任何光环（带 team 价值） | 自身所有面板 -20% |
| 任何死亡触发（高戏剧性） | 基础属性可正常，因为触发条件苛刻 |

**简单平衡检验**：新兵种按配方搓完后，想象 1v1 打现有的战士/弓箭手——应该有胜负场景（克制某方、被某方克），而不是通杀。

## 五、设计工作流（从零搓一个新兵种）

1. **定角色一句话**：「我想做个 ___ 的单位」（例："打完就跑的法师"）。
2. **挑 1-2 个主组件**（中档），最多再加 1 个副组件（低档）。超过 3 个就太杂。
3. **加 0-1 条触发连线**（这是个性来源）。
4. **按税则还债**：把基础属性砍到合理。
5. **选 tint + scale** 区分外观。
6. **写一句话"克制关系"**：它克谁、被谁克。
7. 落到 `.tres` + `.tscn`。

### 示例：搓一个"风暴游侠"
1. 角色："边跑边射连锁闪电的法师"
2. 主组件：连锁(中 3 跳)，副：高速(低 145)
3. 连线：`on_kill → buff_self(移速 +30%, 1.5s)`（杀完加速继续跑）
4. 还债：单发 6（很低），血量 50
5. tint 电蓝 #3A6AE0，scale 1.0
6. 克制：密集脆皮（连锁弹射最大化）；被克：高血近战贴脸
7. 落配置。

整个过程 5 分钟出设计，10 分钟出配置。

## 六、公式化的真正价值

- **填空创作**：不用想"做什么兵种"，而是想"哪格组件还没用过"——空格自动变成新兵种。
- **可解释的平衡**：某个兵种过强，看它的组件档位总和是否超标，砍档位即可，而不是凭感觉改数。
- **关卡快速配兵**：关卡设计者按配方 A/B/C 组合敌方阵容，机制天然多样。
- **逐步代码化**：组件库右边标了代码状态（✅/🟡），可按需解锁新组件——先做全绿（已有代码）的配方，再投资做 🟡 的。

## 七、组件 ↔ 代码实现对照表

| 组件/连线 | 实现位置 | 状态 |
|-----------|----------|------|
| 强度档位（flat/mult） | `stat_set.add_modifier` | ✅ |
| on_hit → debuff/damage | `ProjectileData.effects[]` (SplashEffect/SlowEffect/PoisonEffect) | ✅ |
| on_hit → buff_self | 需在 arrow `_on_hit` 回调 shooter，或近战命中点 apply_buff | 🟡 小改 |
| on_death → 效果 | `die()` 信号 + spawner 生成爆炸 | 🟡 小改 |
| on_kill → 效果 | `take_damage` 致死时回调 `attacker.on_kill(target)` | 🟡 小改 |
| on_ally_death → 效果 | `_ready` 连接 UnitGrid 邻居的 `died` 信号 | 🟡 小改 |
| tick_periodic → 效果 | `_process` / 现有 aura 扫描 | ✅ |
| 减伤/闪避/吸血/再生 | UnitStats 字段 + take_damage / _perform_attack | 🟡 加字段（第一/二组 Enabler） |
| 光环 | building.gd 已有，单位版需移植 | 🟡 |

**建议落地顺序**：先把 ✅ 全绿的组件和连线做熟（能出大量配方），再按需补 🟡。
