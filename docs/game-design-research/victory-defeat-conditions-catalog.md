# RTS 胜负条件分类设计大全

## Context

当前游戏有 4 种胜负模式（DestroyBase / Blitz / SurviveWaves / ExpandDefense），失败条件比较单一（城堡被毁或全灭）。本设计将所有可能的胜负条件系统分类，为后续关卡设计提供"菜单"参考。

现有基础设施：
- `VictoryCondition` 基类：`check()` 返回 0/1/2，`game_ended` 信号
- 单位系统：`died` 信号、`is_dead()`、分组 `player_units`/`enemy_units`、`stats_data.category`（normal/hero/boss）
- 建筑系统：`died` 信号、`is_dead()`、`BuildingType` 枚举、分组 `player_buildings`/`enemy_buildings`
- 据点系统：`CapturePoint` 已有触发→激活→占领流程，带奖励
- 波次系统：`WaveManager` 有 `all_waves_completed` 信号
- 经济系统：`gold` 变量、`add_gold()` 方法

---

## 一、单位相关（Unit）

### 1.1 关键单位存活（VIP Survival）
> **魔兽3经典机制** — "英雄必须存活"

- **胜利**：完成其他目标时该单位仍存活
- **失败**：指定单位死亡
- **配置**：`@export var vip_unit_path: NodePath`
- **变体**：
  - 单个VIP存活（Boss战、英雄关卡）
  - 多个VIP全部存活（"铁三角必须存活"）
  - 至少N个VIP存活（允许部分牺牲）
- **灵感来源**：魔兽3几乎所有任务、SC2虫心之多数关卡
- **现有基础**：`unit.died` 信号可直接连接

### 1.2 全军覆没（Annihilation）
> 已有实现 — VictoryBlitz 使用

- **失败**：己方所有单位死亡
- **配置**：遍历 `player_units` 分组检查 `is_dead()`
- **变体**：
  - 所有单位死亡（当前实现）
  - 某类型单位全灭（如"所有弓箭手阵亡则失败"）
  - 单位数低于阈值（如"部队不足5人则失败"）

### 1.3 护送（Escort）
> **SC2/魔兽3经典任务类型**

- **胜利**：护送目标到达指定地点
- **失败**：护送目标死亡
- **配置**：`@export var escort_target: NodePath` + `@export var destination: NodePath`
- **实现要点**：
  - 护送单位沿固定路线移动（或跟随玩家部队）
  - 目的地用 Area2D 检测到达
  - 沿途可能有伏兵/波次
- **灵感来源**：SC2"恶魔乐园"护送难民、C&C护送车队

### 1.4 刺杀/击杀指定单位（Assassinate）
> **SC2"至尊"关卡** — 击杀3个原始族群首领

- **胜利**：击杀指定敌方单位
- **配置**：`@export var target_units: Array[NodePath] = []`
- **实现要点**：连接目标单位的 `died` 信号，全部死亡时胜利
- **变体**：
  - 击杀单个Boss（现有 `stats_data.category = "boss"` 可直接用）
  - 击杀多个指定目标（击杀清单）
  - 击杀特定类型的单位（如"消灭所有敌方英雄"）
- **现有基础**：`CapturePoint.KILL_TARGETS` 触发类型已实现类似逻辑

### 1.5 击杀计数（Kill Count）
> **帝国时代/SC2成就目标**

- **胜利**：击杀N个敌方单位
- **失败**：—（通常配合其他失败条件）
- **配置**：`@export var kill_target: int = 50`
- **实现要点**：连接所有敌方单位的 `died` 信号，维护计数器
- **现有基础**：`CapturePoint` 的 `MISSION_COUNTER` 已有杀敌计数
- **变体**：
  - 总击杀数
  - 击杀某类型单位数（"消灭30个弓箭手"）
  - 击杀英雄/Boss级单位数

## 二、建筑相关（Building）

### 2.1 摧毁主基地（Destroy Main Base）
> 已有实现 — VictoryDestroyBase

- **胜利**：敌方城堡被摧毁
- **失败**：己方城堡被摧毁
- **现有基础**：完整实现，遍历 `BuildingType.CASTLE`

### 2.2 摧毁指定建筑（Destroy Specific Buildings）
> **SC2"粉碎天空"** — 摧毁4个冷却塔

- **胜利**：摧毁所有指定建筑
- **配置**：`@export var target_buildings: Array[NodePath] = []`
- **实现要点**：连接目标建筑的 `died` 信号
- **变体**：
  - 摧毁N个同类建筑（"摧毁3座敌方兵营"）
  - 摧毁任意组合（"摧毁敌方所有生产建筑"）
  - 摧毁优先级目标（先A后B才能暴露C）

### 2.3 保护建筑（Protect Building）
> **SC2"孤注一掷"** — 保护神器

- **失败**：指定建筑被摧毁
- **配置**：`@export var protected_buildings: Array[NodePath] = []`
- **变体**：
  - 保护单个关键建筑（神器、村民中心）
  - 保护多个建筑（"三座塔不能全部被毁"）
  - 保护NPC建筑（联盟基地）

### 2.4 建造目标（Build Target）
> **帝国时代"奇观胜利"** + **SC2"arence"收集资源建神器**

- **胜利**：建造指定建筑并保持X秒
- **配置**：`@export var required_building_type: BuildingType` + `@export var hold_time: float`
- **变体**：
  - 建造指定类型建筑（"建造城堡"）
  - 建造奇观并守护（帝国时代经典）
  - 升级建筑到指定等级

## 三、收集/资源类（Collection）

### 3.1 收集金币（Gold Collection）
> 利用现有 `gold` 系统

- **胜利**：累计获得N金币
- **配置**：`@export var gold_target: int = 1000`
- **实现要点**：每帧检查 `gold` 值，或连接 `add_gold()` 回调
- **现有基础**：`main.gd` 已有 `gold` 变量和 `add_gold()` 方法
- **变体**：
  - 累计总收入（非当前持有）
  - 当前持有N金币（可花掉但不影响目标）

### 3.2 收集物品/拾取（Item Collection）
> **SC2神器任务**

- **胜利**：收集地图上所有物品/N个物品
- **实现要点**：在地图上放置可拾取的 Area2D 节点，单位靠近触发拾取
- **变体**：
  - 散布在地图上的宝箱
  - 击杀敌人掉落收集物
  - 限时收集（配合时间条件）

### 3.3 占领据点（Capture Points）
> 已有实现 — VictoryBlitz / VictoryExpandDefense

- **胜利**：占领所有据点
- **现有基础**：`CapturePoint` 系统完整

---

## 四、时间相关（Time）

### 4.1 限时生存/坚守（Survive Timer）
> 已有实现 — VictorySurviveWaves

- **胜利**：坚持到计时结束
- **失败**：城堡被毁 / 全灭
- **现有基础**：波次系统间接实现（撑过所有波次=撑过时间）
- **变体**：
  - 纯倒计时（不依赖波次，就是坚持N分钟）
  - 递增难度（随时间敌人越来越强）
  - 阶段性里程碑（"坚守第5分钟获得增援"）

### 4.2 限时完成（Time Attack）
> **SC2成就** — "在X分钟内完成"

- **胜利**：在时限内完成其他目标
- **失败**：超时（即使其他目标还没失败）
- **配置**：`@export var time_limit: float = 300.0`（秒）
- **实现要点**：`_ready()` 记录开始时间，`check()` 中计算剩余
- **变体**：
  - 硬性时间限制（超时直接失败）
  - 软性限制（超时后敌人增强，不直接失败）
  - 分段计时（每阶段有独立时限）

### 4.3 倒计时事件（Countdown Event）
> **SC2"数码清除"** — 倒计时结束引发某事件

- **效果**：倒计时结束后触发事件（不一定是胜负）
- **变体**：
  - 倒计时结束召唤Boss
  - 倒计时结束敌人增援到达
  - 倒计时结束某区域变为致命区域
  - 玩家可选择加速/延缓倒计时

## 五、区域/领土类（Territory）

### 5.1 到达指定位置（Reach Location）
> **C&C撤离任务**

- **胜利**：指定单位到达目标位置
- **配置**：`@export var destination_area: NodePath`
- **实现要点**：Area2D 检测指定单位进入
- **变体**：
  - 所有单位到达
  - 至少一个单位到达
  - 在时限内到达（配合时间条件）

### 5.2 区域清除（Area Clear）
> 现有 `CapturePoint.TriggerType.CLEAR_AREA` 类似

- **胜利**：清除指定区域内所有敌人
- **配置**：`@export var area_center: NodePath` + `@export var area_radius: float`
- **变体**：
  - 清除一个区域
  - 依次清除多个区域（推进战线）
  - 同时清除多个区域（分兵策略）

### 5.3 领土控制积分（Territory Score）
> **英雄连** 占点扣分制

- **胜利**：敌方积分归零
- **失败**：己方积分归零
- **实现要点**：
  - 双方有基础积分（如500）
  - 占领据点多的一方持续扣对方分
  - 据点越多扣分越快
- **灵感来源**：Company of Heroes 胜利点模式

---

## 六、波次/生存类（Wave Survival）

### 6.1 存活波次（Survive Waves）
> 已有实现 — VictorySurviveWaves

- **胜利**：所有波次完成 + 残敌清除
- **失败**：城堡被毁
- **现有基础**：完整实现

### 6.2 无尽模式（Endless Mode）
> **TD类游戏常见**

- **胜利**：无硬性胜利，以存活波数/得分衡量
- **失败**：城堡被毁
- **实现要点**：波次无限循环，难度递增
- **变体**：
  - 纯排行榜模式
  - 每10波一个Boss波

---

## 七、复合条件（Composite）

### 7.1 多条件组合（AND/OR）
> 已有实现 — VictoryExpandDefense（据点 + 波次 + 残敌）

- **AND组合**：所有条件都满足才胜利
- **OR组合**：任一条件满足就胜利
- **混合组合**：胜利 = (A AND B) OR C

### 7.2 多阶段任务（Multi-stage）
> **SC2大多数任务** — 目标动态变化

- **流程**：阶段1目标 → 完成后切换 → 阶段2目标 → ...
- **实现要点**：状态机驱动，每阶段有独立条件
- **示例**：
  - 阶段1：清除前进营地 → 阶段2：护送物资车到前方 → 阶段3：坚守3分钟
  - 阶段1：收集3个神器碎片 → 阶段2：建造封印祭坛 → 阶段3：击杀Boss

### 7.3 条件变化（Dynamic Conditions）
> **魔兽3"诸神之黄昏"** — 中途加入新目标

- **特点**：游戏中途新增或改变目标
- **示例**：
  - 原本只需摧毁基地，5分钟后联盟NPC被攻陷，新增"保护剩余联盟"目标
  - Boss血量50%时召唤增援，新增"击杀增援"条件
  - 玩家选择分支（二选一目标）

## 八、额外/可选目标（Bonus Objectives）

### 8.1 副目标（Side Objectives）
> **SC2三成就系统** / **魔兽3可选任务**

- **性质**：不影响胜负，但给额外奖励
- **奖励类型**：
  - 金币奖励
  - 额外单位增援
  - 解锁科技/建筑
  - 星级/评分（影响关卡总评）
- **示例**：
  - "找到隐藏宝箱"（地图探索）
  - "在5分钟内完成"（速度挑战）
  - "不损失任何单位"（完美通关）
  - "击杀隐藏Boss"（额外挑战）
- **现有基础**：`CapturePoint` 的奖励系统可复用

### 8.2 评分/星级系统（Rating System）
> **SC2成就** / **手游三星制**

- **一星**：完成主线目标
- **二星**：完成主线 + 副目标
- **三星**：完成所有挑战（限时/无损/隐藏目标）
- **现有基础**：`SaveManager` 已有分数系统

---

## 九、特殊/创意条件（Creative）

### 9.1 对称/非对称（Asymmetric）
- 玩家防守，NPC进攻（塔防）
- 玩家只有一个英雄，RPG式闯关
- 双方各有不同的胜利条件（你毁我基地，我撑过10分钟）

### 9.2 资源争夺（Resource Race）
> **帝国时代** + **Northgard经济胜利**

- **胜利**：率先采集/持有N资源
- **变体**：贸易收入达到阈值、人口达到上限、科技全解锁

### 9.3 斩首行动（Decapitation）
- **胜利**：仅击杀敌方指挥官单位
- **失败**：己方指挥官死亡
- **特点**：不需要摧毁基地，刺杀即可

## 十、按"推荐优先级"排序

考虑到现有代码基础和实现难度，建议按以下优先级添加：

| 优先级 | 条件类型 | 理由 |
|---|---|---|
| ⭐⭐⭐ | 1.1 关键单位存活 | 实现简单（连信号），大幅增加关卡设计空间 |
| ⭐⭐⭐ | 1.4 刺杀/击杀目标 | 已有 KILL_TARGETS 基础，改动极小 |
| ⭐⭐⭐ | 2.2 摧毁指定建筑 | 类似 DestroyBase，扩展为指定建筑即可 |
| ⭐⭐ | 4.2 限时完成 | 新增一个计时器，通用性极高 |
| ⭐⭐ | 1.3 护送 | 新任务类型，需要路线系统但体验很不同 |
| ⭐⭐ | 8.1 副目标系统 | 架构改造，一旦完成所有关卡受益 |
| ⭐ | 3.1 收集金币 | 利用现有 gold 系统 |
| ⭐ | 5.1 到达指定位置 | 简单但能做出逃离关卡 |
| ⭐ | 7.2 多阶段任务 | 架构改造，复杂但极大提升体验 |
| ⭐ | 5.3 领土积分 | 全新模式，适合PVP |

## 验证方式

1. 每种新条件创建独立 `.gd` 文件继承 `VictoryCondition`
2. 在测试地图中挂载验证
3. 确认 `main.gd` 的 `_setup_victory_condition()` 能自动发现
4. 确认胜利/失败UI正确显示
