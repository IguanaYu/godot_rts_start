# 建筑驻军系统 — 配置教程

## 一、架构总览

### 节点结构

```
Building (Node2D, building.gd)
├── HealthComponent (Node2D)
├── StaticBody2D
├── BodySprite (Sprite2D)
├── HPBar (ProgressBar)
├── AggroLine (Line2D)
└── Garrison (Node, building_garrison.gd)  ← 驻军系统
```

Garrison 节点挂载在 **building.tscn**（建筑基类场景）上，所有建筑子场景（城堡、兵营、靶场、城墙、修道院）自动继承。箭塔不启用驻军。

### 启用条件

`building_garrison.gd` 的 `_ready()` 中自动判断：

| 条件 | 结果 |
|------|------|
| 建筑 team = PLAYER (0) | 禁用 |
| 建筑 building_type = TOWER (1) | 禁用 |
| 其他敌方建筑 | 启用 |

不需要手动给每个建筑挂脚本，只要建筑是敌方的且不是箭塔，就自动生效。

### 信号连接

building.gd 在受到伤害时发射 `damaged(amount, attacker)` 信号，Garrison 自动监听，根据血量比例决定是否释放驻军。

## 二、工作流程

```
开局
 └─ _ready() → 按建筑类型设置默认配置 → 填满驻军到 max_capacity

运行中（每帧 _process）
 ├─ _production_tick() → 驻军未满时按间隔生产新单位
 ├─ _process_release_queue() → 逐个释放队列中的单位（每0.5s一个）
 └─ _update_garrison_bar() → 更新蓝色驻军条

建筑受击（_on_building_damaged）
 ├─ 血量 ≤ 75%（首次）→ _try_release()，受CD限制
 ├─ 血量 ≤ 40%（首次）→ _force_release()，无视CD
 └─ 建筑被摧毁 → 不释放，清空驻军（奖励快速拆建筑的玩家）

外部触发
 └─ 情绪系统 seek_garrison → 单位跑到建筑旁 → garrison.alert_from_ally() → _try_release()
```

### 释放过程

释放不是一瞬间全部出来，而是逐个释放：

1. 从 garrison 数组取出 `release_batch_size` 个单位放入 `_release_queue`
2. 每 `release_interval`（默认0.5秒）从队列取出一个，在建筑旁生成
3. 生成的单位自带 EnemyAI，立即投入战斗

## 三、默认配置

| 建筑类型 | building_type | 容量 | 生产间隔 | 释放批次 | 兵种配比（权重） |
|---------|:---:|:---:|:---:|:---:|------|
| 城堡 | 2 | 12 | 12s | 4 | 士兵3 : 弓1 : 枪1 |
| 兵营 | 3 | 8 | 10s | 3 | 士兵3 : 弓1 |
| 靶场 | 5 | 6 | 10s | 3 | 弓2 : 士兵1 |
| 修道院 | 4 | 4 | 15s | 2 | 僧侣1 |
| 城墙 | 0 | 3 | 20s | 2 | 士兵1 |

单位类型编号：士兵=0, 弓兵=1, 枪兵=2, 僧侣=3

## 四、如何自定义配置

### 方法一：在编辑器中覆盖（推荐）

1. 双击打开某个建筑子场景（如 `barracks.tscn`）
2. 选中根节点，在 Inspector 面板中找到 **Garrison** 节点的属性
3. 修改以下 @export 参数：

| 参数 | 说明 | 默认值 |
|------|------|-------|
| `garrison_config` | 兵种配比数组 | 按建筑类型自动填充 |
| `production_cooldown` | 生产一个单位的间隔（秒） | 15.0 |
| `max_capacity` | 驻军最大容量 | 10 |
| `release_batch_size` | 每次释放几个 | 3 |
| `release_cooldown` | 释放之间的冷却时间（秒） | 10.0 |
| `release_interval` | 逐个释放的时间间隔（秒） | 0.5 |
| `trigger_hp_high` | 首次触发的血量比例 | 0.75 |
| `trigger_hp_low` | 强制触发的血量比例 | 0.40 |

> **注意**：`garrison_config` 只在非空时生效。如果填了自定义值，就不会使用默认配置。如果想恢复默认，清空这个数组即可。

#### garrison_config 数组格式

每个元素是一个 Dictionary，包含：

```gdscript
{
    "type": 0,        # 单位类型：0=士兵, 1=弓兵, 2=枪兵, 3=僧侣
    "weight": 3,      # 权重（出现概率比例）
    "stats_id": ""    # 变体ID，留空用普通单位，填变体名用变体场景
}
```

**权重例子**：`[{type:0, weight:3}, {type:1, weight:1}]` 表示大约 3/4 概率出士兵、1/4 概率出弓兵。

**变体例子**：`{type:1, weight:1, stats_id:"elite_archer"}` 会使用 `game_data.gd` 中 `ENEMY_VARIANT_SCENES` 注册的精英弓兵场景。

### 方法二：修改默认配置（影响所有同类建筑）

编辑 `scripts/buildings/building_garrison.gd` 的 `_apply_default_config()` 函数：

```gdscript
func _apply_default_config(btype: int) -> void:
    match btype:
        2:  # CASTLE — 改这里影响所有敌方城堡
            garrison_config = [{"type": 0, "weight": 3}, {"type": 1, "weight": 1}, {"type": 2, "weight": 1}]
            max_capacity = 12
            production_cooldown = 12.0
            release_batch_size = 4
        3:  # BARRACKS — 改这里影响所有敌方兵营
            # ...
```

### 方法三：新增建筑类型

如果添加了新的建筑类型（如 FORT），在 `_apply_default_config` 的 match 中加一个分支：

```gdscript
6:  # FORT
    garrison_config = [{"type": 0, "weight": 2}, {"type": 2, "weight": 2}]
    max_capacity = 10
    production_cooldown = 8.0
    release_batch_size = 4
```

同时在 `game_data.gd` 的 `UNIT_SCENES` 中确保有对应单位类型的场景路径。

## 五、外部接口

其他系统可以通过以下方式与驻军系统交互：

### 情绪系统调用

当单位触发 `seek_garrison` 情绪时，会跑到最近的敌方建筑旁调用：

```gdscript
var garrison = building.get_node_or_null("Garrison")
if garrison and garrison.has_method("alert_from_ally"):
    garrison.alert_from_ally()
```

### 查询驻军数量

```gdscript
var garrison = building.get_node_or_null("Garrison")
var count = garrison.get_garrison_count()  # 返回驻军 + 释放队列中的总数
```
