# 大战术释放者 — 配置与生成教程

> 设计意图与避坑指南见 [reference/design/大战术释放_设计要点与避坑指南.md](../reference/design/大战术释放_设计要点与避坑指南.md)。本文档只讲怎么配、怎么生成、怎么调。

## 一、架构总览

### 节点结构

```
GrandTacticReleaser (CharacterBody2D, unit.gd)  ← 继承自 soldier.tscn
├── ... (soldier 原有子节点)
└── GrandTacticReleaserAI (Node, grand_tactic_releaser_ai.gd)  ← 引导逻辑
    └── (运行时) ChannelVisual (Node2D)  ← 视觉光环，加在 owner_unit 上
```

### 文件分布

| 用途 | 路径 |
|---|---|
| 单位 stats | `resources/stats/grand_tactic_releaser_stats.tres` |
| 单位场景 | `scenes/units/grand_tactic_releaser.tscn` |
| AI 脚本 | `scripts/units/grand_tactic_releaser_ai.gd` |
| 视觉光环 | `scripts/units/grand_tactic_channel_visual.gd` |
| 战术 Resource 基类 | `scripts/tactics/grand_tactic_resource.gd` |
| 4 种战术配置 | `resources/tactics/tactic_*.tres` |
| 测试场景 | `scenes/maps/test_grand_tactic.tscn` |
| 测试 spawner | `scripts/systems/test_grand_tactic_spawner.gd` |
| 调试按钮面板 | `scripts/ui/grand_tactic_test_panel.gd` |
| 生成 API | `scripts/systems/game_spawner.gd:512 spawn_grand_tactic_releaser()` |

### 工作流程

```
生成（spawn_grand_tactic_releaser 或测试面板按钮）
 └─ AI._ready()
    ├─ 设 attack_disabled = true（不反击）
    ├─ hold_position.call_deferred()（不移动、不追击）
    ├─ 加入 "grand_tactic_releaser" 组（owner_unit 也加，方便 _aura_process 识别）
    ├─ _show_channel_visual("start") → call_deferred 创建 ChannelVisual 子节点
    └─ _apply_tactic_start.call_deferred() → 按 tactic_id 分发：
       ├─ SUMMON_ELITE: 立刻召唤精英 + link_minion（联动死亡注册）
       ├─ MORALE_BOOST: 设 owner_unit.aura_type/value/range
       ├─ SUMMON_WAVE: 等 tick（_apply_tactic_tick 在 _process 调用）
       └─ FINAL_SUMMON: 等 complete（_apply_tactic_complete）

运行中（AI._process 每帧）
 ├─ _channel_progress += delta
 ├─ 每 effect_interval 秒：触发 _apply_tactic_tick（SUMMON_WAVE 用）
 ├─ 视觉更新（光环进度填充）
 ├─ 中途被击杀 → _interrupt() → 联动死亡清理（如果有 linked_death）
 └─ _channel_progress >= channel_time → channel_completed + _apply_tactic_complete（FINAL_SUMMON 用）
```

## 二、单位属性配置

`grand_tactic_releaser_stats.tres` 的关键值（**已验证，不要乱改**，详见 [memory: grand-tactic-releaser-config](../../../../C:/Users/86132/.claude/projects/e--godot-rts-godot-rts-start/memory/project_grand_tactic_releaser.md)）：

| 字段 | 值 | 说明 |
|---|---|---|
| `max_hp` | 2000 | SLOW 节奏下 ~80s 才能被 5 兵打死，30s 引导期内打不死 |
| `attack_damage` | 0 | 不反击 |
| `attack_range` | 0 | 不反击 |
| `attack_cooldown` | 0 | 不反击 |
| `move_speed` | 0 | 不追玩家、不卡路径，纯引导桩 |
| `sprite_scale` | 1.0 | stats 不乘倍数，由 scene 控制 |
| `category` | "boss" | 用于分类筛选 |

> 实际 sprite 大小 = scene 的 `sprite_scale_x/y` × stats 的 `sprite_scale` × variant_scale。scene 设 1.2，stats 设 1.0，最终就是 soldier 的 2 倍（soldier 默认 0.6）。

## 三、场景配置

`scenes/units/grand_tactic_releaser.tscn` 继承 `soldier.tscn`，关键覆盖：

```ini
[node name="GrandTacticReleaser" instance=ExtResource("base")]
stats_data = ExtResource("releaser_stats")   # 指向上面那个 stats
sprite_scale_x = 1.2                          # soldier 0.6 的 2 倍
sprite_scale_y = 1.2
sprite_offset_y = 18.0
team = 1                                      # ENEMY，必填
alliance_id = 1                               # 必填，否则身份错乱
faction_color = 3                             # RED，必填，否则颜色不对

[node name="GrandTacticReleaserAI" type="Node" parent="."]
script = ExtResource("releaser_ai")
# tactic 字段不在 scene 里设，由生成代码或测试面板注入
```

> **坑**：`team`/`alliance_id`/`faction_color` 三个必须都显式设，缺一个就会出现"颜色是蓝色但实际是敌方"这种错乱。详见 [memory: feedback-unit-scene-setup](../../../../C:/Users/86132/.claude/projects/e--godot-rts-godot-rts-start/memory/feedback_unit_scene_setup.md)。

## 四、战术 .tres 配置

`grand_tactic_resource.gd` 定义所有字段。4 种 tactic_id 用到的字段不同：

### 公共字段（所有战术都要）

| 字段 | 说明 |
|---|---|
| `tactic_name` | 显示名（光环头顶 Label 用） |
| `tactic_id` | 0=SUMMON_WAVE, 1=SUMMON_ELITE, 2=MORALE_BOOST, 3=FINAL_SUMMON |
| `channel_time` | 引导总时长（秒），通常 30 |

### 4.1 SUMMON_WAVE — 持续召唤

```ini
tactic_id = 0
effect_interval = 5.0    # 每 5s 召唤一波
effect_count = 3         # 每波 3 个
effect_unit_type = 0     # UnitType.SOLDIER
effect_stats_id = &""    # 空表示用 type 默认 stats
```

字段含义：引导期间每 `effect_interval` 秒，在释放者周围召唤 `effect_count` 个 `effect_unit_type` 类型的单位。

### 4.2 SUMMON_ELITE — 召唤精英 + 联动死亡

```ini
tactic_id = 1
elite_count = 1
elite_stats_ids = [&"boss_soldier"]   # P1 精英 stats_id
linked_death = true                    # 杀释放者则精英同步死亡
```

字段含义：引导开始时立刻召唤 `elite_count` 个精英，每个用 `elite_stats_ids[i % size]`。`linked_death=true` 时杀释放者会触发 `_cleanup_linked_minions()` 把所有联动精英也杀掉。

### 4.3 MORALE_BOOST — 鼓舞士气 aura

```ini
tactic_id = 2
aura_type = "attack"     # 也支持 "defense" / "regen" / "range_bonus" / "shield"
aura_value = 0.3         # 0.3 = +30% 攻击力
aura_range = 300.0       # 300 像素内的同队友军
```

字段含义：引导开始时把 aura 参数注入 `owner_unit.aura_type/value/range`，复用 `unit.gd._aura_process()` 每 0.5s 扫描同队友军施加 buff。

> **坑**：aura 只作用于**同队**单位（释放者 team=1，所以只 buff 敌方）。测试时释放者周围必须有其他敌方单位才能看到效果。详见 [调试](#六调试方法)。

> **坑**：AI 必须用 `call_deferred` 调 `_apply_tactic_start()`，否则 aura_range 会被 `unit._setup_stats()` 重置回 0（child _ready 先于 parent _ready）。

### 4.4 FINAL_SUMMON — 结束时召唤

```ini
tactic_id = 3
final_summon_groups = Array[Dictionary]([
    {"type": 0, "count": 8},   # 8 个 soldier
    {"type": 1, "count": 4}    # 4 个 archer
])
```

字段含义：引导完成（`channel_time` 秒后）瞬间刷出所有 `final_summon_groups` 里的单位。

## 五、生成方式

### 方式 1：通过 game_spawner（正式生成）

```gdscript
var spawner := get_tree().current_scene.get("spawner_module")
var tactic_preload := preload("res://resources/tactics/tactic_summon_wave.tres")
var releaser := spawner.spawn_grand_tactic_releaser(
    Vector2(300, 0),        # 生成位置
    tactic_preload          # 战术 Resource（可选，不传则不引导）
)
```

`spawn_grand_tactic_releaser()`（[game_spawner.gd:512](../../scripts/systems/game_spawner.gd)）会：
1. 用 `create_unit(SOLDIER, ENEMY, pos, &"grand_tactic_releaser")` 创建单位
2. 注册到 LockstepSync
3. 加到 `EnemyUnits` 节点
4. 播放 spawn 特效
5. 创建 AI 子节点并注入 tactic

### 方式 2：直接实例化场景（测试面板用法）

```gdscript
const ReleaserScene := preload("res://scenes/units/grand_tactic_releaser.tscn")
var unit := ReleaserScene.instantiate()
# add_child 之前注入 tactic，让 AI._ready() 能读到
var ai := unit.get_node_or_null("GrandTacticReleaserAI")
if ai:
    ai.set("tactic", tactic_resource)
enemy_parent.add_child(unit)
unit.global_position = pos
# 兜底：再设一次 team/alliance/color
unit.team = 1
unit.alliance_id = 1
unit.faction_color = FactionClass.ColorId.RED
unit.add_to_group("enemy_units")
```

> **坑**：tactic 必须在 `add_child` **之前**注入。AI._ready 立刻读 tactic 字段，加完再设就晚了。

### 调试按钮面板

测试场景左上角的 `GTTestPanel` 提供 4 个按钮，每个按钮在屏幕右侧随机位置召唤对应战术的释放者：

| 按钮 | 战术 | 颜色 |
|---|---|---|
| 1. 持续召唤 | SUMMON_WAVE | 紫 |
| 2. 召唤精英 | SUMMON_ELITE | 红 |
| 3. 鼓舞士气 | MORALE_BOOST | 金 |
| 4. 结束召唤 | FINAL_SUMMON | 青 |

> "鼓舞士气"按钮会**额外生成 3 个敌方士兵**作为 aura 目标，否则释放者孤零零一个，看不到 buff 效果。

### 控制台日志关键点

| 输出 | 含义 |
|---|---|
| `[GrandTactic] xxx 引导开始 (tactic=..., channel_time=30.0s)` | AI._ready 跑完 |
| `[GrandTactic] MORALE_BOOST 激活 aura (...)` | aura 参数注入完成 |
| `[GrandTactic] tick progress=17%` | 引导进度（每 effect_interval 一次） |
| `[GrandTactic] SUMMON_WAVE 召唤 3 个单位` | 周期性召唤生效 |
| `[GrandTactic] xxx aura=attack buff N 友军: ...` | aura 实际生效（_aura_process 触发） |
| `[GrandTactic] 引导完成，准备释放战术` | channel_completed 信号 |
| `[GrandTactic] FINAL_SUMMON 召唤 12 个单位` | FINAL_SUMMON 完成时召唤 |
| `[GrandTactic] 引导被中断（被击杀）` | 中途击杀触发 _interrupt |
| `[GrandTactic] 联动死亡清理 N 个单位` | linked_death 触发 |

### 常见问题排查

**aura 不生效（伤害数字没变）**
- 看控制台有没有 `aura=attack buff N 友军` 输出
- 没有 → `aura_range` 被重置回 0，检查 `_apply_tactic_start` 是不是用 `call_deferred`
- 有但 N=0 → 释放者周围没有同队单位，MORALE_BOOST 只 buff 同队（敌方）

**释放者颜色不对（变蓝）**
- 检查 scene 里 `team`/`alliance_id`/`faction_color` 三个字段是不是都设全

**释放者会追玩家跑**
- stats 的 `move_speed` 必须 = 0
- AI._ready 会调 `hold_position.call_deferred()`

**释放者反击**
- stats 的 `attack_damage`/`attack_range`/`attack_cooldown` 都设 0
- AI._ready 会调 `set_attack_disabled(true)`

**视觉光环不显示**
- 检查 ChannelVisual 是不是加在 owner_unit 下（不是 current_scene）
- `_create_channel_visual` 用 `_owner_unit.add_child.call_deferred(visual)`

**生成时报 `Parent node is busy setting up children`**
- AI._ready 期间 owner_unit 还在 setup，不能直接 add_child，必须 `call_deferred`

## 七、新增战术步骤

1. 在 `grand_tactic_resource.gd` 的 `TacticId` 枚举加新 ID
2. 在 `_apply_tactic_start` / `_apply_tactic_tick` / `_apply_tactic_complete` 的 match 分发里加新 case
3. 写 `_do_xxx()` 方法（参考现有 `_do_summon_wave` 等）
4. 在 `resources/tactics/` 新建 .tres 文件，配字段
5. 测试面板 `ButtonDefs` 加新按钮（如果想分按钮测）
