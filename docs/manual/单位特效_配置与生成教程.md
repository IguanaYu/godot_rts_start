> 35 个效果按 WC3/SC2 案例归纳为 7 大类。本文档讲怎么挑、怎么调、怎么生成。

## 二、A 类：标识型（头顶图标）

复用 `objective_marker.gd`，4 种图标类型。

| 编号 | 效果 | MarkerType | 推荐颜色 | 适用场景 |
|---|---|---|---|---|
| A1 | boss 紫色王冠 | `CROWN` | 紫 `(0.60, 0.20, 0.90)` | Boss 单位辨识 |
| A2 | elite 蓝色钻石 | `DIAMOND` | 蓝 `(0.40, 0.70, 1.00)` | 精英 / 小头目 |
| A3 | enraged 红色 Skull | `SKULL` | 红 `(0.90, 0.15, 0.15)` | 狂暴状态 |
| A4 | blessed 金色 Star | `STAR` | 金 `(1.00, 0.84, 0.25)` | 神圣 buff / 任务目标 |

### 调用方式

```gdscript
const ObjectiveMarker := preload("res://scripts/effects/objective_marker.gd")

func _spawn_marker_for(unit: Node2D) -> void:
    var marker := ObjectiveMarker.new()
    marker.marker_type = ObjectiveMarker.MarkerType.CROWN
    marker.icon_size = 22.0
    marker.height_offset = -38.0  # 负数 = 向上偏移
    marker.modulate = Color(0.60, 0.20, 0.90)  # 颜色 override
    add_child(marker)
    marker.setup(unit)  # 跟随 unit 移动
```

### 参数

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `marker_type` | `MarkerType` 枚举 | CROWN | 见上表 |
| `icon_size` | float | 24.0 | 图标尺寸 |
| `height_offset` | float | -48.0 | 距目标上方高度（负=向上） |
| `pulse_speed` | float | 4.0 | 呼吸灯频率 |
| `float_amplitude` | float | 5.0 | 上下浮动幅度 |
| `marker_level` | `MarkerLevel` 枚举 | NORMAL | IMPORTANT/CRITICAL 加装饰 |

### 行为

- 自动跟随 `setup(target)` 设置的目标，目标失效自动 `queue_free`
- `z_index = 100`，永远在最上层
- `set_danger(true)` 切换红色闪烁（保护目标濒危用）

## 三、B 类：地面贴花（脚下光圈）

复用 `outpost_status_marker.gd`，9 种 effect_id。

| 编号 | effect_id | 颜色 | 半径 | 视觉 |
|---|---|---|---|---|
| B1 | `inspire` | 金 | 50 | 双层光圈（外淡内深） |
| B2 | `heal` | 绿 | 60 | 圆环 + 淡填充 |
| B3 | `shield` | 蓝 | 38 | 六边形护盾 |
| B4 | `attack` | 红 | 0（外部注入） | 描边（用 territory_radius） |
| B5 | `burn` | 橙 | 42 | 默认通用贴花 |
| B6 | `poison` | 紫 | 42 | 默认通用贴花 |
| B7 | `rage` | 红 | 45 | 默认通用贴花 |
| B8 | `slow` | 蓝 | 45 | 默认通用贴花 |
| B9 | `telegraph_aoe` | 红 | 50 | 默认通用贴花（技能预警用） |

> B5-B9 当前用默认绘制分支（外圈描边 + 内圈 + 淡填充）。Phase 2 会给每个加专属形状（如 burn 加火焰纹、poison 加毒气泡）。

### 调用方式

```gdscript
const StatusMarker := preload("res://scripts/outpost/outpost_status_marker.gd")

# 方式 1：跟随目标单位
func _spawn_shield_on(unit: Node2D) -> void:
    var marker := StatusMarker.new()
    marker.effect_id = &"shield"
    marker.target = unit              # 跟随 unit
    marker.duration_sec = 12.0        # buff 持续时间
    add_child(marker)

# 方式 2：固定地面位置
func _spawn_heal_at(pos: Vector2) -> void:
    var marker := StatusMarker.new()
    marker.effect_id = &"heal"
    marker.center = pos               # 固定位置
    marker.duration_sec = 3.0
    add_child(marker)
```

### 参数

| 参数 | 类型 | 说明 |
|---|---|---|
| `effect_id` | StringName | 见上表 |
| `target` | Node | 可空。非空则每帧跟随 `target.global_position` |
| `center` | Vector2 | target 为空时用此坐标 |
| `duration_sec` | float | 持续时间。0 = 持续到外部 free |
| `set_visual_radius(r)` | 方法 | attack 状态由外部注入半径 |

### 行为

- target 失效或死亡 → 自动 `queue_free`
- 最后 25% 时间进入 2Hz 闪烁警示（buff 即将结束）
- `z_index = 40`，在单位 sprite 之上

### ⚠️ 已知坑

- `_process` 每帧强制 `global_position = center`（target 为空时）。**不要把 marker 作为子节点依赖父节点位置**——必须显式设 `center = 世界坐标`。
- effect_id 拼错不再静默无效，会走默认绘制分支（外圈描边）。

## 四、C 类：身体材质（shader）

直接操作 unit 的 `_effects_material`（ShaderMaterial）。4 组 uniform。

| 编号 | uniform 开关 | 视觉 | 适用场景 |
|---|---|---|---|
| C1 | `hit_flash_enabled` | 白色覆盖（85% 强度） | 受击反馈 |
| C2a | `boss_glow_enabled` | 紫色外发光 + 整体提亮 | Boss 辨识（shader fake 版） |
| C2b | （未做） | WorldEnvironment Glow 全局发光 | Boss 辨识（b 版，待选型） |
| C3 | `enraged_enabled` | 红色 tint 45% | 狂暴状态 |
| C4 | `blessed_enabled` | 金色 tint 40% | 神圣附体 |
| C5 | — | 死亡爆裂（染色爆炸贴图） | 单位死亡 |

### 调用方式

```gdscript
# C1 受击白闪（脉冲式，自动淡出）
func _trigger_hit_flash() -> void:
    _effects_material.set_shader_parameter("hit_flash_enabled", true)
    var tw := create_tween()
    tw.tween_property(_effects_material, "shader_parameter/hit_flash_amount",
        0.0, 0.12)
    tw.tween_callback(func():
        _effects_material.set_shader_parameter("hit_flash_enabled", false))

# C2a Boss 发光（持续开启）
func _enable_boss_glow() -> void:
    _effects_material.set_shader_parameter("boss_glow_enabled", true)
    _effects_material.set_shader_parameter("boss_glow_color",
        Color(0.60, 0.20, 0.90))
    _effects_material.set_shader_parameter("boss_pulse_speed", 3.0)
    _effects_material.set_shader_parameter("glow_width", 4.0)

# C3 狂暴红
_effects_material.set_shader_parameter("enraged_enabled", true)

# C4 神圣金
_effects_material.set_shader_parameter("blessed_enabled", true)

# C5 死亡爆裂
func _spawn_death_burst() -> void:
    var boom := preload("res://scenes/effects/explosion.tscn").instantiate()
    boom.global_position = global_position
    boom.modulate = Color(0.8, 0.2, 0.2)  # 敌方红 / 我方蓝 / Boss 紫
    get_tree().current_scene.add_child(boom)
```

### uniform 一览

| uniform | 类型 | 默认 | 说明 |
|---|---|---|---|
| `hit_flash_enabled` | bool | false | 开关 |
| `hit_flash_color` | vec4 | 白 | 闪的颜色 |
| `hit_flash_amount` | float [0,1] | 0.85 | 覆盖强度（用 tween 淡出） |
| `boss_glow_enabled` | bool | false | 开关 |
| `boss_glow_color` | vec4 | 紫 `(0.6, 0.2, 0.9)` | 外发光颜色 |
| `boss_pulse_speed` | float | 3.0 | 呼吸速度 |
| `enraged_enabled` | bool | false | 开关（红 tint 45%，固定） |
| `enraged_tint` | vec4 | 红 `(1.0, 0.15, 0.0)` | tint 颜色 |
| `blessed_enabled` | bool | false | 开关（金 tint 40%，固定） |
| `blessed_tint` | vec4 | 金 `(1.0, 0.84, 0.25)` | tint 颜色 |

### fragment 顺序（严格遵守）

```
slow → tint → hit_flash → enraged → blessed → boss_glow → outline
```

后开的会覆盖先开的。例如同时开 `hit_flash` 和 `enraged`，白闪会盖住红 tint（受击瞬间正确行为）。

### hook 集成点（Phase 3 待做）

| hook | 位置 | 触发 |
|---|---|---|
| hit_flash | `unit.gd:take_damage()` line 1162 后 | `final_amount > 0` 时 |
| boss_glow | `unit.gd._init_outline_material()` | `stats_data.category == "boss"` 时 |
| enraged | buff 系统（待建） | 狂暴 buff 应用 / 移除 |
| blessed | buff 系统（待建） | 神圣 buff 应用 / 移除 |
| 死亡爆裂 | `unit.gd.die()` line 1259 前 | 死亡瞬间 |

## 六、E 类：连接光束

`beam_effect.gd`，4 种 effect_id。

| 编号 | effect_id | 颜色 | 风格 | 适用场景 |
|---|---|---|---|---|
| E1 | `heal_beam` | 绿 | 持续光束 + 中点脉冲 | Medivac Heal（持续治疗） |
| E2 | `chain_lightning` | 蓝白 | 锯齿折线 | WC3 Chain Lightning |
| E3 | `siphon` | 红 | 双螺旋 + 中心流 | 吸血 |
| E4 | `life_leech` | 红 | 直线 + 流动光点 | 生命汲取 |

### 调用方式

```gdscript
const BeamEffect := preload("res://scripts/effects/beam_effect.gd")

func _spawn_heal_beam(healer: Node2D, target: Node2D) -> void:
    var beam := BeamEffect.new()
    beam.effect_id = &"heal_beam"
    beam.source_pos = healer.global_position
    beam.target_pos = target.global_position
    beam.duration_sec = 2.0        # 0 = 持续到外部 free
    get_tree().current_scene.add_child(beam)
```

### 参数

| 参数 | 类型 | 说明 |
|---|---|---|
| `effect_id` | StringName | 见上表 |
| `source_pos` / `target_pos` | Vector2 | **全局坐标**（_draw 里会减 global_position 转局部） |
| `duration_sec` | float | 0 = 持续到外部 free |

### ⚠️ 坐标坑

- `source_pos` / `target_pos` 必须是**全局坐标**，不是相对于 beam 节点的局部坐标
- 如果 beam 是某 slot 的子节点，传 `left.global_position` 而不是 `left.position`
- 如果 source/target 是移动的单位，需要 `_process` 里每帧更新这两个值（Phase 5 会加 `source_node` / `target_node` 自动跟随）

## 七、F 类：天降事件

`sky_event_effect.gd`，4 种 effect_id。两阶段：telegraph（1.2s 收缩环预警）→ impact（光柱从天而降 + 地面爆裂）。

| 编号 | effect_id | 颜色 | 风格 | 适用场景 |
|---|---|---|---|---|
| F1 | `meteor` | 红橙 | 爆裂 | 陨石术 |
| F2 | `divine_strike` | 金白 | 强光柱 | 神圣审判（WC3 Holy Light） |
| F3 | `arcane_meteor` | 紫 | 奥术 | 奥术陨石 |
| F4 | `resurrection` | 金 | 多光柱齐发 | 复活（WC3 Resurrection） |

### 调用方式

```gdscript
const SkyEvent := preload("res://scripts/effects/sky_event_effect.gd")

func _cast_meteor_at(pos: Vector2) -> void:
    var sky := SkyEvent.new()
    sky.effect_id = &"meteor"
    sky.center_pos = pos           # 落点
    get_tree().current_scene.add_child(sky)
    # 自动两阶段后 queue_free（总时长 ~2.7s）
```

### 参数

| 参数 | 类型 | 说明 |
|---|---|---|
| `effect_id` | StringName | 见上表 |
| `center_pos` | Vector2 | 落点（全局坐标） |

### 行为

- **telegraph 阶段**（0 ~ 1.2s）：红色收缩环从大到小 + 中心十字
- **impact 阶段**（1.2s ~ 2.7s）：光柱底端固定在落点，顶端从高处下降 + 地面强光 + 圆形扩散
- resurrection 额外：4 个小光柱围绕主光柱
- `z_index = 30`

### ⚠️ 注意

- 总时长 ~2.7s，自动 `queue_free`
- 想循环触发（如预览场景），外部用 Timer 重新 spawn
