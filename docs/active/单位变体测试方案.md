# 单位变体测试方案

## 测试关卡选择

使用 **Map 1**（`resources/map1_config.tres`），原因是：
- 地图简单、狭长，单位容易碰面交战
- 已有玩家单位和敌方单位，改配置即可

---

## 测试方法

1. 修改 `map1_config.tres` 的 `enemy_units`，用变体替换原有的敌方单位
2. 运行游戏：
   ```
   "E:\godot\Godot_v4.6.1-stable_win64.exe" --path "F:\godot_game\rts_base\rts-base"
   ```
3. 选择 Map 1（Blitz），观察以下指标

---

## 每单位测试检查项

### ① 盾卫 Shieldbearer
**敌方配置**（替换 enemy_units 前 2 个 soldier）：
```json
{"type": 0, "stats_id": "shieldbearer", "pos": Vector2(460, 170)},
{"type": 0, "stats_id": "shieldbearer", "pos": Vector2(540, 230)},
```

**观察点**：
- [ ] 钢灰色调，明显比普通战士大（scale 1.15）
- [ ] 移动速度极慢（70），感觉像在爬
- [ ] 玩家单位打它伤害很低（减伤 0.55，每次只受 ~45% 伤害）
- [ ] 攻击力低（6），打玩家单位很慢

### ② 狂战士 Berserker
**敌方配置**（替换 enemy_units 中 2 个 soldier）：
```json
{"type": 0, "stats_id": "berserker", "pos": Vector2(960, 150)},
{"type": 0, "stats_id": "berserker", "pos": Vector2(1040, 250)},
```

**观察点**：
- [ ] 暗红色调，体型偏小（scale 0.9）
- [ ] 移动极快（165），比弓箭手还快
- [ ] 攻速极快（0.35s），攻击动画不停
- [ ] 血量极低（65），很快被杀死

### ③ 弩手 Crossbowman
**敌方配置**（替换 enemy_units 中 1 个 archer）：
```json
{"type": 1, "stats_id": "crossbowman", "pos": Vector2(980, 220)},
```

**观察点**：
- [ ] 深棕色色调
- [ ] 每次攻击间隔长（1.8s），但伤害数字飘出 22（远高于普通弓箭手 15）
- [ ] 如果击中盾卫应看到高额伤害

### ④ 火焰法师 Pyromancer
**敌方配置**（替换 enemy_units 中 1 个 archer）：
```json
{"type": 1, "stats_id": "pyromancer", "pos": Vector2(1500, 220)},
```

**观察点**：
- [ ] 烈焰红色调
- [ ] 发射箭矢命中后，目标周围 70 范围内其他单位也掉血（溅射）
- [ ] 多个玩家单位站在一起时，所有单位都受伤
- [ ] 溅射伤害 = 14 × 0.6 = 8（飘字应有 14 和 8 两种）

### ⑤ 冰霜法师 Cryomancer
**敌方配置**（替换 enemy_units 中 1 个 archer）：
```json
{"type": 1, "stats_id": "cryomancer", "pos": Vector2(1560, 240)},
```

**观察点**：
- [ ] 冰蓝色调
- [ ] 被击中的玩家单位身上出现蓝色减速效果（shader 变色）
- [ ] 被击中的单位移动明显变慢（slow 40%），持续 2.5 秒
- [ ] 伤害低（8），但减速效果显著

## 完整测试配置

替换 `resources/map1_config.tres` 中 `enemy_units` 数组为以下内容：

```json
enemy_units = Array[Dictionary]( [
	# ① 盾卫 ×2（替换原第 1-2 个 soldier）
	{"type": 0, "stats_id": "shieldbearer", "pos": Vector2(460, 170)},
	{"type": 0, "stats_id": "shieldbearer", "pos": Vector2(540, 230)},
	# 原第 3-5 个 soldier 保留（对照用）
	{"type": 0, "pos": Vector2(960, 150)},
	{"type": 0, "pos": Vector2(1040, 250)},
	{"type": 0, "pos": Vector2(1020, 200)},
	# ② 狂战士 ×2（替换原 archer）
	{"type": 0, "stats_id": "berserker", "pos": Vector2(1510, 150)},
	{"type": 0, "stats_id": "berserker", "pos": Vector2(1530, 250)},
	# ③ 弩手（替换原 archer）
	{"type": 1, "stats_id": "crossbowman", "pos": Vector2(980, 220)},
	# ④ 火焰法师
	{"type": 1, "stats_id": "pyromancer", "pos": Vector2(1500, 220)},
	# ⑤ 冰霜法师
	{"type": 1, "stats_id": "cryomancer", "pos": Vector2(1560, 240)},
] )
```

> 保留 3 个普通 soldier 作为对照，方便肉眼对比变体 vs 原版的行为差异。

---

## 测试流程

1. 修改 `map1_config.tres`
2. 运行游戏
3. 进入 Map 1
4. 用玩家初始的 5 个单位向右侧敌人推进
5. 逐个观察每个变体单位的表现
6. 测试完改回原配置（或保留新配置用于后续关卡设计）

---

## 预期结果总结

| 单位 | 关键可验证行为 | 失败可能原因 |
|------|--------------|-------------|
| 盾卫 | 灰/慢/硬/低攻 | tint 没生效（检查 variant_tint ≠ Color.WHITE） |
| 狂战士 | 红/快/脆/速攻 | 0.35s 冷却是否生效（stat_set 继承是否正确） |
| 弩手 | 棕/慢/高伤 | 伤害 22 是否覆盖 archer 的 15 |
| 火焰法师 | 红/溅射 | projectile_data 是否接线（SplashEffect 是否创建正确） |
| 冰霜法师 | 蓝/减速 | SlowEffect 是否接线 + shader 减速变色 |

---

## 验收标准

通过 = 5 个变体都出现且各自特征可肉眼观察到。
部分通过 = 至少 tint 和数值类（盾卫/狂战士/弩手）正常工作，远程特效需调试。
失败 = 场景加载出错或单位不出现。
