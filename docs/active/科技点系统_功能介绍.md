# 科技点系统 — 功能介绍

## 一句话定位

科技点是**暗线累积资源**：通过游戏行为（造建筑、造兵、击杀、被动等）持续获得，达到阈值后解锁科技等级（Tier 2 / Tier 3）。**只增不消耗**，是后续科技树 / 三选一合并 / 指挥官差异化的底层支撑。

---

## 核心设计

| 属性 | 说明 |
|------|------|
| **资源性质** | 暗线、只增不消耗、全局共享（不分玩家池） |
| **解锁阈值** | Tier 2 = 1000 点，Tier 3 = 3000 点 |
| **触发方式** | 6 类游戏行为分别按基础点数 + 指挥官倍率结算 |
| **可视化** | 左上角小标签 `Tech: N`（仅显示点数，不打扰玩家） |

---

## 6 类来源 & 基础点数

| 来源分类 (`CATEGORY_*`) | 触发时机 | 基础点数 | 触发位置 |
|---|---|---|---|
| `build_construction` | 玩家建筑建造完成 | 100 ~ 300（按建筑类型） | `building_placer._on_building_construction_finished()` |
| `kill_enemy_building` | 摧毁敌方建筑 | 100 | `building_placer._on_building_died()` |
| `kill_enemy_unit` | 消灭敌方单位 | 1 | `main._on_unit_died()` |
| `own_unit_died` | 己方单位阵亡 | 2 | `main._on_unit_died()` |
| `produce_unit` | 生产一个单位（建筑自动产 + 面板点击造兵） | 5 | `building._spawn_produced_unit()` + `game_spawner.place_player_unit()` |
| `passive` | 每 60 秒被动 | 10 | `main._on_tech_passive_timer()` |

### 建筑类型 → 科技点表

| 建筑类型 | 点数 |
|---|---|
| 城堡 (CASTLE) | 300 |

| 修道院 (MONASTERY) | 200 |
| 靶场 (ARCHERY) | 200 |

## 信号链路

```
游戏行为
  ├─ building._finish_construction()
  │     └─ construction_finished 信号
  │           └─ building_placer._on_building_construction_finished()
  │                 └─ tech_point_manager.add_points(pts, BUILD_CONSTRUCTION)
  │
  ├─ building._spawn_produced_unit() / game_spawner.place_player_unit()
  │     └─ tech_point_manager.add_points(5, PRODUCE_UNIT)
  │
  ├─ main._on_unit_died(unit)
  │     └─ 按 unit.is_in_group("enemy_units" / "player_units")
  │         └─ tech_point_manager.add_points(+1 或 +2, ...)
  │
  ├─ building_placer._on_building_died(building)
  │     └─ team == ENEMY → tech_point_manager.add_points(100, KILL_ENEMY_BUILDING)
  │
  └─ Timer 60s timeout
        └─ main._on_tech_passive_timer()
              └─ tech_point_manager.add_points(10, PASSIVE)

tech_point_manager.add_points()
  ├─ 应用 category 倍率
  ├─ tech_points += final_amount
  ├─ emit points_changed(points) → UI 更新
  └─ _check_level_unlock()
       └─ 跨阈值时 emit level_unlocked(level)
```

## 指挥官差异化（预留）

`tech_point_manager.set_category_multiplier(category, multiplier)` 可针对特定来源设倍率。

举例：未来某指挥官是「击杀流」，可在战前配置时调用：
```gdscript
tech_point_manager.set_category_multiplier(TPD.CATEGORY_KILL_ENEMY_UNIT, 1.5)
tech_point_manager.set_category_multiplier(TPD.CATEGORY_KILL_ENEMY_BUILDING, 2.0)
```

倍率默认 1.0，不调用即等于无差异化。
