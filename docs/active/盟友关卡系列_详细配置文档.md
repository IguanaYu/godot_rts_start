## 前置：需要开发的新机制（4个）

### 机制1：AI 队友初始行为配置（P0）

**目的**：让部分 AI 队友开局就驻防特定位置，而非全部跟随玩家。

**改动 1/3 — [scripts/units/ally_ai.gd](../../scripts/units/ally_ai.gd)**

在第 22 行 `_force_target_pos` 声明后，新增方法：

```gdscript
## 外部调用：设置 AI 队友的初始行为（用于分兵驻防）
func set_initial_behavior(behavior: String, pos: Vector2 = Vector2.ZERO) -> void:
    match behavior:
        "defend":
            ally_state = AllyState.FORCE_DEFEND
            _force_target_pos = pos
            unit.move_to(pos)
        "attack_move":
            ally_state = AllyState.FORCE_ATTACK
            _force_target_pos = pos
            unit.attack_move_to(pos)
        "follow", _:
            pass  # 默认 FOLLOW_PLAYER，无需操作
```

**改动 2/3 — [scripts/systems/game_spawner.gd](../../scripts/systems/game_spawner.gd)**

找到 `spawn_ally_unit_initial()` 函数（约第 166 行），修改签名和函数体：

```gdscript
# 旧签名：
func spawn_ally_unit_initial(unit_type: int, pos: Vector2) -> void:

# 新签名：
func spawn_ally_unit_initial(unit_type: int, pos: Vector2, \
        behavior: String = "follow", defend_pos: Vector2 = Vector2.ZERO) -> void:
```

在函数末尾（生成 AI 节点之后）加：

```gdscript
    if behavior != "follow" and ai.has_method("set_initial_behavior"):
        ai.set_initial_behavior(behavior, defend_pos)
```

**改动 3/3 — [scripts/main.gd](../../scripts/main.gd) 第 686-687 行**

```gdscript
# 旧代码：
for spawn in map_config.ai_allies:
    spawner_module.spawn_ally_unit_initial(spawn.type, spawn.pos)

# 新代码：
for spawn in map_config.ai_allies:
    var behavior: String = spawn.get("behavior", "follow")
    var defend_pos: Vector2 = spawn.get("defend_pos", Vector2.ZERO)
    spawner_module.spawn_ally_unit_initial(spawn.type, spawn.pos, behavior, defend_pos)
```

### 机制2：占领点连续增援波次（P1）

**目的**：占领后不只来一波增援，而是每 N 秒来一波，共 repeat 波。

**改动 1/2 — [scripts/systems_game/capture_point.gd](../../scripts/systems_game/capture_point.gd) 第 42 行后**

```gdscript
@export var ally_reinforcement_repeat: int = 0        # 连续增援波数（0=仅一次，>0=额外重复 N 波）
@export var ally_reinforcement_interval: float = 30.0  # 每波间隔秒数
```

**改动 2/2 — [scripts/main.gd](../../scripts/main.gd) 的 `_on_capture_point_captured` 增援部分**

找到现有增援逻辑（约第 457-476 行），在"第一次延迟增援"之后，加上连续波次：

```gdscript
    # 现有的第一次延迟增援逻辑保持不变...
    
    # === 新增：连续增援波次 ===
    var repeat: int = point.ally_reinforcement_repeat
    if repeat > 0:
        var interval: float = point.ally_reinforcement_interval
        for i in range(repeat):
            var wave_delay: float = delay + (i + 1) * interval
            var timer = get_tree().create_timer(wave_delay)
            timer.timeout.connect(
                _on_ally_reinforcement_timeout.bind(
                    point.ally_reinforcement_groups,
                    point.global_position,
                    null  # target=null 表示用默认 attack_move 行为
                )
            )
```

> 注意：`_on_ally_reinforcement_timeout` 是现有的回调函数，bind 的参数顺序要与现有调用保持一致。实现时请对照现有代码确认 bind 参数。

**改动 3/4 — [scripts/systems_game/ally_commander.gd](../../scripts/systems_game/ally_commander.gd)**

新增分组指挥方法：

```gdscript
## 对指定小队下达攻击指令
func issue_squad_attack_order(squad_id: String, world_pos: Vector2) -> void:
    for u in get_tree().get_nodes_in_group("player_units"):
        if u.owner_id != -2:
            continue
        var ai = u.get_node_or_null("AllyAI")
        if ai != null and ai.squad_id == squad_id:
            ai.issue_attack_order(world_pos)

## 对指定小队下达防御指令
func issue_squad_defend_order(squad_id: String, world_pos: Vector2) -> void:
    for u in get_tree().get_nodes_in_group("player_units"):
        if u.owner_id != -2:
            continue
        var ai = u.get_node_or_null("AllyAI")
        if ai != null and ai.squad_id == squad_id:
            ai.issue_defend_order(world_pos)
```

> 注意：`issue_attack_order` / `issue_defend_order` 是 AllyAI 上已有的方法，确认方法名正确。

### WaveManager 配置（敌方持续压力）

敌方分 3 波交替进攻南北 AI 防线：

| 波次 | 延迟 | 来源位置 | 目标 | 兵力 |
|------|------|---------|------|------|
| Wave 0 | 60s | (1950, 150) 北基地 | AI北队防线 (900,150) | 3士兵 |
| Wave 1 | 90s | (1950, 450) 南基地 | AI南队防线 (900,450) | 3士兵 + 1弓兵 |
| Wave 2 | 120s | 双基地 | 南北防线同时 | 各 2士兵 |

具体 WaveManager 字段配置（对照项目现有波次格式）：

```
waves = [
    {
        "delay": 60.0,
        "spawns": [
            {"type": 0, "pos": Vector2(1900, 150), "count": 3},
        ],
        "attack_target": Vector2(900, 150),
    },
    {
        "delay": 30.0,  # 距上一波30秒（即全局90秒）
        "spawns": [
            {"type": 0, "pos": Vector2(1900, 450), "count": 3},
            {"type": 1, "pos": Vector2(1920, 450), "count": 1},
        ],
        "attack_target": Vector2(900, 450),
    },
    {
        "delay": 30.0,  # 距上一波30秒（即全局120秒）

            {"type": 0, "pos": Vector2(1900, 150), "count": 2},
            {"type": 0, "pos": Vector2(1900, 450), "count": 2},
        ],
        "attack_target": Vector2(900, 300),

]
```

> 注意：WaveManager 的具体字段名（如 `attack_target`）请对照项目现有波次配置格式调整。现有波次系统可能用 `clear_then_next` 等字段控制节奏。
