# 多人地图配置手册

> 配套单机配置见 [地图配置指南.md](地图配置指南.md)。本文档专讲多势力（alliances）新结构。
> 起点 commit: `8fe8587`（多势力系统三层解耦）

## 何时用新结构 vs 旧字段

`MapConfig` 支持两种配置方式，二选一：

| 方式 | 字段 | 何时用 |
|---|---|---|
| **旧字段**（fallback） | `player_units` / `player2_units` / `enemy_units` 等 | 单机地图、不需要多势力、不需要颜色/槽位控制 |
| **新结构**（alliances） | `alliances` 数组 | 多人地图、需要玩家选 slot、需要多色多联盟、需要按玩家数动态生成 |

判定逻辑（`game_spawner.spawn_from_config`）：
- `alliances` 数组为空 → 走旧字段 fallback
- `alliances` 数组非空 → 走新结构，忽略旧字段

**单机地图保持现状**：旧 16 张战役地图都走 fallback，无须改动。
**多人地图必须用新结构**：旧字段不支持多 slot、颜色、动态生成。

## 三层概念速查

配置前先理解三层独立概念（详见 [多人联机势力系统_后续任务.md](../archived/多人联机势力系统_后续任务.md)）：

| 层 | 字段 | 决定 | 配置在哪 |
|---|---|---|---|
| **颜色** | `faction_color` | 视觉贴图目录 | slot.color |
| **玩家** | `owner_id` | 控制权 + 金币归属 | 运行时按 player_id 分配（玩家各自） |
| **联盟** | `alliance_id` | 胜负共享 / 敌对关系 | alliance.id |

**颜色枚举**（`Faction.ColorId`）：

| 数值 | 名称 | 素材目录前缀 | 状态 |
|---|---|---|---|
| 0 | BLACK | `black_*` | 缺素材，fallback 蓝色 |
| 1 | BLUE | `blue_*` | 有 |
| 2 | PURPLE | `purple_*` | 缺素材，fallback 蓝色 |
| 3 | RED | `red_*` | 有（默认敌方色） |
| 4 | YELLOW | `yellow_*` | 缺素材，fallback 蓝色 |
| 5 | NEUTRAL | — | 中立（暂未使用） |

## alliances 结构详解

`MapConfig.alliances` 是 `Array[Dictionary]`，每个元素是一个 alliance。

### alliance 字段

| 字段 | 类型 | 必填 | 含义 |
|---|---|---|---|
| `id` | int | 是 | 联盟 ID。0=玩家方，1=敌方，2/3 预留 PvP |
| `is_ai` | bool | 是 | true=AI 控制全部 slot；false=玩家可加入 |
| `color` | int | 否 | 联盟默认颜色（slot 未指定时用）。玩家方建议不填，由 slot 决定 |
| `slots` | Array[Dictionary] | 是 | 该联盟下的所有槽位 |

### slot 字段

| 字段 | 类型 | 必填 | 含义 |
|---|---|---|---|
| `color` | int | 是 | 该槽位的颜色（`Faction.ColorId`） |
| `start_pos` | Vector2 | 否 | 起始坐标（仅文档/调试用，运行时不强制） |
| `initial_gold` | int | 否 | 该槽位玩家的初始金币（默认用 `map_config.initial_gold`） |
| `units` | Array[{type, pos}] | 否 | 该槽位的初始单位 |
| `buildings` | Array[{type, grid_pos}] | 否 | 该槽位的初始建筑 |

**单位/建筑 type 对照**（同单机）：
- 单位 type: 0=士兵, 1=弓箭手, 2=长矛兵, 3=僧侣
- 建筑 type: 0=墙, 1=塔, 2=城堡, 3=兵营, 4=修道院, 5=射箭场

```
                "initial_gold": 10000,
                "units": [
                    {"type": 0, "pos": Vector2(200, 500)},
                    {"type": 0, "pos": Vector2(200, 580)},
                    {"type": 0, "pos": Vector2(200, 660)},
                    {"type": 1, "pos": Vector2(150, 540)},
                    {"type": 1, "pos": Vector2(150, 620)}
                ],
                "buildings": [
                    {"type": 2, "grid_pos": Vector2i(1, 8)},  # 城堡
                    {"type": 3, "grid_pos": Vector2i(5, 9)}   # 兵营
                ]
            },
            {
                "color": 1,          # BLUE（玩家 2 默认占此槽）
                "start_pos": Vector2(200, 150),
                "initial_gold": 10000,
                "units": [
                    {"type": 0, "pos": Vector2(200, 150)},
                    {"type": 0, "pos": Vector2(200, 230)},
                    {"type": 0, "pos": Vector2(200, 310)},
                    {"type": 1, "pos": Vector2(150, 190)},
                    {"type": 1, "pos": Vector2(150, 270)}
                ],
                "buildings": [
                    {"type": 2, "grid_pos": Vector2i(1, 2)},
                    {"type": 3, "grid_pos": Vector2i(5, 3)}
                ]
            }
        ]
    },
    {
        "id": 1,                    # 敌方
        "is_ai": true,
        "color": 3,                 # RED

            {
                "color": 3,

                    # ...
                ],

                    # ...
                ]
            }
        ]
    }
]
```

## 运行时生成行为（动态生成）

游戏开始时（`main._spawn_dynamic_players`）按以下规则生成：

### 玩家方（alliance 0, is_ai=false）

按 `NetworkManager.player_sessions` 中实际占用的 slot 生成：
- 玩家 1 占 slot 0 → 生成 slot 0 的初始单位/建筑
- 玩家 2 占 slot 1 → 生成 slot 1 的初始单位/建筑
- 没 player 占的 slot → **保持空地，不生成**
- 两玩家选同槽（co-op）→ 该 slot 只生成一次，但两人共享控制权

**默认 slot 分配**（lobby UI 未实现前的 deterministic 分配）：
- player_id 1 → slot 0, color BLUE（主机）
- player_id 2 → slot 1, color YELLOW（客户机）
- player_id 3 → slot 2, color PURPLE（fallback 蓝色，素材未导入）
- player_id 4 → slot 3, color BLACK（fallback 蓝色，素材未导入）

颜色循环表见 `network_manager.SLOT_COLORS = [1, 4, 2, 0]`（避开 RED=3 留给敌方）。

### 敌方（is_ai=true）

`_spawn_ai_alliances_from_config` 全量生成所有 AI alliance 的所有 slot，不管玩家数。

## 配置工作流（Godot 编辑器）

### 方法 A：Inspector 编辑

1. 双击 `resources/your_map_config.tres` 打开
2. Inspector 中找到 `Alliances` 数组
3. 添加元素 → 设置 `id` / `is_ai` / 展开 `Slots` 添加槽位
4. 每个 slot 设置 `color` / `start_pos` / `initial_gold` / `units` / `buildings`

> **注意**：Inspector 编辑嵌套 `Array[Dictionary]` 不太友好。元素多了容易出错，推荐方法 B。

### 方法 B：直接编辑 .tres 文本

1. 用文本编辑器（或 Godot 编辑器的外部编辑器）打开 `.tres` 文件
2. 按 [完整示例](#完整示例2-玩家-co-op--1-敌方-ai) 格式手写
3. 保存后 Godot 编辑器会自动重新导入

参考 [resources/mptest_config.tres](../../resources/mptest_config.tres) 作为模板复制。

### 方法 C：代码生成

需要批量配置多张地图时，写一个 `@tool` 脚本生成 `.tres`。本项目暂无此类工具脚本。

## 常见配置场景

### 场景 1：单势力地图（不允许多人）

只用旧字段，不加 `alliances`：
```
player_units = [...]       # 玩家初始单位
player_buildings = [...]   # 玩家初始建筑
enemy_units = [...]        # 敌方初始单位
enemy_buildings = [...]    # 敌方初始建筑
```

走 fallback，所有玩家方都是 alliance 0 slot 0 BLUE。

### 场景 2：2 玩家 co-op（默认配置）

见上面 [完整示例](#完整示例2-玩家-co-op--1-敌方-ai)。alliance 0 有 2 个 slot，分别 BLUE/BLUE。两人默认各占一个槽（异槽独立控制）。

### 场景 3：4 玩家 co-op

alliance 0 配 4 个 slot，颜色分别 BLACK/BLUE/PURPLE/YELLOW：
```
alliances = [{
    "id": 0, "is_ai": false,
    "slots": [
        {"color": 0, "start_pos": Vector2(...), "units": [...], "buildings": [...]},
        {"color": 1, "start_pos": Vector2(...), "units": [...], "buildings": [...]},
        {"color": 2, "start_pos": Vector2(...), "units": [...], "buildings": [...]},
        {"color": 4, "start_pos": Vector2(...), "units": [...], "buildings": [...]}
    ]
}, {
    "id": 1, "is_ai": true, "color": 3,
    "slots": [...]  # 敌方配置，可以更强大
}]
```

> 当前 relay server 只支持 2 玩家，4 人需要 T3 实现后才能真正测试。

### 场景 4：同槽 co-op（两玩家共享控制）

两人都选 alliance 0 slot 0：单位颜色相同，控制权共享。

**当前限制**：lobby UI 未实现（T1），无法手动选同槽。deterministic 默认是异槽。

### 场景 5：PvP（玩家 vs 玩家）

```
alliances = [{
    "id": 0, "is_ai": false,        # 玩家方 A
    "slots": [{"color": 1, ...}]    # 蓝色阵营
}, {
    "id": 1, "is_ai": false,        # 玩家方 B（不是 AI！）
    "slots": [{"color": 3, ...}]    # 红色阵营
}]
```

**当前限制**：`is_ai=false` 的 alliance 1 不会被 `_spawn_ai_alliances_from_config` 生成，玩家方的动态生成也只看 alliance 0。PvP 需要 T7 实现。

### 场景 6：多 enemy 联盟（多个敌人阵营）

```
alliances = [{
    "id": 0, "is_ai": false, "slots": [...]   # 玩家方
}, {
    "id": 1, "is_ai": true, "color": 3,       # 红色敌方
    "slots": [...]
}, {
    "id": 2, "is_ai": true, "color": 0,       # 黑色敌方（另一阵营）
    "slots": [...]
}]
```

两个敌方 alliance 都会被全量生成。颜色不同视觉区分。

> 注意：素材缺失时，黑色 fallback 蓝色，可能跟玩家方混淆。补素材后才能清晰区分。

## 检查清单（配置多人地图时）

- [ ] `alliances` 数组非空（否则走 fallback）
- [ ] 至少有一个 `is_ai=false` 的 alliance（id=0 玩家方）
- [ ] 至少有一个 `is_ai=true` 的 alliance（敌方）
- [ ] 每个 slot 有 `color` 字段
- [ ] 每个 slot 的 `units` / `buildings` 坐标在 `map_bounds` 范围内
- [ ] 建筑 `grid_pos` 不重叠（每个建筑占 1x1 或更大，参见 `D.get_building_grid_size`）
- [ ] `initial_gold` 合理（玩家方 5000-10000，敌方可以更大）
- [ ] `available_items` 包含玩家需要造的 PlaceMode ID
- [ ] `camera_start` 对准玩家方主力位置
- [ ] `nav_bounds` 覆盖地图可走区域
