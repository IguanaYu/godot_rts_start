# mini/captain — 核心玩法需求文档

> **分支**: `mini/captain`(从 master 拉)
> **定位**: 单人 RPG · survivors-like 割草 · 带队召唤 · 据点攻防
> **状态**: 玩法设计阶段,不涉及代码实现

---

## 1. 一句话定位

> **Dota 式鼠标走位的英雄操作 + 吸血鬼幸存者式的自动攻击成长 + 召唤兵组合技能 + RTS 式据点胜负条件。**

玩家主要时间在**走位**,单位自动攻击;成长来自击杀换金币 → 三选一升级;技能以**召唤兵组合**为核心特色。

---

## 2. 操作模型

| 项 | 设计 |
|---|---|
| 移动 | 鼠标点击移动(Dota 式,非 WASD) |
| 攻击 | 默认自动攻击(最近/锁定目标) |
| 技能 | 默认自动释放,玩家可关闭自动 → 手动释放;关键技能保留主动释放 |
| 操作密度 | 低频决策 + 高频走位,贴近 survivors 体感 |

**核心原则**:玩家的主要输入是"位置"。技能绝大多数交给自动,玩家只在关键时机手动介入(爆发、位移、功能技能)。

---

## 3. 技能系统

### 3.1 技能分类

| 类别 | 释放方式 | 例子 |
|---|---|---|
| **召唤兵技能**(核心特色) | 自动 / 手动 | 召唤3嘲讽兵、召唤3法师火球、天降5火焰近战兵 |
| **被动/触发型自动技能** | 全自动 | 每5秒释放三连射击,命中召唤1小兵 |
| **队长主动技能**(1 个手动槽) | Q + 左键 | 基础效果 + 技能效果组合(见 3.4) |

### 3.4 队长主动技能的组合设计

主动技能不是单一效果,而是**基础效果 + 技能效果**的组合:

- **基础效果**:因指挥官而异,定义队长的"功能定位"。例:
  - 全场狂暴(友军攻速爆发)
  - 命令友方单位进攻某点(类 V 键指挥的主动版)
  - 释放治疗圈(区域治疗)
- **技能效果**:叠加在基础效果上的具体召唤/攻击动作。例:
  - 在指定点位召唤射手群
  - 在指定点位召唤法师群进攻
  - 瞬发伴随召唤(小骷髅式)

**组合示例**:基础"全场狂暴" + 技能"指定点召唤3法师" = 释放瞬间在指定位置召唤3名狂暴化的法师,持续时间内全场友军攻速提升。

> 设计意图:基础效果提供"指挥官个性",技能效果提供"召唤爽感",两者组合让同一个手动槽在不同指挥官手里体验完全不同。

### 3.2 召唤兵技能的设计模式

召唤兵技能是本作最大特色。参考 Dota2 小骷髅(克林克兹)破隐一击瞬间召唤4骷髅兵连发的"瞬间爆兵"质感。设计模板:

- **类型 A · 阵地召唤**:选位置召唤固定站位单位(3法师火球 / 3嘲讽墙)
- **类型 B · 天降召唤**:从天而降,落地伤害 + 持续进攻(5火焰近战兵)
- **类型 C · 瞬发伴随**:破隐/触发瞬间召唤伴随单位连射(小骷髅式)
- **类型 D · 持续驻军**:长存在的小兵班,自动战斗

每个召唤兵有**生命周期**(时限/受击死亡/一次性),避免场景无限膨胀。

### 3.3 自动 vs 手动的设计

- **默认自动**:绝大多数技能自动释放,贴合 survivors 割草感
- **可关自动**:玩家可关闭某技能的自动释放,改为手动(关键技能常用此)
- **始终手动**:少量功能技能(爆发开关、位移冲锋)始终手动,给玩家操作空间

> 设计意图:默认玩家可以"只走位"就玩起来(survivors 体验);想深入操作的玩家可以关自动接管关键技能(Dota 体验)。

---

## 4. 成长系统

| 维度 | 设计 |
|---|---|
| 击杀掉落 | 金币(单一资源,取消经验) |
| 金币来源 | 击杀敌人、消灭建筑、拔除营地 |
| 升级方式 | 三选一(沿用现有 RTS 的三选一框架) |
| 升级内容 | 召唤兵强化、自动技能强化、被动加成、新技能解锁、人口上限提升 |

**成长主轴**:强化"自动部分"(召唤兵 + 自动技能),让玩家的走位收益随时间放大。e.g. 三连射击命中召唤小兵 → 升级后变五连射击 + 召唤2小兵。

---

## 5. 胜负条件

沿用现有 RTS 的据点/基地框架,具体关卡可配置:

| 模式 | 胜利 | 失败 |
|---|---|---|
| 进攻关 | 攻下 N 个据点 / 推平敌方基地 | 队长死亡 / 时限内未达成 |
| 防守关 | 守住基地 N 波/时限 | 基地被毁 / 队长死亡 |
| 混合关 | 攻防动态切换 | 同上 |

**队长死亡**是通用失败条件之一(类似 Dota 英雄阵亡,但本作单局失败而非复活)。

---

## 6. 战斗节奏感参考

- **走位主导**:大部分时间玩家在移动鼠标调整队长位置,自动攻击 + 自动召唤在持续输出
- **瞬时爆兵**:关键技能/触发点瞬间召唤一群小兵,形成"唰一下满屏自己人"的爽感(小骷髅破隐连发质感)
- **成长可见**:每次三选一后,屏幕上的自动火力/召唤数量明显变强
- **据点推进**:有明确战术目标,不是纯survivors的"活到时间结束"

## 7. 已定设计(本轮补充)

### 7.1 召唤兵 AI 与指挥

- **默认行为**:召唤兵**不可指挥**,自动在队长周围攻击最近敌人
- **V 键指挥**:默认**没有**,后续通过"指挥官"系统实现 —— 不同指挥官有不同专属技能,部分指挥官的专属兵种可被指挥
- 即:V 键指挥是某些指挥官的专属能力,不是通用机制

### 7.2 召唤兵上限与生命周期

- **人口系统**:有全局人口上限,满人口无法召唤
- **初始人口**:8
- **人口成长**:人口上限通过三选一升级 + 大增幅节点提升
- **生命周期**:召唤兵有持续时间,到时自动死亡(配合人口循环)
- **CD 限制**:每个召唤技能有独立 CD,防止瞬间铺满
- 设计意图:人口上限 + 持续死亡 + CD,形成"持续召唤、动态更替"的循环,而非一次性铺满

### 7.3 资源系统

- **单一资源**:只保留**金币**(取消经验)
- 金币来源:击杀敌人、消灭建筑、拔除营地
- 金币用途:三选一升级(含人口上限、召唤强化、被动加成、新技能解锁)

### 7.4 队长技能槽位

- **技能来源**:**通过三选一升级获得**(不是预设槽位解锁)
- **总技能数**:约 6 个技能槽,逐次通过三选一填满
- **手动槽位**:其中**最多 1 个**可设为手动释放,其余自动
- **可全自动**:玩家也可以把那 1 个手动槽设为自动,即全技能自动
- **手动释放操作**:Q 键 + 鼠标左键(指向/选位释放)

### 7.5 关卡时长与死亡

- **单局时长**:5–10 分钟
- **队长死亡**:直接失败,无复活/无命数

### 7.6 指挥官系统(后续扩展)

- **指挥官差异**:不同指挥官有不同**专属技能**(召唤兵种/被动/主动)
- **V 键指挥**:作为部分指挥官的专属能力出现,非通用机制
- **定位**:指挥官是本作的重玩性/差异化核心,类似 RPG 的职业/角色选择
- 注:指挥官具体设计留待后续,本轮先定通用玩法骨架

### 7.7 三选一升级池

**升级池类别**(玩家每次升级从池中三选一):

| 类别 | 例子 |
|---|---|
| 新召唤兵技能 | 解锁"3法师火球"、"天降5火焰兵"等,填满技能槽 |
| 召唤兵强化 | 火焰兵攻击力 +20%、召唤兵持续时间 +3秒 |
| 人口上限提升 | 人口 +2、+5 |
| CD 强化 | 召唤技能 CD -15% |
| 被动触发型自动技能 | 每5秒三连射击命中召唤1小兵 |
| 队长被动 | 移速 +10%、攻击力 +15%、生命 +30 |
| 金币/经济强化 | 击杀金币 +20%、拔营地额外金币 |
| 队长主动功能技能 | 解锁"爆发状态"、"冲锋位移",占用那 1 个手动槽 |

**规则**:
- **稀有度**:升级分稀有度(普通/稀有/史诗,具体数值待标定)
- **可多次升级**:同一升级可叠加多次(如火焰兵 +20% 攻击力可叠)
- **手动槽规则**:6 槽里最多 1 个可手动;该槽可空(全自动),也可通过三选一解锁一个功能技能填入

### 7.8 大增幅节点(特殊升级)

**机制**:除常规三选一外,在特定节点触发**大增幅**(人口大幅提升 / 某单位质变强化)。

**触发方式**(二选一,待定):
- 方案 A · 事件驱动:击破敌方城寨 / 击杀精英敌人时触发一次大增幅
- 方案 B · 周期驱动:每 5 或 7 次常规升级,触发一次大增幅

**待定**:触发方式后续通过试玩标定,本轮先确认机制存在。

### 7.9 敌人设计

- **杂兵**:数量多、单体弱,构成 survivors 式割草体验的基础
- **精英**(大个头):单体强,威胁高
- **关系**:精英可能是大增幅节点的事件源之一(方案 A 待定)
- Boss / 更细分敌人类型留待后续设计

### 7.10 难度调节

- 需要,具体分级待定(如简单/普通/困难,或数值缩放)
- demo 阶段先定一个默认难度试玩

### 7.11 据点与 UI

- **据点**:沿用现有 RTS 的占领点机制,本作不做改动
- **UI/HUD**:待原型阶段设计,本轮不定

### 7.12 Demo 范围(先验证技术跑通,再验证玩法好玩)

**验证顺序**:技术跑通 → 玩法好玩 → 深度设计

| 维度 | Demo 内容 |
|---|---|
| 指挥官 | 1 个,基础效果 = **大范围狂暴**(友军攻速爆发) |
| 地图 | 1 张小图,5–10 分钟单局 |
| 队长移动 | 鼠标点击移动(核心) |
| 自动攻击 | 有(核心) |
| 召唤兵技能 | 2–3 个,验证"瞬间爆兵"手感 |
| 被动触发技能 | 1–2 个,验证自动部分成长 |
| 队长主动技能 | 1 个:基础"大范围狂暴" + 技能效果(召唤兵组合) |
| 三选一升级 | 有,约 10–15 个升级项 |
| 大增幅节点 | demo 阶段先不做,或用周期驱动最简版 |
| 人口系统 | 有,初始 8 |
| 敌人 | 杂兵 + 1 种精英 |
| 胜负条件 | 简单一关:守住基地或攻下 1 个据点 |
| 指挥官系统 / V 键 | 砍掉,后续 |
| 多难度 | 1 个默认难度 |
| 多关卡 | 砍掉,1 关验证 |

**Demo 第一阶段目标(技术跑通)**:
1. 鼠标点击移动(Dota 式,非 WASD)
2. 自动攻击(最近敌人)
3. 召唤兵 AI(自动在队长周围攻击,不可指挥)
4. 人口系统(初始 8,满人口禁召)
5. 召唤兵生命周期(持续时间到时死亡)
6. 一个召唤兵技能能跑起来

**Demo 第二阶段目标(玩法好玩)**:
1. 三选一升级池(10–15 项)
2. 2–3 个召唤兵技能 + 1–2 个被动触发技能
3. 队长主动技能(狂暴 + 召唤组合)
4. 杂兵 + 精英敌人
5. 一关完整胜负(守基地或攻据点)
