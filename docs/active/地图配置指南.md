# 地图配置指南

在 Godot 编辑器中调整地图的配置，按以下层级操作。覆盖 map_1 ~ map_20 全部关卡。

## 配置入口总览

| 配置层级 | 在编辑器中选什么 | 控制什么 |
|---------|--------------|---------|
| **地图基础配置** | 根节点 Main → Inspector 中的 Map Config | 金币、单位、建筑、AI队友、环境物、地图边界 |
| **波次配置** | WaveManager 节点 | 敌人波次、时间、数量、目标位置 |
| **胜利条件配置** | VictoryXxx 节点 | 摧毁城堡/生存/暗杀/多阶段等 |
| **占领据点配置** | CapturePoint 节点 | 光圈位置、占领半径、奖励、AI增援、连续波次 |

> **类型对照表**
> - 单位 type: 0=士兵, 1=弓箭手, 2=长矛兵, 3=僧侣
> - 建筑 type: 0=墙, 1=塔, 2=城堡, 3=兵营, 4=修道院, 5=射箭场
> - 可建造 PlaceMode ID: 0=墙, 1=塔, 2=城堡, 3=兵营, 4=士兵, 5=弓箭手, 6=修道院, 7=射箭场, 8=长矛兵, 9=僧侣

> **建筑 grid_pos 说明**
> - 1x1 建筑（墙、塔）：`grid_pos = Vector2i(x, y)`，世界坐标 = `grid * 64 + 32`
> - 2x2 建筑（兵营、修道院、射箭场）：世界坐标 = `grid * 64 + 64`
> - 3x3 建筑（城堡）：世界坐标 = `grid * 64 + 96`

## 一、地图基础配置 (MapConfig)

**操作**: 选择根节点 `Main` → Inspector 中 `Map Config` 属性，展开编辑。

| 属性 | 说明 | Map 1 (闪电战) | Map 2 (基础进攻) | Map 3 (塔防生存) | Map 4 (扩张防守) |
|-----|------|------------|-------------|-------------|-------------|
| initial_gold | 开局金币 | 0 | 10,000 | 6,000 | 8,000 |
| camera_start | 摄像机初始位置 | (500, 350) | (500, 350) | (500, 350) | (500, 600) |
| map_bounds | 地图边界 Rect2 | (-500,-500,2000,1700) | (-500,-500,2000,1700) | (-500,-500,2000,1700) | (-500,-500,2500,1700) |
| player_units | 玩家初始单位 | 3兵+2弓 | 3兵+2弓 | 3兵+2弓 | 3兵+2弓 |
| enemy_units | 敌方初始单位 | 12个 | 3兵+2弓 | 无(波次生成) | 15个 |
| player_buildings | 玩家初始建筑 | 无 | 城堡+兵营 | 城堡+兵营+修道院+塔 | 城堡+兵营 |
| enemy_buildings | 敌方初始建筑 | 1塔+2墙 | 城堡+兵营+塔+2墙 | 无 | 4塔 |

### 详细字段说明

- **player_units / enemy_units** — 单位数组，每项 `{"type": int, "pos": Vector2}`
- **player_buildings / enemy_buildings** — 建筑数组，每项 `{"type": int, "grid_pos": Vector2i}`
- **environment** — 环境物 `{trees: int, rocks: int, bushes: int, sheep: int}`
- **available_items** — 可建造/生产的 PlaceMode ID 数组
- **nav_bounds** — 导航区域顶点数组（改地图大小时需同步调整）

## 一·五、AI 队友配置 (ai_allies)

> 仅单机模式有效。多人模式自动忽略。盟友为黄色，owner_id=-2，alliance_id=0。

**ai_allies** 是 MapConfig 中的数组，格式与 player_units 一致，但可扩展以下可选字段：

| 字段 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| type | int | 必填 | 单位类型（0=士兵, 1=弓兵, 2=枪兵, 3=僧侣） |
| pos | Vector2 | 必填 | 初始位置 |
| behavior | String | "follow" | `"follow"`=跟随玩家 / `"defend"`=驻防 / `"attack_move"`=攻击移动 |
| defend_pos | Vector2 | Vector2.ZERO | behavior="defend" 时的驻防位置 |
| squad_id | String | "general" | 小队标识，用于分组指挥 |

### 配置示例

```
# 跟随玩家（最常见）
{"type": 0, "pos": Vector2(300, 250)}

# 驻防某点（分兵关卡用）
{"type": 0, "pos": Vector2(850, 130), "behavior": "defend", "defend_pos": Vector2(900, 130), "squad_id": "north"}

# 开局攻击移动到某点
{"type": 1, "pos": Vector2(200, 300), "behavior": "attack_move", "defend_pos": Vector2(800, 300)}
```

### 分组指挥

在场景中通过代码触发分组指挥（不在 MapConfig 中配置）：

- `AllyCommander.issue_squad_attack_order(squad_id, world_pos)` — 指定小队转为攻击
- `AllyCommander.issue_squad_defend_order(squad_id, world_pos)` — 指定小队转为防御

占领点被占领时自动触发：在 CapturePoint 节点名为 `RallyCP` 且设置了 `meta/rally_attack_target` 时，占领后自动对 `south` 小队下达攻击指令。详见下方占领据点章节。

## 二、波次配置 (WaveManager)

**操作**: 在场景树中选择 `WaveManager` 节点 → Inspector 中编辑。

> 仅 **Map 3** 和 **Map 4** 有此节点。

### 全局属性

| 属性 | 说明 | Map 3 | Map 4 |
|-----|------|-------|-------|
| clear_then_next | 清完当前波次才出下一波 | true | false（按时间自动出） |

### 每波字段 (waves 数组中每一项)

| 字段 | 类型 | 含义 |
|-----|------|------|
| delay | float | 波次开始延迟（秒） |
| post_clear_delay | float | 清完后到下一波间隔（秒，仅 clear_then_next=true 有效） |
| wave_attack | bool | 是否生成后立即向目标移动 |
| wave_target | Vector2 | 敌人进攻目标位置 |
| groups | Array[{type, count}] | 本波敌人编组（按类型和数量） |
| spawn_point_path | NodePath | 生成点 Marker2D 的路径（如 `NodePath("../NorthWaveSpawn")`） |
| formation | String | 阵型：`"column"` 竖列 / `"grid"` 方阵 |
| spacing | float | 单位间距（像素） |

### Map 3 波次表（3波，清完出下一波）

| 波次 | delay | 单位数 | 组成 | post_clear_delay | 进攻目标 |
|-----|-------|-------|------|-----------------|---------|
| 1 | 30s | 5 | 3兵+2弓 | 60s | (200, 350) |
| 2 | 60s | 9 | 5兵+3弓+1矛 | 60s | (200, 350) |
| 3 | 90s | 15 | 8兵+4弓+3矛 | 0s | (200, 350) |

### Map 4 波次表（5波，按时间自动出）

| 波次 | delay | 单位数 | 进攻目标 |
|-----|-------|-------|---------|
| 1 | 45s | 5 | (500, 600) |
| 2 | 90s | 7 | (500, 600) |
| 3 | 120s | 11 | (500, 600) |
| 4 | 150s | 14 | (500, 600) |
| 5 | 180s | 20 | (500, 600) |

## 三、胜利条件配置

### Map 1 — VictoryBlitz（闪电战：占领据点）

**选择节点**: `VictoryBlitz`

| 属性 | 说明 | 当前值 |
|-----|------|-------|
| capture_points | 据点 NodePath 数组 | 3个据点 |
| activation_radius | 据点激活检测半径 | 200 |

**胜利条件**: 占领全部 3 个据点
**失败条件**: 所有玩家单位死亡
**特殊**: 每占领一个据点，全体玩家单位回复 50% HP

### Map 2 — VictoryDestroyBase（摧毁敌方城堡）

无需额外配置，胜利条件是摧毁敌方城堡，失败条件是己方城堡被毁。

### Map 3 — VictorySurviveWaves（塔防生存）

**选择节点**: `VictorySurviveWaves`

| 属性 | 说明 |
|-----|------|
| wave_manager_path | WaveManager 的 NodePath |

**胜利条件**: 存活过全部波次 + 消灭所有敌人
**失败条件**: 己方城堡被毁

### Map 4 — VictoryExpandDefense（扩张防守）

**选择节点**: `VictoryExpandDefense`

| 属性 | 说明 | 当前值 |
|-----|------|-------|
| wave_manager_path | WaveManager 的 NodePath | ^"WaveManager" |
| capture_points | 据点 NodePath 数组 | 4个据点 |
| activation_radius | 敌人清除检测半径 | 250 |

**胜利条件**: 占领全部 4 个据点 + 存活过全部波次 + 消灭所有敌人
**失败条件**: 己方城堡被毁

## 四、占领据点配置 (CapturePoint)

**操作**: 在场景树中选择 CapturePoint 节点 → Inspector 中编辑。

### 基础属性

| 属性 | 说明 | 默认值 |
|-----|------|-------|
| trigger_type | 触发类型：CLEAR_AREA(清敌) / KILL_TARGETS(杀目标) / MISSION_COUNTER(杀敌计数) | CLEAR_AREA |
| detection_area | 检测区域中心 Marker2D 的 NodePath | 自身位置 |
| detection_radius | 检测半径（清除该范围内敌人后激活） | 200 |
| capture_radius | 占领检测半径 | 80 |
| capture_speed | 每秒占领进度 | 50 |
| can_enemy_capture | 敌方是否也能占领 | true |
| allow_recapture | 是否允许反复争夺 | false |
| enabled_on_start | 开局即激活 | true |

### 奖励属性

| 属性 | 说明 | 默认值 |
|-----|------|-------|
| reward_gold | 占领后金币奖励 | 0 |
| reward_soldiers/archers/lancers/monks | 占领后直接生成的单位 | 0 |

### AI 队友增援属性

| 属性 | 说明 | 默认值 |
|-----|------|-------|
| ally_reinforcement_delay | 占领后延迟生成增援的秒数（0=无增援） | 0.0 |
| ally_reinforcement_groups | 增援编组 `[{type, count}]` | [] |
| ally_reinforcement_repeat | 连续增援额外波数（0=仅一波，>0=额外N波） | 0 |
| ally_reinforcement_interval | 每波间隔秒数 | 30.0 |

> **增援单位行为**: 生成后自动 attack_move 向最近的敌方建筑/单位。

### 集结点阶段切换（关卡4专用）

在 CapturePoint 节点上设置 meta 属性可实现占领后自动触发小队转攻：

1. 节点命名为 `RallyCP`
2. 添加 meta：`meta/rally_attack_target = Vector2(x, y)` （目标坐标）
3. 占领后自动调用 `AllyCommander.issue_squad_attack_order("south", target)`

### 连续增援示例

```
# 占领后5秒来第1波，30秒第2波，55秒第3波（共3波×5单位=15个）
ally_reinforcement_delay = 5.0
ally_reinforcement_groups = [{"type": 0, "count": 3}, {"type": 1, "count": 2}]
ally_reinforcement_repeat = 2        # 额外2波（总共3波）
ally_reinforcement_interval = 25.0   # 每波间隔25秒
```

## 五、快速调整指南

### 改敌人数量和位置
- **初始敌人**: 根节点 Main → Map Config → enemy_units，修改 type 和 pos
- **波次敌人**: WaveManager → waves → 对应波次的 units，增删条目、改 type 和 pos

### 改光圈（据点）位置
- 场景树选中 CapturePoint 节点 → 2D 视图中拖动
- 同步调整 capture_radius、capture_speed、reward_gold 等

### 改波次时间
- WaveManager → waves → 修改 delay 和 post_clear_delay

### 改敌人进攻目标
- WaveManager → waves → 修改 wave_target (Vector2)

### 改开局金币
- 根节点 Main → Map Config → initial_gold

### 改可用建筑/兵种
- 根节点 Main → Map Config → available_items，添加/删除 PlaceMode ID

### 改地图大小
- 根节点 Main → Map Config → map_bounds
- 同步调整 nav_bounds 和场景中的 NavigationRegion2D

## 六、手写 .tscn 场景文件注意事项

> 如果用编辑器创建场景可跳过本节。仅当手动编写 .tscn 文件时需要关注。

**每个非根节点必须有 `parent` 属性**，否则场景加载到60%后静默卡死（无任何报错）：

```
# 正确 — 根的直接子节点
[node name="Ground" type="ColorRect" parent="."]

# 正确 — 嵌套子节点
[node name="Stage0" type="Node" parent="VictoryMultiStage"]

# 错误 — 缺少 parent，场景实例化静默失败
[node name="Ground" type="ColorRect"]
```

### 标准场景骨架

一个配置驱动生成的关卡场景（无预放置单位）需要以下节点：

- `Ground` (ColorRect) — 背景色块
- `NavigationRegion2D` — 导航网格
- `Camera2D` — 摄像机
- `PlayerUnits` / `EnemyUnits` / `Buildings` (Node2D) — 空容器（由 config 填充）
- `SelectionBox` / `PreviewRect` (ColorRect) — UI 辅助
- `ResultLabel` (Label) — 胜负显示
- `AttackMoveIndicator` (Label) — 攻击移动指示
- `VictoryXxx` (Node) — 胜利条件
- 可选：`WaveManager` + `*WaveSpawn` (Marker2D) — 波次系统
- 可选：`CapturePoint` (Area2D) + `DetectionArea` (Marker2D) — 占领据点

## 七、胜利条件选型建议

| 胜利条件 | 适用场景 | 注意事项 |
|---------|---------|---------|
| VictoryDestroyBase | 最常用。摧毁敌方城堡即胜 | 自动检测玩家城堡存活为失败条件，无需配 NodePath |
| VictoryDestroyBuildings | 摧毁指定建筑 | 配置驱动生成时用 `target_building_type` 按类型扫描，不用 NodePath |
| VictorySurviveWaves | 塔防生存 | 需要 WaveManager |
| VictoryMultiStage | 多阶段任务 | **避免在 Stage0 用 VictorySurviveTimer**（见下方警告） |
| VictoryComposite | 组合条件（AND/OR） | 子节点为其他 Victory 条件 |

> **设计陷阱**: VictorySurviveTimer（强制生存N秒）会阻止玩家提前通关。如果关卡已有 WaveManager 提供时间压力，直接用 VictoryDestroyBase。玩家提前打完敌人就应该直接赢。
