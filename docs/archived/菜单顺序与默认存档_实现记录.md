## Context

当前流程：**主菜单 → 开始游戏 → 存档选择 → 关卡选择**，每次都要在存档选择界面点一次，对常用单档玩家是多余操作。

目标流程：**主菜单 → 开始游戏 → 关卡选择**（直接读上次活动存档）；切换存档需求通过主菜单新增的"切换存档"按钮进入。

已确认决策：
- **默认存档策略**：记住上次活动存档（持久化到 `settings.cfg` 的 `[save]` section），首次进入用 `slot_0` 兜底
- **开始游戏按钮文字**：保持只显示"开始游戏"
- **按钮顺序**：开始游戏 → 切换存档 → 多人 → 设置 → 退出

**文件**：[scripts/systems/save_manager.gd](scripts/systems/save_manager.gd)

### 1a. 新增常量（插入位置：line 8 `const SLOTS := 3` 之后）

### 1b. 新增 3 个函数（插入位置：line 156 `save_current_data` 函数之后、`# === 游戏会话追踪 ===` 注释之前，约 line 157）

```gdscript
# === 默认存档（快速进入）===

func get_default_slot() -> int:
	var config := ConfigFile.new()
	if config.load(SETTINGS_PATH) == OK:
		var slot: int = config.get_value("save", "last_active_slot", 0)
		if slot >= 0 and slot < SLOTS:
			return slot
	return 0


func set_default_slot(slot: int) -> void:
	var config := ConfigFile.new()
	config.load(SETTINGS_PATH)   # 不存在也无所谓，下面 save 会创建
	config.set_value("save", "last_active_slot", slot)
	config.save(SETTINGS_PATH)


# 一键进入默认存档：设 current_slot → 不存在则初始化空档 → 写回记录
func enter_default_slot() -> void:
	var slot := get_default_slot()
	set_current_slot(slot)
	if not slot_exists(slot):
		save_slot(slot, _empty_slot())
	set_default_slot(slot)
```

**复用的现有 API**：
- `set_current_slot(slot)`（line 141）
- `slot_exists(slot)`（line 40）
- `save_slot(slot, data)`（line 63）
- `_empty_slot()`（line 80）

**为什么不直接在 `set_current_slot` 里同步写 settings.cfg**：保持单一职责。`set_current_slot` 是内存状态，`set_default_slot` 是持久化状态，分开调用更清晰，也避免 save_select 在某些调试路径下意外写盘。

**布局调整**（line 138-142）：

### 2c. 改写 `_on_start_pressed`（替换 line 229-230）

**旧代码**：
```gdscript
func _on_start_pressed() -> void:
	get_tree().change_scene_to_file("res://scenes/save_select.tscn")
```

**新代码**：
```gdscript
func _on_start_pressed() -> void:
	var sm := get_node_or_null("/root/SaveManager")
	if sm and sm.has_method("enter_default_slot"):
		sm.enter_default_slot()
	get_tree().change_scene_to_file("res://scenes/level_select.tscn")
```

**为什么用 `has_method` 守卫**：SaveManager 是 autoload（project.godot 已配置），理论上一定存在；但 `enter_default_slot` 是新方法，万一 SaveManager 脚本未热重载，避免运行时崩。降级路径：直接跳 level_select，level_select 自己有对 `_current_slot < 0` 的兜底（save_manager.gd:146 的 `get_current_data` 返回空档）。

## 改动 3：存档选择界面同步默认档

**文件**：[scripts/ui/save_select.gd:396-402](scripts/ui/save_select.gd#L396-L402)

**旧代码**：
```gdscript
func _on_slot_selected(slot: int) -> void:
	var sm := _get_save_manager()
	sm.set_current_slot(slot)
	# 如果是新存档，先初始化
	if not sm.slot_exists(slot):
		sm.save_slot(slot, sm.load_slot(slot))
	get_tree().change_scene_to_file("res://scenes/level_select.tscn")
```

**新代码**（加一行 `set_default_slot`，让"在存档选择界面选过的档"成为下次开始游戏的默认档）：
```gdscript
func _on_slot_selected(slot: int) -> void:
	var sm := _get_save_manager()
	sm.set_current_slot(slot)
	sm.set_default_slot(slot)   # ← 新增：本次选择记为下次默认
	# 如果是新存档，先初始化
	if not sm.slot_exists(slot):
		sm.save_slot(slot, sm.load_slot(slot))
	get_tree().change_scene_to_file("res://scenes/level_select.tscn")
```

## 用户配置说明

- **默认行为（零配置）**：游戏自动读 `user://settings.cfg` 的 `[save] last_active_slot`
- **首次进入**：自动落到 `slot_0`，不存在则创建空档（沿用 save_select 选新档的同款初始化）
- **切换默认档**：主菜单 → "切换存档" → 选目标档 → 自动记为新默认；下次点"开始游戏"直接进该档
- **重置记忆**：删除 `%APPDATA%\Godot\app_userdata\<项目名>\settings.cfg` 中的 `[save]` 段
