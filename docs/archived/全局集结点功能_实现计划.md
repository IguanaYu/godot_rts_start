# 全局玩家集结点功能

## Context

项目里已有"每个兵营各自一个集结点"的实现（`building.gd` 的 `set_rally_point` / 选中建筑右键 / 产兵后 `move_to`），但和实际需求对不上：

1. 现有产兵用 `move_to`（**纯移动不攻击**），需要 `attack_move_to`（移动并攻击）
2. 现有是**每兵营各自**，需要**全局一个**（面板召唤单位不属于任何兵营，只有全局能覆盖）
3. 没有"放置瞬间让现有单位立刻出发"的能力

需求确认：**全局一个集结点** + 放置瞬间**相机视野内所有玩家单位**立刻 attack_move + **Y 键**进入放置模式 + 后续兵营产兵/面板召唤自动 attack_move 到集结点。

## 改动 5：main.gd — 新增 `_set_global_rally` 和 `_rally_units_in_view` 方法

**位置**：在 main.gd 第 1270 行 `func spawn_unit_near` 之前（或文件末尾任意函数位置）插入两个新方法。复用 [game_camera.gd:132](../../scripts/systems/game_camera.gd#L132) `get_camera_view_rect()` 和 [unit.gd:1093](../../scripts/units/unit.gd#L1093) `attack_move_to`：

```gdscript
func _set_global_rally(pos: Vector2) -> void:
    global_rally_point = pos
    has_global_rally = true
    if _rally_indicator == null:
        _rally_indicator = Node2D.new()
        _rally_indicator.set_script(load("res://scripts/effects/rally_point_indicator.gd"))
        _rally_indicator.z_index = 5
        add_child(_rally_indicator)
    _rally_indicator.setup_global(pos)
    # 弹入动画
    _rally_indicator.scale = Vector2(0.5, 0.5)
    var tween := create_tween()
    tween.tween_property(_rally_indicator, "scale", Vector2.ONE, 0.25) \
        .set_trans(Tween.TRANS_BACK).set_ease(Tween.EASE_OUT)
    # 视野内所有玩家单位立刻 attack_move
    _rally_units_in_view(pos)


func _rally_units_in_view(pos: Vector2) -> void:
    var view_rect: Rect2 = camera_module.get_camera_view_rect()
    for u in get_tree().get_nodes_in_group("player_units"):
        if is_instance_valid(u) and not u.is_dead() and view_rect.has_point(u.global_position):
            u.attack_move_to(pos)
```

完整改后：
```gdscript
func spawn_unit_near(type: int, pos: Vector2, team: int) -> CharacterBody2D:
    var offset := Vector2(randf_range(-50, 50), randf_range(-50, 50))
    var unit := create_unit(type, team, pos + offset)
    unit.net_id = _next_net_id
    _next_net_id += 1
    LockstepSync.register_unit(unit)
    if team == UnitScript.Team.PLAYER:
        _player_units_node.add_child(unit)
        unit.add_to_group("player_units")
        if _upgrade_manager:
            _upgrade_manager.apply_all_stat_upgrades_to_unit(unit)
        spawn_spawn_effect(pos + offset, team, unit)
    else:
        _enemy_units_node.add_child(unit)
        spawn_spawn_effect(pos + offset, team, unit)
        unit.add_to_group("enemy_units")
        var ai := Node2D.new()
        ai.name = "EnemyAI"
        ai.set_script(load("res://scripts/units/enemy_ai.gd"))
        unit.add_child(ai)
    return unit
```

> 风险：6 处调用者（victory_hold_point / upgrade_manager / boss_ai / adaptive_reinforcement / ambush_trigger / capture_point）都是当语句用，GDScript 忽略返回值合法，不受影响。

    var units: Array = config.get("units", [])

    # 生成单位
    var spawned: Array = []
    for unit_type in units:
        var u = spawner_module.spawn_unit_near(unit_type, target_pos, UnitScript.Team.PLAYER)
        if u:
            spawned.append(u)

    # 空投特效：灰尘
    spawner_module.spawn_dust_effect(target_pos)

    # 浮动文字
    spawner_module.show_floating_text("Unit Drop", Color(0.3, 0.8, 1.0), target_pos)

## 验证方法

启动：
```
"E:\godot\Godot_v4.6.1-stable_win64.exe" --path "F:\godot_game\rts_base\rts-base"
```

测试步骤：
1. 进入关卡，造 2-3 个兵营等产兵，把部分单位移到屏幕边缘外
2. **按 Y** → 光标变攻击样式 → **左键点地**：
   - 视野内己方单位立刻朝该点 attack_move（屏幕外单位不动 ✓）
   - 路径上放敌兵，确认单位会停下来打（验证是 attack_move 不是纯 move）
   - 旗帜指示器出现在点击点
3. 让兵营再产兵 → 新兵出来后自动 attack_move 到集结点
4. 用指挥官技能空投（V 键 UNIT_DROP）→ 空投单位自动 attack_move 到集结点
5. **右键退出 Y 模式**；选中一个兵营，**右键地面** → 也能更新全局集结点（旗帜位置移动）
6. 按 Q/W/A 确认 Y 模式被正确取消（互斥）
7. 按 Y 再按 Y（toggle）→ 退出模式不放置
8. 切换关卡，确认移除 building 的 set_rally_point 后不报错
