## 0. 关卡概览

| 项 | 值 |
|----|----|
| 关卡名 | Tomb Defense（古墓防御） |
| 地图尺寸 | 2000×1700 |
| 初始金币 | **5000** |
| 可用物品 | 全兵种+建筑（8种） |
| 指挥官技能 | [0,1,2] |
| 初始玩家部队 | 3S+2A+1 Monk（**1010**） |
| 初始玩家建筑 | Castle + Barracks + Monastery |
| 敌方 | 2个 Tomb（古墓）各3守卫 + 4波敌人 |
| 胜利条件 | 存活4波 |

### 布局示意

```
玩家基地                 Tomb1(600,300)    Tomb2(600,700)
  (100,200)              3S守卫             3S守卫
  Castle+Barracks+Monk   CP:400金+1Monk     CP:500金+1Monk
                          (detection=150)    (detection=150)

波次刷点：W1/W3来自WaveSpawnUpper(1400,200)
          W2/W4来自WaveSpawnLower(1400,800)
```

### 波次序列

| 波次 | 延迟 | 兵力 | 战力 |
|------|------|------|------|
| W1 | 50s | 5S+3A | 1320 |
| W2 | 30s | 7S+4A | 1810 |
| W3 | 25s | 8S+5A+2L | 2530 |
| W4 | 20s | 10S+6A+3L | 3135 |

### WaveModifier：占领古墓会增强后续波次

每个古墓使W3+3S+2A，W4+3S+2A。

---

## 1. 维度 A：空间布局 ⚠️

**A1**：玩家(100,200) ↔ Tomb1(600,260) = **500px** < 700px ❌

**A2**：Tomb1(600,300) ↔ Tomb2(600,700) = **400px** < 800px ❌

波次从1400,200和1400,800来，离玩家1000+px，距离充裕。

---

## 2. 维度 B：据点战力 ✅

Tomb守卫 3S×2 = 900战力，玩家1010+5000金可轻松拿下。

但**占领古墓会增强波次**，这是关键风险——占Tomb拿奖励的同时波次更强，决策权衡。

---

## 3. 维度 C：节奏曲线 ✅ 设计亮点

**C3**：map6(1500)→map7(5000)，Δ=+3500 ✅ 刚好达标

**核心设计：占墓增强波次的权衡**
- 每个Tomb给400-500金 + 1 Monk
- 但使W3/W4各+3S+2A
- 玩家需决策：占墓拿资源（但波次变强）vs 不占（但少资源）
- **"古墓增强波次"是机制关，节奏由玩家选择决定**

---

## 4. 维度 D：经济续战 ✅

- 5000金充裕 + Castle产金 + Barracks产兵
- 全兵种可用，可建塔防守
- 有Monastery可产Monk治疗

---

## 5. 维度 E：兵种目标 ✅

- 全兵种可用，自由度最高的一关之一
- 单一目标（存活），无冲突
- 指挥官技能3个可选

---

## 6. 维度 F：AI 技术 ⚠️

- Tomb守卫间距400px，会互相拉怪（但同属古墓区域，可能是设计意图）
- 波次路径清晰
- WaveModifier 动态调节波次强度，机制复杂但设计精良
