# 技能系统重构 — 需求与设计文档

---

## 一、现状

### 1.1 技能系统现状
- **没有"技能"这个抽象概念**。所有能力散落在 `unit.gd` 各处，各自为政
- 攻击能力（锥形AoE、链式闪电、召唤）硬编码在 `_perform_attack()` 中，互斥不可组合
- 主动能力（嘲讽、劝化、隐身、闪现）各自在 `_physics_process` 里跑独立逻辑
- 被动属性（闪避、回复、减伤）通过 `stat_set` 修饰器实现，相对干净
- Monk 治疗有独立的 `HEAL` 状态机，和 `ATTACK` 并列
- 所有能力通过 `UnitStats` 中的零值字段判断"是否有此能力"，非零即存在

### 1.2 视觉交付现状
- 近战攻击直接调用 `take_damage()`，无视觉弹道
- Archer 和投射单位走弹道（已有 `arrow.gd` + `ProjectileData` 框架）
- 链式闪电和锥形AoE 是代码绘制的 Line2D/Polygon2D
- **召唤骷髅是凭空出现的**，没有任何前置视觉
- 所有指挥官技能都是即时生效，无弹道

### 1.3 情绪系统现状
- 每个敌方单位自带 `unit_emotion.gd` 子节点
- 半血以下概率触发 5 种情绪（暴怒/咆哮/呼救/驻守/恐惧）
- 各自有独立 CD（30s），各自决策，互不协调
- 没有统一的资源管理

### 1.4 敌方 AI 现状
- 有 `patrol_center` 和 `patrol_radius`，但**没有追击距离限制**
- 敌人一旦进入 CHASE 状态会追到天涯海角
- 玩家无法通过操作技巧"拉兵"

## 二、需求

### 2.1 技能框架统一（核心需求）
- 建立可插拔的技能组件体系，单位可以自由增删技能
- 技能数据用 .tres 配置驱动，代码只实现通用逻辑
- 技能由以下要素组成：**触发条件 + 目标选择 + 交付方式 + 命中效果 + 冷却/资源管理**

### 2.2 效果交付规范化
- 需要"飞到目标"的技能（召唤、治疗、劝化、护盾）统一走弹道交付
- 解决"凭空出现"的视觉问题
- 嘲讽、闪现、隐身等范围型/自身型技能保持即时

### 2.3 蓝条资源系统
- 有技能的单位消耗蓝条（仅法师和射手类型，用枚举标识，可扩展）
- 近战单位无蓝条
- 蓝条在 HP 条下方显示

### 2.4 技能释放可视化
- 单位触发技能时头顶弹出技能名浮动文字
- 便于测试和玩家理解

### 2.5 自动攻击/技能释放逻辑重构
- 统一 AI 决策循环：优先检查技能触发条件，再检查攻击
- 技能有优先级：治疗 > 劝化 > 嘲讽 > 护盾 > 攻击
- 目标选择：选最近的；治疗选最近且需要治疗的单位（不满血）
- 技能使用冷却和蓝耗控制频率

### 2.6 Monk 治疗统一
- Monk 的独立 HEAL 状态机移除，统一到技能框架

### 2.7 敌方拉兵系统
- 参考 SC2/War3/Dota2 的 leash 机制
- 每个敌人有最大追击距离（leash range），超出则回头
- 允许玩家通过操作勾引少数敌人出来打

### 2.8 据点指挥官系统（后续）
- 每个据点有一个隐形单位"据点指挥官"，不可选中/攻击
- 指挥官有蓝条 = 士气值，建筑越多初始蓝越多
- 有辐射区域，区域内的敌方单位被攻击时，指挥官可指派技能
- 建筑被摧毁 → 蓝上限下降；建筑全灭 → 指挥官消灭
- 情绪技能迁移到指挥官，按需消耗蓝量释放
- 蓝量耗尽后部分单位触发恐惧逃跑
- 后续可扩展：造兵、主动进攻等

## 三、改动点汇总

### 3.1 新增文件
| 文件 | 说明 |
|------|------|
| `scripts/skills/skill_resource.gd` | 技能数据 Resource（.tres 配置） |
| `scripts/skills/skill_component.gd` | 技能运行时组件（挂载到单位） |

| 文件 | 改动 |
|------|------|
| `scripts/units/unit.gd` | 集成 SkillComponent，技能AI决策，移除旧独立process，移除HEAL状态机，蓝条管理 |
| `scripts/stats/unit_stats.gd` | 新增 skills 数组、mana_type、max_mana、mana_regen 字段 |
| `scripts/effects/arrow.gd` | 支持命中时执行任意回调 |

| 文件 | 说明 |
|------|------|

## 四、阶段性目标

### 第一阶段目标：视觉基础
**解决"看不到技能在放"的问题**
- 单位触发技能时头顶弹出绿色技能名浮动文字
- 召唤骷髅改为弹道落地生成，不再凭空出现
- 弹道框架支持任意命中回调

### 第二阶段目标：技能框架
**建立可插拔的技能体系**
- SkillResource + SkillComponent 框架搭建
- 7 个技能效果迁移到新框架
- 蓝条系统上线
- 旧情绪系统移除
- 所有原有功能正常运作

### 第三阶段目标：AI 智能
**单位自动决策更合理**
- 统一 AI 决策循环（技能 > 攻击 > 待机）
- 技能优先级系统
- 敌方 leash 拉兵机制
- 目标选择策略完善

### 第四阶段目标：据点指挥官
**情绪系统升级为据点级资源战**
- 指挥官隐形单位 + 辐射区域
- 士气机制（建筑→蓝上限）
- 情绪技能回归，指挥官统一管理
- 拉兵与士气联动

## 五、设计原则

1. **可插拔**：技能是 Resource + Component 组合，单位可自由增删
2. **渐进改造**：每一轮改动不影响现有功能，可独立测试和上线
3. **配置驱动**：技能行为由 .tres 决定，代码只实现通用框架
4. **弹道统一视觉**：需要飞到目标的技能走弹道，解决凭空出现
5. **玩家操作空间**：允许拉兵等微操技巧，不强制玩家硬碰硬
